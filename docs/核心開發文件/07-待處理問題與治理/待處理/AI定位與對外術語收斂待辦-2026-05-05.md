## 1. 裁決

術語表與近期提交已把對外定位收斂為「AI 關係梳理助手」與「梳理結果」，並明確 AI 不替代持牌治療師；但代碼中仍有多處對外或 prompt 層殘留「AI 心理師」「治療師」「臨床心理師」「深度剖析」等強專業定位。這是**代碼落後於核心文件與產品定位**。

內部 API、DB、路由與型別可繼續保留 `judgment / plaintiff / defendant`，但用戶可見文案與 AI 角色設定不得把系統表述為心理師、治療師、法院或仲裁者。

## 2. 已知殘留

目前只讀審計已看到以下候選，實作時需全量掃描，不限於本清單：

1. `frontend/src/assets/i18n/zh-TW.ts`：`AI 心理師分析中`、`心理師的反饋`、`AI 心理師深度分析中`、`兩人的陳述一起提交給 AI 心理師進行深度分析` 等。
2. `backend/src/services/ai.service.ts`：`資深的伴侶關係治療師`、`資深伴侶治療師`、`真正的治療師`、`深度分析` 等 prompt 殘留。
3. `backend/src/services/domain-classification.service.ts`：`你是一位臨床心理師`。
4. `backend/src/services/interview-prompt-utils.ts`：雖已聲明不是持牌心理師，但仍有「好的治療師像水一樣跟隨」等角色比喻，需改成非身份化描述。

## 3. 目標口徑

1. 對外角色：AI 關係梳理助手 / AI relationship guide / safe listening space。
2. 對外結果：梳理結果 / 分析結果 / 關係整理，不用「判決」作主文案。
3. 對外安全聲明：AI 僅供自我探索與溝通整理，不構成心理診斷、法律意見或專業治療。
4. Prompt 可引用 NVC、EFT、Gottman、ACT 等理念，但不得自稱治療師、臨床心理師或專業治療提供者。
5. 風險/安全場景仍應建議尋求持牌專業支持，但語義是 referral，不是 AI 自己扮演該角色。
