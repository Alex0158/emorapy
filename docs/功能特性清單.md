## 1. 核心模組（已上線）

- **快速體驗**：快速建案、結果頁、補證據、session 刷新/關聯。
- **正式案件流程**：Case -> Judgment -> Reconciliation -> Execution 全鏈路。
- **心理畫像訪談**：consent、SSE 對話、async pipeline、my-story、feedback、刪除資料。
- **聊天室 Chat v1**：建房邀請、訊息流、SSE、轉判決、離房/踢人、判決狀態。
- **管理台**：admin RBAC、配置、審計、報表、jobs、alerts、feature flags。

## 4. 關鍵流程能力（實作口徑）

## 4.1 訪談流程

1. start（需 consent，且受限流/業務限額）  
2. respond/skip（SSE）  
3. end（完成或進 processing）  
4. pipeline 五步處理  
5. 失敗可 retry  

## 4.2 聊天轉判決流程

1. 建房與邀請  
2. 消息往返與 SSE  
3. A 發起 request-judgment  
4. 後端按可見性與 included_message_ids 裁切輸入  
5. 生成 case/judgment 並回填房間狀態  

## 4.3 快速體驗流程

1. 建 session  
2. 建 quick case  
3. 可補證據  
4. 拉取判決  
5. 引導到正式模式  

---

## 5. 重要約束（當前默認）

- 訪談：max turns 25、soft target 15、turn interval 3s
- 訪談：業務每小時/每日上限 + start/respond 中間件限流
- 聊天：最小發訊間隔 5s、30s 最多 6 則
- metrics：可關閉；production 需 token 或白名單 IP
