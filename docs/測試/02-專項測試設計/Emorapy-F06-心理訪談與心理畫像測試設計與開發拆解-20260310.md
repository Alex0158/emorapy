對應流程：`P03 心理訪談閉環`

## 1. 本次測試目標

F06 的目標是將 consent、start、respond、skip、end、retry、delete 整條鏈路做成可回歸、能揭露業務缺陷的測試覆蓋。本輪優先驗證的業務規則：

1. **consent**：未 consent 不得 start；giveConsent 後 user.psych_consent_given 為 true。
2. **start**：需 consent、每日/每小時限額、放棄舊 IN_PROGRESS session 建立新 session。
3. **respond/skip**：需 session 屬 userId、status=IN_PROGRESS、未達 maxTurns、未違反 turnInterval。
4. **end**：IN_PROGRESS → PROCESSING 或 COMPLETED；內容不足時可直接 COMPLETED。
5. **retry**：PROCESSING_FAILED → PROCESSING，可重試 pipeline。
6. **delete**：deleteAllData 清除 narratives、insights、turns、sessions，並重置 consent。

## 2. 代碼真實規則（以實際代碼為準）

### 2.1 功能/頁面/API 基線

依據：

1. `docs/核心開發文件/功能特性清單.md` F06
2. `docs/核心開發文件/業務流程整合.md` P03
3. `docs/核心開發文件/接口-功能-頁面-Mapping.md`

F06 主鏈路：

1. `POST /api/v1/psych-profile/consent` — 同意授權
2. `POST /api/v1/interview/start` — 啟動訪談
3. `GET /api/v1/interview/:id` — 訪談詳情
4. `POST /api/v1/interview/:id/respond`（SSE）— 回答
5. `POST /api/v1/interview/:id/skip`（SSE）— 跳題
6. `POST /api/v1/interview/:id/end` — 結束訪談
7. `GET /api/v1/interview/:id` — 結果讀取
8. `POST /api/v1/interview/:id/retry` — 重試失敗 session
9. `GET /api/v1/psych-profile` — 讀心理畫像
10. `GET /api/v1/psych-profile/feedback` — 讀回饋歷史
11. `DELETE /api/v1/psych-profile` — 刪除心理資料

### 2.2 psych-profile.service 真實規則

核心位於 `backend/src/services/psych-profile.service.ts`：

| 方法 | 業務規則 | 錯誤碼 |
|------|----------|--------|
| getProfile | 需 userId 存在；查 narratives、insights、richness_score | NOT_FOUND |
| getFeedbackHistory | 查 COMPLETED 且 feedback_card 非空 sessions，最多 50 筆 | — |
| giveConsent | 更新 user.psych_consent_given=true、psych_consent_at=now | NOT_FOUND |
| deleteAllData | 刪除 profileInsight、profileNarrative、interviewTurn、interviewSession；重置 consent | NOT_FOUND |

### 2.3 interview.service 真實規則

核心位於 `backend/src/services/interview.service.ts`：

| 方法/分支 | 業務規則 | 錯誤碼 |
|-----------|----------|--------|
| startSession | psych_consent_given 必須為 true | CONSENT_REQUIRED |
| startSession | 每日 substantive session 數達 dailySessionLimit | RATE_LIMIT_EXCEEDED |
| startSession | 每小時 substantive session 數達 startRateLimit | RATE_LIMIT_EXCEEDED |
| respond | session 存在且 user_id 匹配 | NOT_FOUND |
| respond | session.status === IN_PROGRESS | SESSION_COMPLETED |
| respond | turns.length >= maxTurns | MAX_TURNS_REACHED |
| respond | 距 lastTurn.created_at < turnIntervalMs | TURN_TOO_FAST |
| end | IN_PROGRESS → PROCESSING（有足夠 turns）或 COMPLETED | — |
| retry | PROCESSING_FAILED → PROCESSING | — |

### 2.4 psych-profile routes 真實規則

核心位於 `backend/src/routes/psych-profile.routes.ts`：

- `GET /` — authenticate，getProfile
- `GET /feedback` — authenticate + requireConsent，getFeedbackHistory
- `POST /consent` — authenticate，giveConsent
- `DELETE /` — authenticate + requireConsent，deleteAllData

### 3.2 邊界

| 責任點 | 現有測試 | 建議補測 |

| getProfile userId 不存在 | 無 | psych-profile.service |

| 責任點 | 現有測試 | 建議補測 |
|--------|----------|----------|

| session 不屬於 userId | 無 | interview.service NOT_FOUND |
| requireConsent middleware | 無 | psych-profile routes/controller |

### 3.4 狀態轉移

| 責任點 | 現有測試 | 建議補測 |
|--------|----------|----------|
| IN_PROGRESS → PROCESSING（end 後） | 無 | interview.service |

| PROCESSING_FAILED → PROCESSING（retry） | 無 | interview.service |
