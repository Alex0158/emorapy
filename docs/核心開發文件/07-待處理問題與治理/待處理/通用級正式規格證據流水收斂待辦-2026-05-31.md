## 文件偏差

目前偏差主要是「內容層級不穩定」，不一定是代碼錯誤。典型風險：

1. 通用級 PRD / 架構 / 測試基線把單次 artifact 寫成正文後，讀者可能誤認為它是長期完成定義。
2. release / evidence 已更新時，分散在多個正式正文中的舊操作描述更容易漂移。
3. Web / App / Backend 各自的進度細節若直接寫進通用級，會模糊「產品核心規則」與「平台投影進度」的邊界。
4. 長篇命令與 artifact 名稱降低 PRD、SRS、ADR、測試基線作為設計與開發依據的可掃描性。

## 目標狀態

1. `00-跨端產品核心/` 保留跨端產品語義、能力狀態、指標邊界與不得宣稱事項；平台 evidence 細節只保留必要索引。
2. `04-共用機制/` 與 `05-工程架構與共享層/` 保留共用規則、架構裁決、ADR、共享層邊界與觸發條件；遷移批次命令與一次性測試結果移到治理或 evidence。
3. `08-測試規範與驗收/` 保留測試分層、gate 意義、進場條件與不可替代規則；具體 pass artifact 清單改鏈到 `90-證據與盤點/`。
4. 若通用機制在 Web 與 App 有不同實作或不同完成口徑，必須在 `50-跨端Mapping與Parity/` 或 `07-待處理問題與治理/待處理/` 形成明確差異記錄。
