# 報告 05：流程整合測試環境（Integration Flow with Real DB）

## 1. 環境定位

- 目標：驗證「快速體驗主流程」的閉環行為（session -> case -> judgment 等）。
- 入口：
  - `backend/package.json` -> `npm run test:integration:flow`
  - 或 CI job `backend-quick-flow`

## 2. 啟動與 gate 機制

- `quick-experience.flow.test.ts` 以 `RUN_FLOW_TESTS === 'true'` 作為執行開關。
- 未設置時使用 `describe.skip`，避免在未備妥 DB 的環境誤報失敗。

## 3. 關鍵變量

- 必需：
  - `RUN_FLOW_TESTS=true`
  - `TEST_DATABASE_URL` / `DATABASE_URL`（可連線 PostgreSQL）
- 常見：
  - `NODE_ENV=test`
  - `SKIP_DB_INIT=true`（測試流程控制）
  - `JWT_SECRET=test-*`
  - `OPENAI_API_KEY=sk-dev-test-placeholder`
  - `SUPPRESS_LOGS=true`

## 4. 服務依賴（本環境）

- PostgreSQL：必需（CI 以 `postgres:16` service 提供）
- Prisma migration：CI 先執行 `npx prisma migrate deploy`
- OpenAI：在 flow 測試檔內以 `jest.mock('../../src/services/ai.service')` 替代

## 5. 驗證清單

- 先確認 DB 可連線與 migration 完成
- 執行 `npm run test:integration:flow`
- CI 觀察 `Backend Quick Flow Integration` job

## 6. 風險與改進

- 若忘記設 `RUN_FLOW_TESTS=true`，測試會被跳過，容易造成「假綠燈」。
- 此環境覆蓋真實 DB，但 AI 仍為 mock，尚非完整外部依賴整合。
- 建議新增「flow strict」模式：要求顯式輸出 skip reason 並在 CI 以條件阻擋。
