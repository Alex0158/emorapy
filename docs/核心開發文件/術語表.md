# CJ 術語表（核心版）

**文檔版本**：v1.0  
**最後更新**：2026-03-05

---

## 角色與身份

| 術語 | 定義 |
|---|---|
| 訪客（Guest） | 未登入前台用戶，可進快速體驗與部分 chat 入口。 |
| 用戶（User） | 持有 JWT 的前台登入用戶。 |
| 管理員（Admin） | 持有 admin token 的後台運維用戶，走獨立 RBAC。 |
| Session 用戶 | 無 JWT，但持有 `session_id`（`X-Session-Id`）的匿名會話身份。 |

## 路由/頁面術語

| 術語 | 定義 |
|---|---|
| ProtectedRoute | 需登入路由守衛，未登入跳 `/auth/login`。 |
| PublicRoute | 公開認證頁守衛，已登入跳 `/case/list`。 |
| 弱入口 | 可直接 URL 進入但不經強守衛的頁面（如 `/chat/room/:roomId`）。 |
| 候選保留路由 | 主要僅做 redirect 或兼容用途的路由（如 `/auth`）。 |

## 接口治理術語

| 術語 | 定義 |
|---|---|
| 已使用 | 後端接口存在，且前端有調用證據。 |
| 候選廢棄 | 後端接口存在但前端主流程未接線，尚未正式下線。 |
| 已確認廢棄 | 明確不再使用且已完成替代與下線計劃。 |
| 主註冊表 | `全接口清單-主文檔.md`，接口狀態唯一權威來源。 |

## 流程與狀態術語

| 術語 | 定義 |
|---|---|
| Quick Case | 快速體驗建案，通常由 session 驅動。 |
| Collaborative Case | 同設備雙人輪流輸入的協作聽證案件。 |
| Interview Session | 心理訪談會話，具輪次與 pipeline 狀態。 |
| Chat -> Judgment | 聊天室語料轉換為 case + judgment 的鏈路。 |
| 冪等（Idempotent） | 重複請求只產生一次有效結果（例如 chat request-judgment）。 |

## 核心狀態值

| 類別 | 狀態值 | 說明 |
|---|---|---|
| Case | `draft` / `submitted` / `judgment_ready` / `judgment_failed` | 案件生命週期 |
| Interview | `IN_PROGRESS` / `PROCESSING` / `COMPLETED` / `PROCESSING_FAILED` | 訪談主狀態 |
| ChatRoom | `solo_active` / `invite_pending` / `group_active` / `judgment_requested` / `judgment_completed` / `judgment_failed` / `archived` | 聊天房狀態 |
| Pairing | `pending` / `active` / `cancelled` / `temp` | 配對狀態 |

## SSE 事件術語

| 事件 | 來源 | 說明 |
|---|---|---|
| `token` | interview SSE | AI 回覆分詞流 |
| `metadata` | interview SSE | 回合意圖/domain 等元數據 |
| `safety_alert` | interview SSE | 安全風險告警 |
| `complete` | interview SSE | 本次流式回合完成 |
| `message` | chat SSE | 新訊息事件 |
| `room_status` | chat SSE | 房間狀態變更 |
| `invite` | chat SSE | 邀請狀態事件 |
| `ready` / `ping` | chat SSE | 連線可用與保活 |

## 運維術語

| 術語 | 定義 |
|---|---|
| Health 探針 | `/health`、`/health/ready`、`/health/live`。 |
| Metrics 探針 | `/metrics`，Prometheus 格式輸出。 |
| RBAC Permission | admin 權限字串，如 `ops:read`、`config:write`。 |

## 文檔治理術語

| 術語 | 定義 |
|---|---|
| SSOT | Single Source of Truth，本目錄主文檔為一級權威。 |
| 二級映射文檔 | 既有 `docs/` 下舊文檔，保留但從屬於核心文檔。 |
| 追溯閉環 | API -> 功能 -> 頁面 -> 流程 -> 術語可雙向定位。 |
