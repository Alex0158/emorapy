本文件承接正式發布、部署後驗證與回滾口徑，不再以分散的平台指引或根層說明稿作裁決。

## 1. 發布前檢查

必做：

1. 環境變量檢查
2. backend / frontend 構建
3. 單元與必要集成測試
4. 核心文檔守衛（`npm run docs:check`）
5. 關鍵主鏈路 smoke 預演

建議命令：

```bash
./scripts/validate-env.sh
./scripts/ci-local.sh
npm run docs:check
./scripts/test-critical-paths.sh
```

## 2. 建議發布順序

1. 合併到正式分支
2. 打版本 tag
3. 部署前台與管理端
4. 部署後端
5. 執行 smoke 與健康驗證
6. 觀察 logs、alerts、metrics

## 3. 部署後驗證

最低驗證集：

1. `GET /health`
2. `GET /health/ready`
3. `GET /health/live`
4. `POST /api/v1/sessions/quick`

建議腳本：

```bash
BACKEND_URL=https://<backend-domain> FRONTEND_URL=https://<frontend-domain> ./scripts/verify-deployment.sh
```

手動補充抽測：

1. 快速體驗：session -> case -> judgment
2. 訪談：start -> respond / skip -> end -> result
3. 聊天室：建房 -> 發訊息 -> 邀請 -> 接受 -> request-judgment

## 4. 回滾口徑

前台：

1. 回到上一個穩定部署並 promote
2. 或回退到上一個 tag 重新部署

後端：

1. 回退到上一個成功部署
2. 必要時按 commit 重新部署

數據庫：

1. 優先要求 migration 前後兼容
2. 必要時以審慎方式修正 migration state

## 5. 高風險項

本輪若涉及以下能力，發布後必須加驗：

1. 訪談同意與 retry
2. 聊天室 request-judgment 權限與 room 狀態收斂
3. `/metrics` 保護
4. Redis-backed runtime 與 `AI Stream` 恢復
