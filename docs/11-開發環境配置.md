# 開發環境配置（代碼對齊版）

**文檔版本**：v3.3  
**最後更新**：2026-04-04  
**對齊基準**：`backend/package.json`、`frontend/package.json`、`.env.example`、`scripts/*.sh`

---

## 1. 工具鏈

- Node.js：`>=20.19.0`
- npm：建議 10+
- DB：PostgreSQL（開發）
- Redis：本地開發推薦啟用（建議 Docker）

---

## 2. 後端環境

路徑：`backend`

### 2.1 安裝與配置

```bash
cd backend
npm install
cp .env.example .env
npm run dev:redis:up
npm run prisma:generate
npm run prisma:migrate
```

本地後端 `.env` 現已默認接 `REDIS_URL=redis://127.0.0.1:6379`。若不想使用本地 Redis，需自行移除該配置；否則後端會按線上口徑走 Redis-backed runtime。

### 2.2 常用腳本

- `npm run dev`：`tsx watch src/index.ts`
- `npm run build`：`tsc`
- `npm run build:prod`：`tsc --project tsconfig.prod.json`
- `npm start`：啟動 `dist/index.js`
- `npm run dev:redis:up` / `dev:redis:logs` / `dev:redis:down`：管理本地 Redis
- `npm run lint`：Biome
- `npm run test:unit` / `test:integration` / `test:integration:flow`
- `npm run prisma:generate` / `prisma:migrate`

### 2.3 重要注意

- `postinstall` 會觸發 `npm run build`（CI / 新機初始化需注意時間）
- 訪談與聊天室能力依賴 OpenAI key，若需純本地可開 `AI_MOCK=true`
- `AI Stream`、lock、cache、metrics 都會讀 `REDIS_URL`；本地啟 Redis 可更接近 production/staging 行為

## 3. 前端環境

路徑：`frontend`

### 3.1 安裝與配置

```bash
cd frontend
npm install
cp .env.example .env
```

### 3.2 常用腳本

- `npm run dev`：Vite
- `npm run build`：`tsc -b && vite build`
- `npm run lint`：Biome
- `npm run test` / `test:run`：Vitest run
- `npm run test:coverage`
- `npm run test:e2e`：Playwright

### 3.3 重要注意

- 必需 `VITE_API_BASE_URL`
- `/admin/*` 路徑由 `VITE_ADMIN_LOGIN_URL` 轉跳，不是本前台內嵌 admin

---

## 4. 本地啟動順序

1. 後端 Redis 先起：`npm run dev:redis:up`
2. 後端再起：`npm run dev`
3. 前端再起：`npm run dev`
4. 驗證：
   - `GET /health`
   - 後端啟動日誌應出現 `Redis connected for AI Stream runtime`
   - 快速體驗能建 session/case
   - 訪談頁能進入（已 consent）

---

## 5. 一鍵與驗證腳本

- `./scripts/start-dev.sh`：同時拉起前後端
- `./scripts/verify-integration.sh`：基礎連通驗證
- `./scripts/ci-local.sh`：本地 CI 回歸

---

## 6. 常見故障排查

### 6.1 後端啟動失敗

- 缺必填 env：`DATABASE_URL/JWT_SECRET/OPENAI_API_KEY`
- Prisma client 未生成或 migration 未跑

### 6.2 前端 API 404/跨域

- `VITE_API_BASE_URL` 配置錯誤
- 後端 `ALLOWED_ORIGINS` 未包含前端地址（生產環境如 Vercel 須將實際前台網址如 `https://mother-bear-court.vercel.app` 加入，否則 Quick Experience 等會 CORS 403）

### 6.3 訪談/聊天異常

- 檢查 OpenAI 配置與限流
- 檢查後端 log 與前端網路事件流（SSE）
- 若日誌出現 `falling back to in-memory runtime`，代表本地 Redis 未啟動或 `REDIS_URL` 不可用
