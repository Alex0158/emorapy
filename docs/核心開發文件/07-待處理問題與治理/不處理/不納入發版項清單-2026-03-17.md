# 不納入發版項清單（2026-03-17）

**文檔版本**：v1.0  
**最後更新**：2026-03-17  
**定位**：明確標記本輪不納入 `v1.3.1` 發版收口範圍的事項，避免後續與缺陷或阻塞項混淆。

---

## 1. 判定原則

- 不影響本輪核心業務主鏈可用性。
- 不屬於本輪已承諾必須完成的發版證據項。
- 即使不在本輪完成，也不改變「核心功能已收斂」的結論。

---

## 2. 本輪不納入項

| 編號 | 項目 | 說明 | 後續建議 |
|---|---|---|---|
| O-01 | React Native / Expo APP 方案落地 | 目前仍停留在規劃文檔，不屬於本輪 Web 發版範圍 | 待 Web 版本穩定後再拆獨立迭代 |
| O-02 | Flow strict CI / 更強的 skip 阻擋機制 | 現有 flow 測試已可跑真 DB，但更嚴格的 CI gate 不納入本輪 | 後續在 CI 工程治理迭代補齊 |
| O-03 | `.last-run.json` 與 CI 工件的更完整整合 | 本輪已補本地同步腳本，但更完整的 CI/IDE 工件整合仍屬基建優化 | 後續補 CI 工件整合與自動上傳 |
| O-04 | 完整外部 AI provider 真整合 smoke | 本輪 flow 以 AI mock 驗證主鏈，不納入 provider 級外部依賴整合 | 後續如需更高 SLA 再補 |
| O-05 | 更大範圍的視覺/窄視口專項設計回歸 | 本輪已完成主鏈頁交互收口，但不做全站像素級驗收 | 發版後按設計治理專項處理 |

---

## 3. 不可與本清單混淆的事項

以下事項雖尚有後續動作，但仍屬本輪發版必備材料，不應放入「不納入發版項」：

- 發版前手動回歸結果
- 已知風險清單
- 發版前回歸總記錄

---

## 4. 結論

- 本清單中的項目均不應再回灌為「本輪代碼未完成」。
- 後續若有人以這些項目否定本輪收斂結論，應先對照本清單確認是否屬於本輪範圍外事項。
