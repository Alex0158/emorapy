# 用戶端核心流程總覽

本文件承接 `CJ` 面向普通用戶的正式產品主鏈路，覆蓋 Web / Mobile 前台流程，不覆蓋 Admin 治理。

## 1. 本子域回答什麼

本子域只回答以下問題：

1. 快速體驗、正式案件、判決、Repair Journey、心理訪談、聊天室、配對如何分成不同主流程。
2. 這些流程各自由哪些頁面入口、哪些 API 族承接。
3. 各主流程在哪裡交接、何時需要回到認證、共用機制或接口主冊。

本子域不回答：

1. 某個 API 是否已使用、候選廢棄或已確認廢棄。
2. Admin Web、健康檢查、任務治理與後台報表。
3. `packages/*`、workspace、共享 package 的工程落點。

## 2. 主流程分組

`CJ` 的前台主流程不是單一路徑，而是至少六組穩定流程：


| 流程組            | 主功能 / 流程              | 主入口                                                 | 主輸出                                        |
| -------------- | --------------------- | --------------------------------------------------- | ------------------------------------------ |
| 快速體驗           | `F01` / `P01`         | `/quick-experience/create`                          | quick case、結果頁、升格承接                        |
| 協作聽證           | `F02`                 | `/quick-experience/collaborative`                   | collaborative case，判決讀取由 quick result 承接   |
| 正式案件           | `F03` + `F04` + `P02` | `/case/*`、`/judgment/:id`                           | formal case、judgment                       |
| Repair Journey | `F05`                 | `/reconciliation/*`、`/execution/*`                  | plan bundle、repair track、checkin、dashboard |
| 心理訪談與畫像        | `F06` + `P03`         | `/profile/index`、`/interview/*`、`/profile/my-story` | interview session、result、psych profile     |
| 聊天轉判決          | `F07` + `P04`         | `/chat/room*`                                       | chat room、invite、message、judgment handoff  |
| 配對與關係檔案        | `F08`                 | `/profile/pairing`                                  | pairing、relationship profile               |


其中 `F09` 認證與身份承接不在本子域裁決，需回 [../01-認證與會話/00-認證與會話總覽.md](../01-認證與會話/00-認證與會話總覽.md)。

## 3. 頁面入口族群

### 3.1 快速體驗與協作

1. `/quick-experience/create`
2. `/quick-experience/result/:id`
3. `/quick-experience/collaborative`

這一組允許訪客與 `Session` 用戶進入，是最低摩擦入口，但不等於正式案件流程。

### 3.2 正式案件與判決

1. `/case/list`
2. `/case/create`
3. `/case/:id`
4. `/case/:id/review`
5. `/judgment/:id`

這一組屬登入後主流程，且建案前受 pairing gate 影響。

### 3.3 Repair Journey

1. `/reconciliation/:judgmentId`
2. `/reconciliation/:judgmentId/:id`
3. `/execution/dashboard`
4. `/execution/:planId/checkin`
5. `/execution/:planId/replan`

這一組只在已有 judgment 後成立，不應與判決生成混寫成同一頁責任。

### 3.4 心理訪談與個人檔案

1. `/profile/index`
2. `/profile/settings`
3. `/profile/pairing`
4. `/profile/my-story`
5. `/interview/:sessionId`
6. `/interview/:sessionId/result`

這一組涵蓋 consent、訪談、畫像與背景檔案，不等於案件主流程的附錄。

### 3.5 聊天室

1. `/chat/room`
2. `/chat/room/:roomId`

這一組是弱入口；能否發邀請、發起判決、進入正式 judgment，仍取決於 user/session 身份與房間狀態。

## 5. 固定交接規則

以下交接規則屬正式正文，不應在頁面或 dated 報告中各自改寫：

1. quick flow、協作聽證、chat 房間與正式案件不是同一條業務流，只能在明確 handoff 點交接。
2. `/quick-experience/collaborative` 只承接協作建案；判決展示由 `/quick-experience/result/:id` 承接。
3. 正式案件的 `draft -> submitted -> judgment_ready` 與 judgment 消費分開裁決；建案在 `/case/*`，判決消費在 `/judgment/:id`。
4. Repair Journey 只在 judgment 可用後成立；`reconciliation`、`execution`、`replan` 不得回頭改寫案件建立語義。
5. 心理訪談的正式前台承接頁是 `result / my-story`；不得默認宣告任何場景都直接以 psych profile 改寫判決結果。
6. chat judgment ready 只代表可以 handoff 到正式判決消費；若當前仍是訪客，需先登入回跳。
7. pairing 是正式建案前置 gate，不是建案後的補充提示。

## 6. 高風險流程點

以下內容應視為高風險前台鏈路：

1. quick experience 的 session 恢復與 result 回訪。
2. case review 生成 judgment 的高成本 AI 調用。
3. Repair Journey 的 `deferred / paused / replanning / resume`。
4. 訪談 SSE、resume、retry 與 processing failed。
5. chat invite、`request-judgment`、judgment handoff 的競態與冪等。
6. 通知中心的 `read / dismiss / snooze / act`，尤其 active repair journey 的 `snooze` 邊界。

高風險證據入口在 [../90-證據與盤點/README.md](../90-證據與盤點/README.md)，活躍回歸入口在 [../測試/README.md](../測試/README.md)。
