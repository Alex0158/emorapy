## 守衛補強

本輪新增以下真實性守衛（已進入 `npm run docs:check:truth`）：

1. `全接口清單-主文檔.md` 端點行精確匹配（避免 path 子串誤命中）。
2. 端點狀態匯總值與主冊明文統計值一致性校驗（`已使用/候選廢棄/已確認廢棄`）。
3. `全接口清單-主文檔.md` 與 `接口-功能-頁面-Mapping.md` 端點狀態逐條對齊。
4. 主冊與 Mapping 的額外端點行（非後端註冊、非允許的 stream 展開別名）檢測。
5. `GET /api/v1/streams/:scopeType/:scopeId` 的四條展開路徑在主冊與 Mapping 的完整性校驗。
6. 根層 7 份旗艦文檔反引號測試路徑（`*.test.ts`/`*.test.tsx`/`*.e2e.ts`）存在性校驗。
7. 根層 7 份旗艦文檔反引號腳本路徑（`scripts/*.mjs|*.js|*.sh|*.ts`）存在性校驗。
8. `全接口清單-主文檔.md` 的前端證據路徑不得回退到 `frontend-admin/pages/*` 與 `frontend/versionInfo` 舊寫法，且必須包含標準化路徑 token。
