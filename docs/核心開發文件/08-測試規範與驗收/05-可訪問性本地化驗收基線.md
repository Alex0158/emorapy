## 1. 定位

本文定義 Emorapy 可訪問性、本地化與內容設計的驗收口徑。它不替代 [../04-共用機制/07-可訪問性本地化與內容設計治理基線.md](../04-共用機制/07-可訪問性本地化與內容設計治理基線.md)，而是把治理要求落到測試、inspection、手動證據與不得宣稱事項。

當前 Web / Admin 已有 Testing Library role 查詢、focus / keyboard hook 單測、i18n catalog、static a11y contract gate、route/state-level axe smoke 與 manual A11Y evidence JSON contract。App 已有 shared UI accessibility contract gate 與 `TextInput` label / hint static gate。這些只構成本地與自動化基線，仍不等於 WCAG 2.2 AA、正式 screen reader pass、i18n completeness、VoiceOver / TalkBack 或 Dynamic Type 完整驗收。

## 2. 驗收分層

| 驗收層 | 適用範圍 | 最小證據 |
| --- | --- | --- |
| Unit / Component | button、dialog、select、tabs、upload、AI stream、form field | Testing Library `getByRole` / `getByLabelText` / focus tests |
| Route / Flow | quick、formal case、chat、interview、profile、Admin | Playwright 或手動鍵盤流程證據 |
| Automated A11Y | Web / Admin 頁面 | public / authenticated Web route smoke、Admin smoke、static contract gate |
| Manual A11Y | 高風險流程、AI stream、modal、toast、error | keyboard-only、focus order、screen reader smoke、reduced motion 檢查 |
| L10N | zh-TW / en-US catalog、locale switch、API response message、語音朗讀、日期/數字格式 | key completeness、fallback 檢查、backend message locale test、截斷/溢出 inspection |
| App Native | React Native screen、navigation、Deep Link、Push、upload | native accessibility label / role / hint、touch target、screen reader、dynamic type smoke |
| Content Review | 高敏關係文案、錯誤、等待、刪除、AI failure、安全分流 | 術語表對照、非目標對照、風險文案 inspection |
| Design QA | Web / Admin redesign、重要 responsive state | source visual 與 browser screenshot 同 viewport 合併比較；記錄 typography、spacing、token、asset、copy、互動與 console evidence |

## 4. Web / Admin / App 差異

| 平台 | 驗收口徑 |
| --- | --- |
| Web | 以 WCAG 2.2 參考、semantic HTML、ARIA APG、keyboard/focus、route smoke、i18n catalog 為主 |
| Admin Web | 以表格、篩選、配置、token、danger action、audit log、報表的可理解性與鍵盤操作為主 |
| App | 以 React Native accessibility props、native screen reader、touch target、dynamic type、Deep Link / Push landing failure 為主 |

Web 通過不代表 App 通過；Admin 通過不代表普通用戶 Web 通過。App native accessibility 必須由 `mobile/`、App platform adapter、native smoke 或人工證據承接，不得由 Web role tests 外推。

## 5. 不得宣稱

1. 沒有全量 axe/WCAG 掃描與人工審查前，不得宣稱 WCAG 2.2 AA 或任何外部等級；route-level axe smoke 只能證明所列路由無自動化 axe 違規。
2. 沒有 screen reader smoke 前，不得宣稱輔助技術已覆蓋。
3. 沒有對應平台的 i18n key completeness / fallback / hardcoded literal 防回流檢查前，不得宣稱該平台雙語完整。
4. 沒有 native App screen reader / dynamic type / platform smoke 前，不得宣稱 App accessibility 已完成；RNTL accessibility props gate 只能證明 shared UI contract。
5. 內容設計不能只用「已有翻譯」代表安全；高敏關係與 AI 文案需按術語表、PRFAQ、AI 風險、安全分流逐項審查。
6. mock-backed、static 或公開頁面的 design QA 不得外推為 authenticated / destructive / true-service flow 已驗收；報告必須列出未覆蓋 state 與 blocker。

## 6. 維護規則

1. 新增頁面、主流程或高風險 modal 時，至少補一條 role/focus/keyboard 或 inspection 證據。
2. 新增 i18n key 時，zh-TW / en-US 都需同步；若刻意不翻譯，需寫明平台或發布原因。
3. 新增 form field / credential input / file input / select / textarea 時，必須可用 `getByLabelText` 或 `getByRole` 定位；credential input 需檢查 autocomplete。
4. 新增 App screen 時，必須同步 [../20-App端/01-App導航與平台Adapter基線.md](../20-App端/01-App導航與平台Adapter基線.md)、[../50-跨端Mapping與Parity/01-App首輪能力與工程落點Mapping.md](../50-跨端Mapping與Parity/01-App首輪能力與工程落點Mapping.md) 與本文。
5. 新增 backend-owned API response / stream payload message 時，需補 `translateBackendMessage()` 覆蓋與 focused backend i18n / response formatter test；不得要求 Web/Admin/App 以 ad hoc 中文字串 map 修補 backend message。
6. App 新增可見文案時必須通過 `npm --prefix mobile run copy:check`；直接 JSX text、visible prop hardcoded literal 或 visible state template 不得繞過 App i18n catalog。
7. 若後續引入 axe、Playwright accessibility scan、i18n completeness script 或 App native smoke，必須回寫本文、RTM 與文件治理規則。
8. Product Design build 的 QA evidence 必須保存 source visual、implementation screenshot、viewport/state、full/focused comparison、iteration history、互動與 console 結果；只保存分開截圖不能宣稱完成比較。

## 7. Web 十項修復 Gate

Web 十項工程治理已歸檔，子方案與完成證據保留在 `07-待處理問題與治理/已處理/`。本文只保留長期 gate 與不得替代規則；具體修復命令、route pass 數量、artifact 名稱與歷史缺陷細節歸證據或待辦文件。

| 修復項 | 必須證據 |
| --- | --- |
| Admin lint / build | `npm run lint --workspace frontend-admin` 與 `npm run build --workspace frontend-admin` 通過 |
| Admin test entry | `frontend-admin` workspace 級最低 test command；credential-gated E2E 的 env keys、skip 條件與真服務證據路徑文件化 |
| Animation skipped tests | deterministic unit test / browser smoke，或保留明確待辦；不得以總測試 pass 覆蓋 skip |
| icon-only button | `npm run web:a11y:contracts` 靜態 gate 清零；樣本具備 `type="button"` 與 i18n accessible name |
| HTML lang / i18n | locale switch 後 `document.documentElement.lang` 同步；zh-TW / en-US completeness / fallback gate；format helper test |
| form label / autocomplete | `getByLabelText` / `getByRole(..., { name })` 可定位；credential autocomplete 檢查；placeholder 不作唯一 label |
| notification semantic | open / CTA / snooze / dismiss role/name 測試；Enter / Space 行為；action button 不誤觸 open |
| Chat reply preview semantic | reply preview action 使用 native button / link；role/name、keyboard、focus 回歸測試；不得保留未登記 `div role=button` |
| P0 true-service evidence | 真服務或 release-like backend artifact；mock-backed Playwright、credential skip、單純截圖不得宣稱 release 完成 |
| shared API client | shared client contract tests、Web adapter tests、對應 domain service tests、完整 frontend test / build |

若任一修復新增衍生問題，必須先補待辦或在原待辦追加「衍生問題」小節，再完成修復與重跑 gate。
