## 口徑定義

- `GuardType`
  - `ProtectedRoute`：需登入，未登入跳 `/auth/login`；直連保護頁 URL 且未登入時一律跳轉（hydration 逾時亦視為未登入）。
  - `PublicRoute`：登入後不可訪問，會跳 `/case/list`。
  - `Public`：不經路由守衛。
- `InCount`：可到達該頁的上游入口數（按路由跳轉來源去重估算）。
- `OutCount`：從該頁可跳轉的下游頁面數（按路由去重估算）。
- `狀態`
  - `主入口`：主流程高頻路由。
  - `弱入口`：公開或外部導入，易受 URL 直訪影響。
  - `候選保留`：主要只做重定向或兼容用途。
- `功能完成度`
  - `已完成`：頁內可直接觸發核心 API，且有前端證據。
  - `跨功能依賴`：本頁觸發後導頁到他頁完成核心 API。
  - `待驗證`：後端能力存在，但頁面入口或調用證據不足。

## 可達性補充規則

- `ProtectedRoute` 與 `PublicRoute` 均有 hydration 等待；hydration 未完成前不渲染保護內容，逾時（5 秒）一律導向登錄。
- frontend-admin：無 admin token 直連 `/admin/*`（除 `/admin/login`）會重定向至 `/admin/login`。
- Header 導航是主要常駐入口：`/quick-experience/create`、`/case/list`、`/execution/dashboard`、`/chat/room`、`/profile/index`、`/profile/settings`。
- 動態路由主入口由業務操作生成：`/case/:id`、`/judgment/:id`、`/reconciliation/:judgmentId/:id`、`/execution/:planId/checkin`、`/interview/:sessionId`。
- `chat/room*` 為弱入口，但不等於匿名即可完成全部操作；chat owner / participant 權限仍受 user/session 身份與房間角色約束。
