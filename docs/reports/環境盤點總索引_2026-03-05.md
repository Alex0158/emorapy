## 服務使用總覽（跨環境）

- PostgreSQL（Prisma）: 核心必需；unit 可跳過初始化或改 SQLite。
- OpenAI API: 配置層必需；test/CI 多使用 placeholder 或 mock 路徑。
- Redis: 開發/測試可選；生產下分散鎖語義建議視為準必需。
- SMTP: 可選；未配置時郵件流程會 graceful skip。
- Prometheus `/metrics`: 可開關；生產要求 token 或 IP allowlist 保護。
- Slack Webhook（ops alerts）: 可選；配合排程告警與去重。
- Railway/OpenAI Billing API（成本報表）: 可選；不可用時回傳 partial，不阻斷主流程。
- Playwright（E2E）: 僅 CI admin-e2e / e2e 環境使用。

## 核心結論

- 目前「可直接執行」的主要環境鏈路完整覆蓋：本地開發、unit、integration、flow、CI admin-e2e、生產。
- 目前最大缺口是 **Staging/Preview 未制度化**（無獨立 pipeline、無固定環境契約檔、無獨立 secrets/驗證門檻）。
- 文件存在局部漂移（例如 `PORT` fallback 口徑、Docker 映射示例與 `EXPOSE` 不一致），已在各報告的風險段落逐一標註。
