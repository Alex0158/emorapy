## 🧪 測試概述

### 測試目標

1. **功能正確性**：確保所有功能按預期工作
2. **穩定性**：確保系統穩定運行
3. **安全性**：確保安全機制有效
4. **性能**：確保響應時間和吞吐量達標

### 測試覆蓋率目標

- **單元測試**：80%+
- **集成測試**：60%+
- **關鍵路徑**：100%

**實現對齊**：單元測試位於 `backend/tests/unit/`，覆蓋 config、controllers、middleware、routes、services、utils、jobs；集成測試位於 `backend/tests/integration/`（含 smoke、quick-experience 流程）。無獨立 repository 層，service 測試通過 mock Prisma 或依賴注入完成。

## 📋 測試類型

### 1. 單元測試

**測試對象**：Service層、工具函數、業務邏輯

**測試框架**：Jest

**示例**：

```typescript
// tests/unit/services/auth.service.test.ts
import { AuthService } from '../../../src/services/auth.service';
import { UserRepository } from '../../../src/repositories/user.repository';
import bcrypt from 'bcrypt';

describe('AuthService', () => {
  let authService: AuthService;
  let userRepository: jest.Mocked<UserRepository>;
  
  beforeEach(() => {
    userRepository = {
      findByEmail: jest.fn(),
      create: jest.fn()
    } as any;
    
    authService = new AuthService(userRepository, emailService);
  });
  
  describe('register', () => {
    it('應該成功註冊新用戶', async () => {
      // Arrange
      const registerData = {
        email: 'test@example.com',
        password: 'password123',
        nickname: 'Test User'
      };
      
      userRepository.findByEmail.mockResolvedValue(null);
      userRepository.create.mockResolvedValue({
        id: 'uuid',
        email: registerData.email,
        nickname: registerData.nickname
      } as User);
      
      // Act
      const result = await authService.register(registerData);
      
      // Assert
      expect(result.user.email).toBe(registerData.email);
      expect(result.token).toBeDefined();
      expect(userRepository.create).toHaveBeenCalled();
    });
    
    it('應該拒絕已存在的郵箱', async () => {
      // Arrange
      userRepository.findByEmail.mockResolvedValue({
        id: 'uuid',
        email: 'test@example.com'
      } as User);
      
      // Act & Assert
      await expect(
        authService.register({
          email: 'test@example.com',
          password: 'password123'
        })
      ).rejects.toThrow('EMAIL_EXISTS');
    });
  });
});
```

### 2. 集成測試

**測試對象**：API端點、數據庫操作、外部服務集成

**測試框架**：Jest + Supertest

**示例**：

```typescript
// tests/integration/auth.routes.test.ts
import request from 'supertest';
import app from '../../src/app';
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

describe('POST /api/v1/auth/register', () => {
  beforeEach(async () => {
    // 清理測試數據
    await prisma.user.deleteMany();
  });
  
  it('應該成功註冊用戶', async () => {
    const response = await request(app)
      .post('/api/v1/auth/register')
      .send({
        email: 'test@example.com',
        password: 'password123',
        nickname: 'Test User'
      })
      .expect(200);
    
    expect(response.body.success).toBe(true);
    expect(response.body.data.user.email).toBe('test@example.com');
    expect(response.body.data.token).toBeDefined();
  });
  
  it('應該拒絕無效的郵箱格式', async () => {
    const response = await request(app)
      .post('/api/v1/auth/register')
      .send({
        email: 'invalid-email',
        password: 'password123'
      })
      .expect(400);
    
    expect(response.body.success).toBe(false);
    expect(response.body.error.code).toBe('VALIDATION_ERROR');
  });
});
```

### 3. E2E測試

**測試對象**：完整用戶流程

**示例流程**：

```typescript
describe('完整案件流程', () => {
  it('應該完成從註冊到判決的完整流程', async () => {
    // 1. 註冊用戶
    const registerResponse = await request(app)
      .post('/api/v1/auth/register')
      .send({ email: 'user1@test.com', password: 'pass123' });
    
    const token = registerResponse.body.data.token;
    
    // 2. 創建配對
    const pairingResponse = await request(app)
      .post('/api/v1/pairing/create')
      .set('Authorization', `Bearer ${token}`);
    
    // 3. 創建案件
    const caseResponse = await request(app)
      .post('/api/v1/cases')
      .set('Authorization', `Bearer ${token}`)
      .send({
        pairing_id: pairingResponse.body.data.pairing.id,
        title: '測試案件',
        plaintiff_statement: '這是原告的陳述...',
        defendant_statement: '這是被告的陳述...'
      });
    
    // 4. 生成判決
    const judgmentResponse = await request(app)
      .post(`/api/v1/cases/${caseResponse.body.data.case.id}/judgment`)
      .set('Authorization', `Bearer ${token}`);
    
    expect(judgmentResponse.body.success).toBe(true);
    expect(judgmentResponse.body.data.judgment).toBeDefined();
  });
});
```

## 🛠️ 測試工具配置

### Jest配置

**文件**：`jest.config.js`

```javascript
module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'node',
  roots: ['<rootDir>/src', '<rootDir>/tests'],
  testMatch: ['**/__tests__/**/*.ts', '**/?(*.)+(spec|test).ts'],
  collectCoverageFrom: [
    'src/**/*.ts',
    '!src/**/*.d.ts',
    '!src/**/*.test.ts'
  ],
  coverageThreshold: {
    global: {
      branches: 80,
      functions: 80,
      lines: 80,
      statements: 80
    }
  },
  setupFilesAfterEnv: ['<rootDir>/tests/setup.ts']
};
```

```typescript
// ✅ 好的命名
describe('AuthService', () => {

// ❌ 不好的命名
describe('AuthService', () => {

});
```

### 2. 測試結構

```typescript
it('應該成功註冊用戶', async () => {
  // Arrange: 準備測試數據
  const data = { email: 'test@example.com', password: 'pass123' };
  
  // Act: 執行被測試的方法
  const result = await authService.register(data);
  
  // Assert: 驗證結果
  expect(result.user.email).toBe(data.email);
});
```
