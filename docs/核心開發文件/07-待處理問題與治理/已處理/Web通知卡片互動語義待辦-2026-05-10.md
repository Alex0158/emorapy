## 1. 問題

通知中心把卡片正文做成 `div role="button" tabIndex={0}`，再在同一張卡片下方放多個真 `<Button>` 操作。這種模式對 screen reader、鍵盤順序、事件邊界與語義理解都不穩定；使用者可能無法清楚區分「打開通知」與「執行 CTA / 稍後提醒 / 封存」。

## 2. 證據

`frontend/src/pages/Notifications/index.tsx`：

```tsx
<div
  role="button"
  tabIndex={0}
  className={cn('w-full text-left', ...)}
  onClick={() => void handleOpen(notification)}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      void handleOpen(notification);
    }
  }}
>
  ...
</div>

<div className="mt-3 flex flex-wrap gap-2">
  <Button size="sm" onClick={() => void handleAct(notification)}>...</Button>
  <Button variant="outline" size="sm" onClick={() => void handleSnooze(notification.id)}>...</Button>
  <Button variant="ghost" size="sm" onClick={() => void handleDismiss(notification.id)}>...</Button>
</div>
```

同輪主站單測通過：

```bash
npm run test:run --workspace frontend
```

結果：182 個測試檔通過，2105 passed，6 skipped。但現有通知頁測試只有 3 條，沒有證明 semantic HTML / keyboard-only flow / focus order 已覆蓋。

## 3. 核心文件依據

1. `08-測試規範與驗收/05-可訪問性本地化驗收基線.md` 要求 Web 以 semantic HTML、ARIA APG、keyboard/focus 為驗收口徑。
2. `04-共用機制/07-可訪問性本地化與內容設計治理基線.md` 要求新增 icon-only button、toast、status、upload 等互動控件時必須有 accessible name、keyboard/focus 口徑與測試或 inspection。
3. `頁面清單.md` 把 `/notifications` 定為 ProtectedRoute 主入口，功能完成度為已完成；因此互動語義不能只靠滑鼠可點擊。

## 4. 風險

1. `div role=button` 需要手動補齊所有 native button 行為，容易漏掉 disabled、focus、name、Space/Enter 行為細節。
2. 卡片正文與卡片操作按鈕的語義分離不足，screen reader 可能讀不出清楚 action model。
3. `new Date(notification.created_at).toLocaleString()` 未接 `getLocale()`，日期格式也缺少 locale policy 驗收，與 L10N-T-002 待建立狀態相關。

## 5. 目標狀態

1. 將可打開通知的區域改為語義化 `<button type="button">` 或清楚拆分為標題 link / explicit open button。
2. CTA、snooze、dismiss 與 open 行為在 DOM 和 keyboard order 上互不混淆。
3. 補充 Testing Library role/name 測試，至少覆蓋 Enter / Space 打開通知與 action button 不誤觸 open。
4. 日期格式使用 `getLocale()` 或共用 format helper，並納入 L10N gate。

## 5.1 修復裁決

本問題納入 [Web五項修復主控方案-2026-05-10.md](./Web五項修復主控方案-2026-05-10.md) 的 P1-A 階段。通知 open 行為必須從 `div role=button` 改為 native semantic control；CTA、snooze、dismiss 不得被 open control 包住。日期格式化需改用 locale-aware helper，並補 role/name、keyboard 與 action 不誤觸 open 的測試。
