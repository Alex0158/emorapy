本目錄承接真 DB、staging、服務預檢、運行時收口與高風險抽檢的核驗材料。

這些文件只代表當次核驗結果，不直接外推為正式規格裁決。

Web P0 true-service / release-like smoke 由根層 `npm run web:p0:true-service:smoke` 產生，對應 runner 為 `backend/scripts/web-p0-true-service-smoke.ts`。正式可引用 artifact 應命名為 `Web-P0-True-Service-*.json` 並落在本目錄；該 runner 會建立測試使用者、case、evidence、chat room 與 judgment，必須顯式設定 `RUN_WEB_P0_TRUE_SERVICE_SMOKE=true` 並提供 `WEB_P0_SMOKE_REPORT_PATH`，若 `DATABASE_URL` 非本機 host 還必須顯式設定 `WEB_P0_SMOKE_ALLOW_REMOTE_DB=true`，並記錄 target API、target DB presence、`AI_MOCK` 狀態與逐步結果。2026-05-12 本機 local backend + local Postgres + local Redis + `AI_MOCK=true` pass artifact 為 [Web-P0-True-Service-Local-2026-05-12T19-08-00+08-00.json](./Web-P0-True-Service-Local-2026-05-12T19-08-00+08-00.json)；它可證明本機 no-cost true-service 閉環，但不替代 release / production target artifact 或 Admin credential-backed E2E。mock-backed Playwright、credential-gated skip 或沒有 artifact 的 console output 不得替代此證據。

Web Admin credential-backed E2E 證據由 `npm --prefix e2e run test -- --reporter=json` 產生，必須提供 `E2E_BASE_URL`、`E2E_ADMIN_EMAIL`、`E2E_ADMIN_PASSWORD`，permission denied flow 還必須提供 `E2E_LIMITED_ADMIN_EMAIL`、`E2E_LIMITED_ADMIN_PASSWORD`，並建議加 `E2E_STRICT=true` 防止 credential 缺失時 silent skip。正式可引用 artifact 應命名為 `Web-Admin-Credential-E2E-*.json` 並落在本目錄；2026-05-12 本機 Admin Web + local backend + local Postgres pass artifact 為 [Web-Admin-Credential-E2E-Local-2026-05-12T19-39-30+08-00.json](./Web-Admin-Credential-E2E-Local-2026-05-12T19-39-30+08-00.json)，其 `stats.expected=10`、`stats.skipped=0`、`stats.unexpected=0`。本機 artifact 不替代 release / production target credential evidence。

如果尚未確認 target DB/API 可安全承接 mutating smoke，允許先產出 `Web-P0-True-Service-Preflight-*.json` 作為 blocker 證據。Preflight 只能記錄 guard failure、backend health、redacted env、remote DB guard 與下一步 prerequisite，不得被當作 pass artifact。

Web / Admin A11Y automated baseline 證據由 `npm run web:a11y:contracts`、`npm run test:a11y --workspace frontend` 與 `npm run test:a11y --workspace frontend-admin` 組成，正式 artifact 命名為 `Web-A11Y-Automated-Baseline-*.json`。此 artifact 只證明 static contract 與選定 route/state-level axe smoke 通過，不得替代全量 WCAG 2.2 AA、screen reader、manual keyboard-only、全狀態矩陣或 Web P0 true-service release 證據。

Web / Admin manual A11Y evidence 由根層 `npm run web:a11y:manual-evidence:check` 驗證，對應 schema runner 為 `scripts/check-web-a11y-manual-evidence.mjs`。正式可引用 artifact 應命名為 `Web-A11Y-Manual-*.json` 並落在本目錄；嚴格模式要求 keyboard-only 覆蓋 quick experience、case list/detail、notifications、chat room、auth、Admin login 與 Admin ops jobs，至少一組 screen reader run 通過，且 `interactive_surfaces` 覆蓋 modal、dropdown、toast、upload、form validation、async loading、error recovery 與 remaining route-state matrix，並附可解析 artifact path。模板見 [Web-A11Y-Manual-Evidence-Template.json](./Web-A11Y-Manual-Evidence-Template.json)，人工產證流程見 [Web-A11Y-Manual-Evidence-Runbook-2026-05-12.md](./Web-A11Y-Manual-Evidence-Runbook-2026-05-12.md)，可用 `npm run web:a11y:manual-evidence:template:check` 驗證欄位形狀；模板、runbook 或 blocked artifact 不得替代正式 pass evidence，也不得用來宣稱 WCAG 2.2 AA、完整 screen reader 或全狀態矩陣完成。

若尚未取得人工 keyboard-only / screen reader pass evidence，允許產出 `Web-A11Y-Manual-Preflight-Blocked-*.json` 作為 blocker 證據。2026-05-12 blocker artifact 為 [Web-A11Y-Manual-Preflight-Blocked-2026-05-12T19-45-00+08-00.json](./Web-A11Y-Manual-Preflight-Blocked-2026-05-12T19-45-00+08-00.json)；它只記錄 strict gate 缺正式 `Web-A11Y-Manual-*.json` pass artifact，不得替代正式 evidence。

AI request ledger / Admin costs pricing release preflight 由 `npm --prefix backend run ops:ai-pricing:check` 與 `npm --prefix backend run ops:release-db:dry-run` 驗證。若 release backend 尚未提供 `AI_COST_PRICING_JSON` 或尚未取得 release / production non-local DB parity evidence，允許產出 `AI-Pricing-Release-Env-Blocked-*.json` 作為 blocker 證據。2026-05-12 blocker artifact 為 [AI-Pricing-Release-Env-Blocked-2026-05-12T14-38-24Z.json](./AI-Pricing-Release-Env-Blocked-2026-05-12T14-38-24Z.json)；它只記錄 pricing env / release DB evidence 缺口，不記錄價格數值，不代表價格已審批，也不得替代 `AI_COST_PRICING_JSON` release env、Admin costs precision claim 或 release DB pass artifact。對應的正式產證流程見 [AI-Pricing-Release-Env-Runbook-2026-05-12.md](./AI-Pricing-Release-Env-Runbook-2026-05-12.md)，正式 pass artifact 可從 [AI-Pricing-Release-Env-Pass-Template.json](./AI-Pricing-Release-Env-Pass-Template.json) 複製，但 Template 本身不得作 pass evidence。

Web / Admin 相關待處理任務的 prompt-to-artifact 完成審計見 [Web-Pending-Tasks-Completion-Audit-2026-05-14.md](./Web-Pending-Tasks-Completion-Audit-2026-05-14.md)，外部 owner 交接 JSON 見 [Web-Pending-Tasks-Handoff-2026-05-14.json](./Web-Pending-Tasks-Handoff-2026-05-14.json)。這些文件只作 completion audit 與 blocker 交接，不是 pass artifact；根層 `npm run web:pending:completion:audit` 可輸出 non-strict 盤點並驗證 handoff JSON 的 blocker id / owner action 結構，`npm run web:pending:completion:audit:contract` 固定 JSON / blocker / strict exit-code contract，`npm run web:pending:completion:audit:strict` 在正式 `Web-A11Y-Manual-*.json`、release pricing env、價格審批與 non-local DB parity evidence 未齊前必須失敗，審計結論必須保持 `未完成`。
