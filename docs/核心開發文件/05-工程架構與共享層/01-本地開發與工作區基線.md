本文件承接 repo 根 package.json、各端 package.json/tsconfig 的真實啟動方式、工作區分工、本地聯調順序與共享 package 的正式工程口徑。

## 1. 工作區角色

| 目錄 | 角色 | 物理現狀 |
| --- | --- | --- |
| `frontend/` | 主 Web 端 | root workspace |
| `frontend-admin/` | 管理端 Web | root workspace |
| `mobile/` | Expo App | repo-local app；已從 Expo template tabs 轉為 Emorapy route group shell，`mobile/src/platform` 已有 API / SecureStore / SSE / upload / notifications / linking / lifecycle / telemetry runtime adapter 與 M0-M5 baseline 測試入口 |
| `backend/` | API、jobs、health、metrics | repo-local service；獨立 package 與 tsconfig |
| `packages/contracts/` | 共享契約 | root workspace shared package；已被 `frontend/`、`frontend-admin/` 與 `backend/` 局部消費 |
| `packages/api-client/` | 共享 transport 與客戶端基線 | root workspace shared package；`frontend/` 已消費 M1-M5 domain client 與 AI stream pure helper；`frontend-admin/` 已用於 request transport baseline，但尚未下沉 domain API client |

共享 package 的 app 消費必須同時滿足 import、tsconfig/Vite/Metro alias、`package.json` dependency 與 lockfile link 一致；目前 `frontend/` 聲明 `@cj/contracts` 與 `@cj/api-client`，`frontend-admin/` 聲明 `@cj/contracts` 與 `@cj/api-client`，`mobile/` 以 repo-local `file:../packages/*` dependency 聲明 `@cj/contracts` 與 `@cj/api-client` 並由 Metro aliases 指向本 repo packages。

`frontend/` 與 `frontend-admin/` 不再保留 app-local package-lock；乾淨安裝、CI 與 Vercel 都必須從 repo root 使用 root `package-lock.json`，再用 `npm run ... --workspace <app>` 執行 app script。

root-owned script 若以 bare import 使用工具，必須由 root manifest 聲明依賴；例如靜態頁匯出腳本的 Playwright runtime 由 root `devDependencies.playwright` 負責，而不是依賴 app-local `node_modules` 的物理位置。

`mobile/src/platform` 已具備可運行 adapter 與測試入口，但它仍不等於 SecureStore / Push / upload / Deep Link / SSE / telemetry 已完成 native 驗收；啟 App 版服務或開發 App feature 時，必須另看 [../20-App端/01-App導航與平台Adapter基線.md](../20-App端/01-App導航與平台Adapter基線.md) 與 [../50-跨端Mapping與Parity/01-App首輪能力與工程落點Mapping.md](../50-跨端Mapping與Parity/01-App首輪能力與工程落點Mapping.md)。

## 2. 本地工具鏈

最低要求：

1. Node.js 24.x（以各 package `engines` 為準）
2. PostgreSQL
3. Redis，本機 `redis-server`
4. npm

本機開發版固定以 Localhost 運行，日常不使用 Docker 作必要前提。Redis 固定優先使用本機 `redis-server`，當前標準為 `redis://127.0.0.1:6379` 與 `ALLOW_SIMPLE_LOCK=false`；Docker Redis 只可作輔助替代，不作應用本身的正式本機入口。

本地若不啟 Redis，可以開發，但不等價於 production。

App true-service smoke 為避免污染遠端或 production DB，推薦使用本機隔離 Postgres。已提供 `backend/docker-compose.postgres.yml` 與 `backend` scripts `dev:postgres:up` / `dev:postgres:down` / `dev:postgres:logs`，標準連線為 `postgresql://cj:cj_dev_pass@127.0.0.1:55432/cj_dev`。若 Docker daemon 不可用，可使用同等本機 Postgres 16 cluster；執行 `npm --prefix mobile run smoke:true-service -- --run ...` 時必須以 `APP_SMOKE_BACKEND_DATABASE_URL` 明確指向該本機 DB，否則腳本會讀取 `backend/.env` 並阻擋 non-local DB。

## 3. 建議啟動順序

首選一鍵啟動：

```bash
./scripts/start-dev.sh
```

這會確認/啟動本機 Redis，並以當前 `git rev-parse HEAD` 注入 backend `EMORAPY_COMMIT_SHA` 與 legacy `CJ_COMMIT_SHA`，再啟動 backend `3001`、主站 `5173` 與 Admin `5175`。

手動後端：

```bash
cd backend
npm install
cp .env.example .env
redis-server --port 6379 --dir /tmp/cj-redis-dev --appendonly yes
npm run prisma:generate
EMORAPY_COMMIT_SHA=$(git -C .. rev-parse HEAD) npm run dev
```

前端：

```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```

管理端：

```bash
cd frontend-admin
npm install
cp .env.example .env
npm run dev
```

## 4. 本地聯調固定檢查

1. 後端 log 應能確認 Redis runtime 是否接通
2. `GET /health` 可用；development 可因 scheduled jobs disabled 顯示 `degraded`，但 database 與 lock checks 必須 healthy
3. backend `/version`、主站 `/version.json`、Admin `/version.json` 都應回報當前 HEAD
4. `VITE_API_BASE_URL` 指向正確 API base
5. `frontend-admin` 能登入並成功讀取 `GET /admin/me`
6. 快速體驗能完成建 session / case / judgment
7. 訪談與聊天室能走通 SSE / stream 主鏈路

## 5. 共享 package 使用規則

`packages/contracts` 與 `packages/api-client` 是正式共享層，但不應因此假設所有類型與邏輯都已完全下沉，也不應假設所有 app 都已接入。

主站 Web 的 `frontend/src/services/api/{auth,session,case,chat,interview,psychProfile,pairing,judgment,reconciliation,execution,notifications}.ts` 已按 adapter / strangler pattern 消費 `@cj/api-client` 的 M1-M5 domain client；`frontend/src/utils/aiStreamState.ts` 已消費 `@cj/api-client` 的 AI stream pure helper。`frontend/package.json` 與 root `package-lock.json` 必須保留 `@cj/api-client` workspace dependency / lock entry；不得依賴偶然 hoist。

共享層最小責任：

1. 契約、enum、response shape
2. 純 transport 與共用客戶端行為
3. 純邏輯與可跨端重用的工具

不應下沉：

1. Web / RN UI
2. router / navigation 副作用
3. 平台 storage 實作
4. 原生能力與瀏覽器專屬能力

## 6. 本地與遠端環境

本地開發的日常入口只承認 Localhost。遠端 production 只作正式發布、發布後驗證與明確認知風險的排查；不再保留常設 staging 作日常推薦入口。若需要一次性 sandbox / preview 聯調，必須在任務上下文中明確說明用途、資料邊界與不得替代 production release gate 的限制。
