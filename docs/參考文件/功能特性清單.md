# JobPortal 功能特性清單

本文件從產品能力角度定義系統功能，不承載接口細節。以現有程式碼實作為唯一事實來源，聚焦可操作業務功能。

**關聯文檔**：

- 接口與模組：`全接口清單-主文檔.md`、`接口描述/`
- 頁面與路由：`頁面清單.md`
- 追溯矩陣：`接口-功能-頁面-Mapping.md`
- 業務流程：`業務流程整合.md`

---

## 0. 文檔說明

### 0.1 範圍與邊界

- **納入**：有頁面或明確業務流程可觸達的可操作功能（含管理端）。
- **不納入**：純技術基礎設施、E2E 測試專用接口、後端主表直接 CRUD 且無前端常規入口的能力。詳見第 7 章。

### 0.2 功能項標準欄位

| 欄位          | 說明                                      |
| ------------- | ----------------------------------------- |
| 功能名稱      | 標準命名                                  |
| 功能目的      | 解決什麼問題                              |
| 角色          | 學生端 / 僱主端 / 管理端 / 公共（可多選） |
| 入口頁面      | 主路由或子分頁                            |
| 關聯頁面      | 相關路由                                  |
| 關聯 API 群組 | 對應 `全接口清單-主文檔.md` 模組        |
| 前置條件      | 使用前須滿足的條件                        |
| 核心操作      | 用戶可執行的主要操作                      |
| 主要業務規則  | 關鍵邏輯與限制                            |
| 例外流程      | 異常情況處理                              |

### 0.3 深化拆分欄位（新增）

在保留既有「功能級」表格的前提下，新增「頁面 × API 小功能點」明細表，並按章節內聯維護（不再集中獨立維護），欄位如下：

| 欄位          | 說明                                                                                     |
| ------------- | ---------------------------------------------------------------------------------------- |
| 頁面名字      | 使用 `頁面清單.md` 的中文頁面語義名（如：登入（Login）、管理儀表板（AdminDashboard）） |
| 功能點/特性點 | 在該頁面下可獨立驗收的最小能力點                                                         |
| 描述          | 融合「功能目的 + 核心操作 + 前置條件 + 主要業務規則」後的精煉描述                        |
| API           | `method + path`（一列僅對應一個 API；無 API 的前端能力可標記 `N/A`）                 |
| API名字       | 以前端實作命名為優先（如 `statsApi.getJobPostingTrends`）；若為直呼叫則用語義名        |
| 是否深入完成  | `已完成 / 部分完成 / 未完成`                                                           |
| 來源依據      | `主文檔 = API欄`、`Mapping = 頁面名字 + 功能點/特性點`、`代碼 = API名字欄`         |

`是否深入完成` 判定口徑：

- `已完成`：主流程、前置條件、規則、例外在現有代碼與文檔可追溯且一致。
- `部分完成`：已有主流程，但存在弱入口、Mock、或規則/例外描述仍待補全。
- `未完成`：缺乏可追溯代碼證據或僅有概念描述。

## 1. 認證與帳號

### 1.1 登入

| 欄位          | 說明                                                 |
| ------------- | ---------------------------------------------------- |
| 功能名稱      | 登入                                                 |
| 功能目的      | 讓用戶以帳密或 SSO 建立會話並取得角色與權限          |
| 角色          | 公共                                                 |
| 入口頁面      | `/login`                                           |
| 關聯頁面      | `/sso-callback`、`/auth/azure/callback`          |
| 關聯 API 群組 | 認證與註冊                                           |
| 前置條件      | 已有帳號                                             |
| 核心操作      | 帳密登入、Azure SSO 登入、忘記密碼、取得當前用戶     |
| 主要業務規則  | 登入成功後依角色導向預設首頁；SSO 回調建立會話後分流 |
| 例外流程      | 驗證失敗提示錯誤；SSO 回調失敗導回登入頁             |

### 1.2 註冊

| 欄位          | 說明                                                              |
| ------------- | ----------------------------------------------------------------- |
| 功能名稱      | 註冊                                                              |
| 功能目的      | 讓學生或僱主建立帳號並完成基本資料填寫                            |
| 角色          | 公共                                                              |
| 入口頁面      | `/register`、`/register/:role`                                |

| 關聯 API 群組 | 認證與註冊                                                        |
| 前置條件      | 未登入                                                            |

| 例外流程      | 驗證碼過期提示重發；AIS 同步失敗不阻擋註冊                        |

## 2. 學生端核心功能

### 2.1 首頁與職位探索

| 欄位          | 說明                                                                             |
| ------------- | -------------------------------------------------------------------------------- |
| 功能名稱      | 首頁與職位探索                                                                   |
| 功能目的      | 讓學生瀏覽熱門職位、熱門企業與平台統計，並進入職位搜索與詳情                     |
| 角色          | 學生端                                                                           |
| 入口頁面      | `/`、`/job-search`                                                           |
| 關聯頁面      | `/job-detail/:id`                                                              |
| 關聯 API 群組 | 職位管理、統計與分析                                                             |
| 前置條件      | 可未登入訪問首頁；已登入學生可使用完整功能（搜索、收藏、投遞）                   |
| 核心操作      | 首頁輪播/熱門職位/熱門企業、職位搜索（多關鍵字、篩選、排序、分頁）、職位詳情查看 |
| 主要業務規則  | 職位搜索支援多條件組合；職位詳情可觸發申請與收藏（須審核通過）                   |
| 例外流程      | 無符合條件職位時顯示空狀態；職位已下架時提示                                     |

### 2.2 職位申請

| 欄位          | 說明                                                                                             |
| ------------- | ------------------------------------------------------------------------------------------------ |
| 功能名稱      | 職位申請                                                                                         |
| 功能目的      | 讓學生向僱主投遞職位申請，並追蹤申請狀態                                                         |
| 角色          | 學生端                                                                                           |
| 入口頁面      | `/job-detail/:id`                                                                              |
| 關聯頁面      | `StudentProfile.applications`、`/student-applications`                                       |
| 關聯 API 群組 | 申請管理、審核管理                                                                               |
| 前置條件      | 已登入學生、至少一份簡歷、職位狀態為招聘中、學生帳號審核通過                                     |
| 核心操作      | 選擇簡歷與求職信、提交申請、查看申請列表與詳情、查看審核進度、撤回申請、下載簡歷附件（申請關聯） |
| 主要業務規則  | 同一職位僅能申請一次；申請後可撤回（依業務時限）；外部職位僅記錄投遞                             |
| 例外流程      | 簡歷未完成提示補全；已申請過提示；職位不可用時禁用按鈕                                           |

以下能力存在於程式碼或後端，但**不納入**本功能特性清單主體，原因如下。

### 7.1 純技術/內部

- **E2E 測試專用接口**：僅供自動化測試使用，無業務用戶觸達。
- **後端主表直接 CRUD**：管理員直接操作職位/僱主主表且無前端常規入口，屬內部管理工具。
- **僱主信息記錄內部接口**：審核流程內部記錄，前端通常不直接操作。
- **未掛路由頁面元件**：如 `RegisterStudentPreview`、`Profile`、`Admin`、`AuditManagement`、`DataAnalysisCopy` 等存在但未掛載常規路由，暫不納入主功能清單。
- **定時任務（Quartz）**：`jobportal.account-expiry`、`jobportal.account-disable`、`jobportal.job-deadline` 屬後台調度能力，無前端可操作入口，於 `業務流程整合.md` 維護流程描述。

### 7.2 低優先/待確認

- **AI 聊天系統**：`FloatingAIChat` 等組件存在，但若未正式啟用則不視為主功能。
- **數據備份**：系統設置中有入口，但若為 Mock 或未完全打通，可標記為待確認。
- **工作流引擎 / 多級審核**：若為 Mock 或 UI 階段，不展開為獨立功能項。

### 7.3 候選廢棄接口對應功能

- 主接口清單中標記為「候選廢棄（前端未用）」的 API，其對應能力暫不列入本清單；若後續業務確認啟用，再補充。

### 7.4 候選廢棄路由（保留路由）

- `/register-success`
- `/company`
- `/analytics-realtime`
- `/analytics-pages`
- `/analytics-behavior`
- `/analytics-system`
- `/system-settings`

## 8. 維護規範

1. 新增可操作功能時，須同步更新本清單、`頁面清單.md`、`全接口清單-主文檔.md`、`接口-功能-頁面-Mapping.md`。
2. 功能項須至少對應一個入口頁面與一個 API 群組。
3. 角色命名統一為：學生端、僱主端、管理端、公共。
4. 與頁面清單、主接口清單若有差異，須在本文件或關聯文檔中明示邊界與原因。
