## 2.1 五輪方案分析與裁決（2026-05-12）

1. **產品定位輪**：核心文件已定 CJ 是 AI 關係梳理助手，不是心理治療、法律裁決或危機救援。修復必須優先改「AI 自稱 / 被稱作什麼」，而不是刪除所有心理學框架。
2. **合規與信任輪**：`AI 心理師 / AI Therapist / 臨床心理師` 會讓用戶誤解為持牌服務；`不是持牌治療師` 與「必要時尋求持牌專業支持」反而是必要邊界，需允許但保持 referral 語義。
3. **Prompt 行為輪**：後端 prompt 可引用 NVC、EFT、Gottman、ACT 等理念作分析框架，但 system prompt 不得把模型設為治療師或臨床心理師；輸出目標改為整理觀點、情緒脈絡、支持方式與情境判斷。
4. **Web 文案輪**：首頁流程、quick result、collaborative submit 等用戶可見狀態應使用「AI 梳理 / AI 正在整理關係脈絡 / AI reflection」，降低權威與診斷暗示，仍保留「分析結果」作一般能力描述。
5. **治理守衛輪**：只靠一次 `rg` 不足以防回流；新增 `ai:positioning:check` 作 repo contract，並把它寫回 AI 風險基線和本待辦驗收命令。

## 3. 目標口徑

1. 對外角色：AI 關係梳理助手 / AI relationship guide / safe listening space。
2. 對外結果：梳理結果 / 分析結果 / 關係整理，不用「判決」作主文案。
3. 對外安全聲明：AI 僅供自我探索與溝通整理，不構成心理診斷、法律意見或專業治療。
4. Prompt 可引用 NVC、EFT、Gottman、ACT 等理念，但不得自稱治療師、臨床心理師或專業治療提供者。
5. 風險/安全場景仍應建議尋求持牌專業支持，但語義是 referral，不是 AI 自己扮演該角色。
