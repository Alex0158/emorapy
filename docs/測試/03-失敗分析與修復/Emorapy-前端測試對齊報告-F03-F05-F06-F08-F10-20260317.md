## 1. 範圍

本輪依 `docs/測試/00-規範與原則/Emorapy-測試設計鐵律.md` 對以下模塊做前端側重新核查：

重點不是補數量，而是確認：

- 管理台與主前端關鍵頁面是否仍具可回歸性

## 2. 本輪結論

結論：本輪未發現新的前端業務缺陷，發現並收斂了 2 類測試質量問題。

### 2.1 `CaseDetail` 預期錯誤分支把 logger 打進 stderr

文件：

- `frontend/src/pages/Case/Detail/index.test.tsx`

現象：

- `getCase` / `submitCase` 失敗分支本來就是受測行為
- 頁內 `logger.error` 將預期錯誤直接輸出到 stderr

判定：

- 非產品缺陷
- 屬測試輸出信噪比問題

處理：

- mock `@/utils/logger`
- 保留原本對 `message.error`、retry、導航與錯誤 fallback 的業務斷言

### 2.2 `ProfilePairing` 測試首屏與關係檔案區塊混入 antd 假警告

文件：

- `frontend/src/pages/Profile/Pairing/index.test.tsx`

現象：

- 未激活關係表單時，antd 會輸出 `useForm is not connected`
- 測試過程同時會帶入 responsive observer 相關警告噪音

判定：

- 非產品缺陷
- 屬 UI library 警告污染，不應遮蔽真失敗

處理：

- 在測試內僅過濾已知 `useForm` 假警告
- 不改產品代碼，不吞掉其他未知錯誤

### F08 配對與檔案

- pairing active/pending/失敗/retry/relationship profile 保存行為仍與當前實作一致
- 本輪未發現新的權限或導航缺陷

### F10 Admin

- 本輪未發現新的管理台前端缺陷

按本輪範圍，`F03/F05/F06/F08/F10` 前端測試與當前產品行為一致，沒有新增未收斂的真業務缺陷。
