## 1. 判定口徑

1. 本清單只放「現碼允許或現行 gate 尚未覆蓋，因此必須被明確標記」的剩餘風險。
2. 若問題是待修代碼缺陷或跨端/跨環境差異，應另開或更新 `07-待處理問題與治理/待處理/` 的專項文件，而不是放在本清單中。
3. 若問題已被代碼、測試、正式 evidence 或治理任務關閉，只能作歷史回鏈，不得繼續列為活躍風險。
4. Web / Admin / App 的證據不得相互替代；任何 mock-backed 或 local-only 證據都必須在使用處標明證明範圍。

## 4. 發版與文件使用規則

1. `R-01` 是證據邊界風險，不是待修 Web UI / Admin UI 代碼缺陷。
2. Web P0 true-service local artifact 只能證明其指定 target；若 release gate 要求 staging / release / production target，必須為該 target 重新產生 artifact。
3. `AI_MOCK=true` 的 pass 可以作 no-cost release-like smoke，但必須保留 `ai_mock=true` 或等價標記，不得被摘要成「AI provider 已驗證」。
4. App release sign-off 仍由 `App跨端Parity落地待辦-2026-05-05.md`、`App外部ReleaseSignoff待辦-2026-05-16.md` 與 `npm --prefix mobile run release:completion:audit` 裁決；Web P0 true-service 不替代 TestFlight、physical device、push provider delivery 或 production native crash runtime evidence。
