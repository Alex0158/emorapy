## 1. 全局請求機制

- HTTP 主通道：`services/request.ts`（Axios）
- SSE 訪談：`services/sseRequest.ts`
- SSE 聊天：`services/api/chat.ts` 的 `connectChatStream`

自動注入：
- `Authorization`（普通 API）
- `X-Session-Id`（快速體驗）
- `X-Locale`（語系）

## 2. API 模組清單（前端調用）

## 2.1 Auth（`services/api/auth.ts`）

- `POST /auth/register`
- `POST /auth/login`
- `POST /auth/send-verification-code`
- `POST /auth/verify-email`
- `POST /auth/reset-password`
- `POST /auth/reset-password-confirm`
- `POST /auth/claim-session`

## 2.2 Session（`services/api/session.ts`）

- `POST /sessions/quick`
- `POST /sessions/refresh`

## 2.3 Case（`services/api/case.ts`）

- `GET /cases/by-session`
- `POST /cases/quick`
- `POST /cases/collaborative`
- `POST /cases`
- `GET /cases`
- `GET /cases/:id`
- `PUT /cases/:id`
- `POST /cases/:id/submit`
- `POST /cases/:id/evidence`

## 2.4 Judgment（`services/api/judgment.ts`）

- `POST /judgments/generate/:id`
- `GET /judgments/:id`

## 2.5 Reconciliation（`services/api/reconciliation.ts`）

- `POST /judgments/:id/reconciliation-plans`
- `GET /judgments/:id/reconciliation-plans`

- `POST /reconciliation-plans/:id/select`

## 2.6 Execution（`services/api/execution.ts`）

- `POST /execution/confirm`
- `POST /execution/checkin`
- `GET /execution/status`
- `GET /execution/dashboard`

## 2.7 Pairing（`services/api/pairing.ts`）

- `POST /pairing/create`
- `POST /pairing/join`
- `GET /pairing/status`
- `POST /pairing/cancel`

## 2.8 User（`services/api/user.ts`）

- `GET /user/profile`
- `PUT /user/profile`
- `GET /profile/me`
- `PUT /profile/me`

補充：
- `POST /user/avatar` 已封裝於 `user.ts/uploadAvatar`，由 request 攔截器附加 token。
- `POST /auth/claim-session` 屬 auth 成功後的弱依賴後置流程，與 quick flow 共用責任。
- `/profile/pairing` 主責仍是配對與關係檔案；若在配對成功態觸發 `POST /interview/start`，屬 F06 次責入口。

## 2.9 Content（`services/api/content.ts`）

- `GET /content-items`

備註：後端的 recommendations/link 端點已存在，但前端主流程未深度使用。

## 2.10 Psych Profile（`services/api/psychProfile.ts`）

- `GET /psych-profile`
- `GET /psych-profile/feedback`
- `POST /psych-profile/consent`
- `DELETE /psych-profile`

## 2.11 Interview（`services/api/interview.ts` + `sseRequest.ts`）

普通 HTTP：
- `POST /interview/start`
- `GET /interview/resume`
- `GET /interview/:id`
- `POST /interview/:id/end`
- `POST /interview/:id/retry`

SSE：
- `POST /interview/:id/respond`
- `POST /interview/:id/skip`

## 2.12 Chat（`services/api/chat.ts`）

- `POST /chat/rooms`
- `GET /chat/rooms/:roomId`
- `POST /chat/rooms/:roomId/invites`
- `POST /chat/invites/:inviteCode/accept`
- `POST /chat/invites/:inviteCode/decline`
- `GET /chat/rooms/:roomId/messages`
- `POST /chat/rooms/:roomId/messages`
- `POST /chat/rooms/:roomId/request-judgment`
- `GET /chat/rooms/:roomId/judgment-status`
- `POST /chat/rooms/:roomId/leave`
- `POST /chat/rooms/:roomId/kick-b`
- `GET /chat/rooms/:roomId/stream`（SSE）

補充：
- `POST /chat/invites/:inviteCode/accept` 收斂為 `User only`；接受 invite 前需先登入。
- 匿名 A 房主建立 `POST /chat/rooms/:roomId/invites` 時，仍需 `room.session_id` 與當前 `canonical session_id` 匹配。
- `GET /chat/rooms/:roomId/judgment-status` 僅負責 handoff；正式 judgment 詳情消費由登入後的 `/judgment/:id` 承接。

## 2.13 Admin（`services/api/admin.ts`）

- 使用獨立 admin token（與普通 token 分離）
- 調用 `/admin/*`：login/me/jobs/health/config/users/reports/audit/admin-users/runtime/alerts/feature-flags

## 3. SSE 契約（前端消費）

## 3.1 訪談 SSE

事件：
- `token`
- `metadata`
- `safety_alert`
- `complete`
- `error`

超時策略：
- 30 秒未收到 token：`RESPONSE_TIMEOUT`
- 60 秒無事件：`CONNECTION_TIMEOUT`

## 3.2 聊天 SSE

事件：
- `ready`
- `ping`
- `message`
- `invite`
- `room_status`
- `system`（保留）

---

## 4. 授權口徑

- 普通 JWT：主要受保護業務 API
- Session ID：快速體驗弱保護路徑
- Admin JWT：僅 `/admin/*`
- 注意：`chat/room*` 路由層未包 `ProtectedRoute`，授權在 API/後端層完成

---

## 5. 維護規則

新增或修改接口時，至少同步：
1. `frontend/src/services/api/*`（或 `request.ts` / `sseRequest.ts`）  
2. 本文件  
3. `docs/後端設計/03-API設計.md`  
4. `docs/核心開發文件/功能特性清單.md`（若能力邊界受影響）
