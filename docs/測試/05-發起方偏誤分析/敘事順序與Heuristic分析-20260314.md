# 敘事順序與 Heuristic 設計分析

**日期**：2026-03-14  
**對應**：發起方偏誤分析中的「敘事順序」「Heuristic」兩項

---

## 一、敘事順序：深入分析與優化方案

### 1.1 現況：A 永遠先於 B 的具體位置

| 位置 | 檔案 | 內容 | 影響 |
|------|------|------|------|
| 情感分析 prompt | `ai.service.ts` L387-390 | `角色 A 的描述：` 在前，`角色 B 的描述：` 在後 | AI 先讀 A、先分析 A，personA 可能獲得更完整的結構化輸出 |
| 情感分析 JSON 結構 | 同上 L421-430 | personA 欄位在前，personB 在後 | 輸出順序可能影響後續消費時的注意力分配 |
| 判決主 prompt | `ai.service.ts` L827-830 | `角色 A：` 在前，`角色 B：` 在後 | 原始陳述的呈現順序 |
| **判決回應結構** | `ai.service.ts` L874-884 | **「1. 先對角色 A 說話」「2. 再對角色 B 說話」** | **強制規定輸出順序，首因效應最直接** |
| 調整比重段落 | L914-915 | `角色 A：[X]%` 在前 | 比例呈現順序 |
| 可直接用的對話 | L932-935 | 角色 A 的對話範本在前 | 建議順序 |
| 責任評估 prompt | L1126-1129 | 角色A陳述、角色B陳述 | 結構化評估的輸入順序 |

**核心問題**：判決 prompt 明確要求「先對角色 A 說話、再對角色 B」，這會讓 AI 在生成時：
1. 先投入認知資源處理 A
2. A 的段落通常更長、更細（因有具體陳述可引用）
3. 讀者（用戶）先讀到 A，可能形成「A 是主訴方、B 是回應方」的框架
4. 首因效應：先接收的資訊對後續判斷的錨定作用

### 1.2 優化方案

#### 方案 A：順序中性化（推薦，改動最小）

**做法**：在判決 prompt 的「你們之間發生了什麼」段落前，加入明確指令：

```
**順序中性原則**：以下「先對 A、再對 B」僅為寫作結構，不代表責任輕重或誰的視角更可信。你必須對雙方給予同等深度的理解與篇幅。若你發現自己對某一方的描述明顯更長或更細，請回頭平衡。
```

**優點**：不改輸出結構、不影響用戶閱讀習慣，僅透過指令降低模型對順序的過度依賴。  
**缺點**：依賴模型遵守，無法完全消除順序效應。

---

#### 方案 B：依情境動態決定先後

**做法**：根據情感分析結果，決定先對誰說話：
- 若 `personA.communicationPattern` 含「自我否認」「長期被否定」等，先對 A（因 A 更需要被聽見）
- 若 `personB.communicationPattern` 含「未被充分表達」「沉默」等，先對 B（因 B 的視角容易被忽略）
- 預設：隨機或輪替（例如依 case_id 尾數決定）

**優點**：順序與臨床需求掛鉤，可減少「總是 A 先」的系統性偏誤。  
**缺點**：需改動 prompt 生成邏輯，且「誰更需要先被聽見」的判斷可能出錯。

---

#### 方案 C：並行結構，取消先後

**做法**：改為「同時對雙方說話」的段落結構，例如：
- 「角色 A，你…；角色 B，你…」交錯成一段
- 或「你們之間發生了什麼」改為「從 A 的視角…從 B 的視角…」兩小段並列，不標 1、2

**優點**：從根本上消除先後順序。  
**缺點**：可能影響閱讀流暢度，且需較大 prompt 改動。

---

#### 方案 D：情感分析與責任評估的輸入順序隨機化

**做法**：在呼叫 AI 前，以 50% 機率對調 `plaintiffStatement` 與 `defendantStatement`，並同步對調「角色 A」「角色 B」的標籤；取得結果後再映射回原始 plaintiff/defendant。

**優點**：從輸入層消除順序效應，最徹底。  
**缺點**：
- 需在整個 pipeline 中維護「當前是否已對調」的狀態
- 情感分析、判決、責任評估、摘要等皆需一致對調與還原
- 實作複雜度高，且可能引入新 bug（對調還原錯誤）

---

### 1.3 融合方案：前置梳理 + 順序中性原則

在「你們之間發生了什麼」之前加入**前置梳理層**，以事件為中心建立因果框架，減少「A 先 = A 主」的錨定；再搭配**順序中性原則**，形成雙重防護。

**前置梳理（隱式，不直接展示給用戶）**：
```
在撰寫回應前，請先完成以下梳理：
1. 時間線：依發生順序列出關鍵事件，不按「誰先說」排序
2. 因果鏈：每個事件如何引發下一個？雙方各自的行為與反應如何互相影響？
3. 關鍵轉折：哪個時刻讓衝突升級或僵持？
重要：此梳理應以事件為中心，不以「角色 A 的陳述先出現」來推斷因果。
```

**順序中性原則**：在「先對 A、再對 B」段落前加入「順序僅為寫作結構，不代表責任輕重；須對雙方給予同等深度與篇幅」。

### 1.4 建議實施順序

1. **短期**：在判決 prompt 加入「前置梳理」+「順序中性原則」。
2. **中期**：若角色反轉測試仍顯示順序偏誤，再評估方案 B（動態決定先後）或 C（並行結構）。
3. **長期**：若需徹底消除，再考慮方案 D（輸入隨機對調，需完整設計與測試）。

## 二、Heuristic 設計：邏輯說明與產品定位評估

### 2.1 Heuristic 是什麼

`deriveRatioFromSignals` 是責任比例的**規則型 fallback**：當 AI 結構化評估失敗或需校準時，用「溝通模式關鍵字」推估誰的「調整空間」較大。

**輸入**：
- `analysis.personA.communicationPattern`（AI 對角色 A 的溝通模式描述）
- `analysis.personB.communicationPattern`（同上，角色 B）
- `plaintiffStatement`（A 的原始陳述）
- `defendantStatement`（B 的原始陳述）

**輸出**：`{ plaintiff, defendant }`，其中 plaintiff 對應角色 A（發起方）。

**邏輯**：`plaintiff` 從 50 起算，越高表示 A 的調整空間越大。

### 2.2 計分規則

| 模式類型 | 關鍵字 | 分數 | 作用對象 |
|----------|--------|------|----------|
| 高壓負向 | 批評、指責、翻舊帳、攻擊、蔑視、羞辱、控制、威脅 | +14 | 出現在誰的 communicationPattern，誰的調整空間增加 |
| 追逐型 | 追逐、情緒化、逼問、高壓 | +6 | 同上 |
| 迴避型 | 迴避、沉默、退縮、敷衍、石牆 | +8 | 同上 |
| 正向 | 承認、願意、修復、傾聽、同理 | -6 | 同上，調整空間減少 |
| 陳述壓力 | 總是、從不、每次都、你就是、你根本 | +6 | 直接掃原始陳述 |
| 陳述緩和 | 我願意、我可以、我會改、我想理解 | -4 | 同上 |

**計算式**：
```
plaintiff += scorePattern(personA) - scorePattern(personB)
plaintiff += statementPressure(plaintiffStatement) - statementPressure(defendantStatement)
```

**解讀**：
- A 有「批評、追問」→ A 的調整空間增加（符合：追問會讓對話更難，A 可先調整）
- B 有「迴避、沉默」→ B 的調整空間增加（plaintiff 減少 = defendant 增加）
- A 有「我願意改」→ A 的調整空間減少（已展現意願）

### 2.3 與產品定位的對齊

**產品定位**（來自項目概述、產品設計）：
- 伴侶衝突的第三方判決與關係修復
- 「調整空間」= 在此互動循環中，誰可以率先做出不同選擇的餘地
- 非道德責任、非過錯歸屬

**Heuristic 的設計意圖**：
- 用 Gottman 四騎士、EFT 追逐-迴避等概念，將「溝通模式」轉為可解釋的調整空間推估
- 當 AI 失敗時，提供一個不隨機的、有臨床依據的 fallback

**合理性評估**：

| 面向 | 評估 | 說明 |
|------|------|------|
| 理論對齊 | ✓ 合理 | 批評、蔑視、迴避、石牆等為 Gottman/EFT 常見概念，與伴侶治療文獻一致 |
| 語義對齊 | ✓ 合理 | 「誰的改變更容易帶動正向連鎖」與「調整空間」的產品定義一致 |
| 發起方偏誤 | ⚠ 需注意 | 發起方常是「主動提出問題」的一方，陳述中易出現追問、指責等詞，可能系統性提高 A 的調整空間；但這不一定是錯——若 A 確實在追問、翻舊帳，臨床上常會建議 A 先調整 |
| 權重主觀性 | ⚠ 可優化 | 14 vs 6 vs 8 等數值為經驗設定，缺乏實證校準；可考慮用測試案例回歸調參 |
| 單方陳述 | ✓ 已處理 | B 留空時 clamp 在 45–55，避免單方主導 |

### 2.4 結論：Heuristic 對產品定位是否合理

**結論：合理，但需監測。**

1. **設計邏輯**：用溝通模式推估「誰的改變更能打破循環」，與「調整空間」的產品語義一致，且 fallback 有臨床依據。
2. **潛在偏誤**：發起方因角色特性（先說、先追問）可能被 heuristic 系統性標記為「較多調整空間」。這在「約會遲到」等案是正確的（A 翻舊帳 → A 需調整），但在「家務分工」等案若 A 是受害者、B 是迴避方，heuristic 應正確給 B 較多空間——關鍵在於 `personB.communicationPattern` 是否被充分識別。
3. **建議**：
   - 保留現有設計，不刪除 heuristic（其為 AI 失敗時的重要防線）
   - 用角色反轉測試驗證：對調 A/B 陳述後，heuristic 輸出的比例是否合理反轉
   - 若測試顯示 heuristic 過度懲罰「追問型發起方」，可微調權重（例如將「追逐、逼問」從 +6 降為 +4），或加入「發起方折扣」邏輯（需謹慎，避免過度校正）

## 三、發起方偏誤優化計劃（總覽）

| 項目 | 決策 | 行動 | 優先級 |
|------|------|------|--------|
| **敘事順序** | 需優化 | 判決 prompt 加入「前置梳理」+「順序中性原則」 | P0 |
| **陳述篇幅** | 合理 | 不需優化（模擬自白模式） | — |
| **Heuristic** | 合理 | 保留，以角色反轉測試監測 | — |
| **歸因偏誤提醒** | 可加強 | 在判決、責任評估 prompt 中重複強調（單方陳述時警惕採信發起方） | P1 |
| **原告/被告語義** | 可優化 | 對外展示統一改為「角色 A / 角色 B」 | P2 |
| **角色反轉測試** | 驗證手段 | 新增 1–2 案（A/B 陳述對調），驗證比例是否合理反轉 | P1 |

**實施順序**：
1. P0：敘事順序優化（前置梳理 + 順序中性原則）
2. P1：歸因偏誤加強、角色反轉測試案例
3. P2：原告/被告語義統一（視資源而定）
