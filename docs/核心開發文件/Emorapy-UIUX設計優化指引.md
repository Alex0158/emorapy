**適用範圍**：前端開發、UI/UX 設計、產品企劃

## 1. 核心設計鐵律與邊界 (Core Rules & Boundaries)

在進行任何頁面重構與優化時，必須嚴格遵守以下規則：

1. **最高鐵律：功能與業務邏輯零破壞**
   - 絕對不能破壞或影響現有頁面的任何功能、特性或業務邏輯（包括數據流、API 調用、用戶權限、狀態機流轉等）。
   - 若考慮升級操作方式（如簡化交互或優化流程），必須基於對該優化點背後的業務邏輯、用戶需求和細節的全面深入理解。
   - **底線**：確保不會意外降低用戶體驗、引入 bug 或影響性能。如果對某個邏輯理解不足，寧可保持原狀。
2. **設計自由度：服務於業務目標的重組**
   - 不要受限於現有頁面的視覺結構。可以新增頁面（如引導頁、儀表板）、刪減冗餘頁面，或重組頁面邏輯以提升整體用戶旅程。
   - 所有變更必須維持數據一致性和安全性。
3. **從零構想：拋棄舊有包袱**
   - 完全忽略現有頁面的生硬風格、UI 元素與交互模式。
   - 採用人類頂級審美眼光，從零開始重新構想，設計出符合「高端、專業、具同理心」的現代化頁面系統。

## 2. 品牌與視覺美學定義 (Global Aesthetics)

以頂級 Graphic Designer 視角，確保視覺一致性與情感共鳴：

- **色彩計畫 (Color Palette)**
  - **主色調 (Primary)**：**寧靜鼠尾草綠 (Sage Green)**。傳遞療癒、生機、平靜與專業感，能在用戶情緒高漲時起到視覺降溫的作用。
  - **強調色 (Accent)**：**溫暖陶土色 (Terracotta)** 或 柔和珊瑚橘。用於 CTA (呼籲行動) 按鈕、重要提示與情感連結處，保留人情味。
  - **背景色 (Background)**：**珍珠白/米白 (Off-white)**。取代純白，降低螢幕對比帶來的視覺刺眼感，營造安全的包覆感。
  - **文字色 (Text)**：深炭灰色 (Dark Charcoal) 取代純黑，提升閱讀舒適度。
- **字體規範 (Typography)**
  - **標題字體**：幾何無襯線體（如 `Plus Jakarta Sans` 或 `Outfit`），展現現代、高端與科技感。
  - **內文字體**：高易讀性無襯線體（如 `Inter` 或 `Noto Sans TC`），確保長篇判決或對話的閱讀體驗。
- **UI 元素風格**
  - **形狀**：大量使用大圓角 (16px - 24px)，消除銳利邊緣，傳遞柔軟與包容。
  - **質感**：適度引入毛玻璃效果 (Glassmorphism) 與柔和的彌散陰影 (Soft Drop Shadows)，建立空間層次感。
  - **留白 (White Space)**：放大元素間距，減少認知負荷，讓畫面「呼吸」。

## 3. 交互與用戶體驗原則 (Interaction & UX Principles)

以 2026 年頂級水準核心 Skill **「Intent-Driven Adaptive UI with Explainable Human-AI Harmony」** 為最高指導原則：

- **Intent-Driven (意圖驅動)**
  - 每個頁面/元件都從「用戶此刻的核心意圖是什麼？」出發重新組織。
  - **預測下一步**：不展示一堆按鈕，而是基於現有狀態（如案件 `draft`, `submitted`, `judgment_ready`）預測並優先呈現最可能的下一步行動（例如：首頁對回訪用戶直接顯示「繼續上次的訪談」或「查看最新判決」）。
- **Adaptive UI (自適應介面)**
  - 介面根據用戶角色、時間、歷史行為與當前進度動態調整布局與內容優先級（在不修改後端 API 的前提下，利用前端現有狀態如 `caseStore`, `authStore` 實現）。
  - **移動端優先**：全面改用**底部導航列 (Bottom Tab Bar)**，提升單手操作體驗。
- **Explainable & Human-AI Harmony (可解釋的 AI 和諧)**
  - **透明度**：在 AI 判決或和好方案生成時，讓用戶清楚「AI 怎麼得出這個結論？」（例如：在判決旁顯示「基於你們在對話中提到的 3 個關鍵衝突點...」的簡短推理路徑）。
  - **人類掌控感**：強調用戶的選擇權，提供一鍵「我自己來」或「重新生成」的切換，避免 AI 顯得過於強勢或冷酷。
- **2026 頂級視覺與性能風格**
  - **Organic/Anti-Grid 佈局**：非對稱但平衡的排版，打破傳統後台的死板網格。
  - **Glassmorphism 2.0**：輕觸感深度的毛玻璃效果，營造空間感與呼吸感。
  - **Kinetic Typography**：標題隨滾動或狀態改變產生微動，增加生命力。
  - **Purposeful Micro-Interactions**：每一個動畫（如 Hover、成功打卡）都有明確目的與情感反饋。
  - **Performance-First**：嚴格控制 LCP < 1.8s、INP < 150ms。

## 4. 核心頁面重構藍圖 (Page Design Blueprint)

### 4.1 首頁 (Home - `/`)
- **目的**：傳遞 "Turn conflict into connection" 品牌價值，最大化轉化率。
- **Intent-Driven 狀態分流**：
  - **訪客/新用戶**：展示 Hero 區塊與品牌價值，主推「快速體驗」。
  - **回訪/登入用戶**：Hero 區塊自動替換為「Adaptive Dashboard」，直接顯示當前最緊急的下一步（如：「你有一個判決待查看」或「繼續未完成的訪談」）。
- **核心亮點：操作流程模擬視角 (Flow Simulation)**
  - **保留並升級**：保留現有極具創意的「雙手機模擬對話」概念。
  - **優化方向**：將手機模型升級為更具質感的 3D 或極簡玻璃邊框 (Glassmorphism 2.0)；優化打字機與 AI 介入的動畫節奏，讓「降溫 -> 分析 -> 和好」的過程更具戲劇性與說服力。
- **佈局**：
  - **Organic/Anti-Grid 佈局**：打破死板的左右對稱，利用浮動卡片與柔和漸變背景創造空間感。
  - **Kinetic Typography**：大標題隨滾動產生微動效果。

### 4.2 快速體驗流程 (`/quick-experience/create`, `.../result/:id`)
- **目的**：無摩擦體驗 AI 能力，降低防備心。
- **佈局與交互**：
  - **Create**：改為**全螢幕沉浸式對話表單 (Typeform 風格)**。隱藏全局導航，一次只展示一個問題。大字體提問，寬大輸入框，底部懸浮「下一步」。
  - **Result (Explainable AI)**：AI 分析時顯示具科技感與安撫感的呼吸燈骨架屏。結果以卡片分層呈現，**必須加入「AI 是如何得出這個結論的」簡短說明**，提升信任感。底部強烈 CTA 引導註冊保存。

### 4.3 認證流程 (`/auth/login`, `/auth/register`)
- **佈局**：左右/上下分欄佈局，採用 Organic 非對稱比例（如 4:6）。一側為品牌意象圖與 Slogan，另一側為極簡表單。
- **UI 元素**：浮動標籤輸入框 (Floating labels)，大型圓角提交按鈕，支援社交登入（若未來擴充）。

### 4.4 案件與儀表板 (`/case/list`, `/case/create`, `/case/:id`)
- **佈局與 Intent-Driven**：
  - **Dashboard (原 List)**：徹底廢棄傳統表格 (Table) 視圖。頂部新增「關係健康度」或「連續修復天數」的激勵模塊。下方為「進行中」大卡片（高亮下一步行動）與「歷史紀錄」時間軸。
  - **Detail**：雙方陳述採「對話氣泡式」對比排版，證據區塊採相冊網格展示，降低嚴肅的「法庭」感，提升「修復」感。

### 4.5 判決與和好方案 (`/judgment/:id`, `/reconciliation/*`, `/execution/*`)
- **佈局與 Explainable AI**：
  - **Judgment**：將長篇 Markdown 拆分為**手風琴折疊面板 (Accordion)** 或標籤頁 (Tabs)。在每個判決段落旁，增加「AI 洞察標籤」，解釋該段落是基於哪段對話或證據生成的。
  - **Reconciliation**：橫向滑動卡片 (Carousel) 展示不同方案，清晰標註難度與預估時間。提供「一鍵換一批」或「我自己微調」的選項，增強人類掌控感。
  - **Check-in (Purposeful Micro-Interactions)**：引入儀式感。上傳照片區塊設計為拍立得相框樣式，點擊完成時觸發具情感反饋的微交互動畫（如 Confetti 或溫暖的發光效果）。

### 4.6 心理訪談與聊天 (`/interview/:sessionId`, `/chat/room/:roomId`)
- **佈局與交互**：
  - **對話介面**：極簡、沉浸式的對話框。AI 氣泡使用主色 (Sage Green)，用戶氣泡使用中性色。
  - **Purposeful Micro-Interactions**：AI 回覆時顯示平滑的打字機動畫。底部輸入框上方提供「快捷情緒標籤」按鈕（如：我需要冷靜、跳過），減少用戶在情緒激動時的打字負擔。

## 5. 技術實施建議

1. **樣式框架**：
   - 保持 React 19 + Vite 架構。
   - 引入 **Tailwind CSS** 替換部分繁瑣的 Less，以加速客製化 UI 與響應式佈局的開發。
   - 透過 Ant Design 的 `ConfigProvider` 深度覆寫 Design Token（如 `colorPrimary`, `borderRadius`, `colorBgContainer` 等），使其完全貼合新視覺規範。
2. **動畫庫**：
   - 繼續深化使用 **Framer Motion**，統一全局的頁面切換動畫（如 Fade-in, Slide-up）與微交互。
3. **狀態與 API**：
   - **嚴格禁止**修改 Zustand Store 的核心邏輯與 React Query / Axios 的 API 請求參數。UI 組件僅作為狀態的渲染層與事件的觸發層。
