## 角色與身份

| 術語 | 定義 |
|---|---|
| 訪客（Guest） | 未登入前台用戶，可進快速體驗與部分 chat 弱入口；若無有效 session，不能穩定持有匿名房間能力。 |
| 用戶（User） | 持有 JWT 的前台登入用戶。 |
| 管理員（Admin） | 持有 admin token 的後台運維用戶，走獨立 RBAC。 |
| Session 用戶 | 無 JWT，但持有 `session_id`（`X-Session-Id`）的匿名會話身份；可承接 quick flow 與部分 chat 能力。 |

## 路由/頁面術語

| 術語 | 定義 |
|---|---|
| ProtectedRoute | 需登入路由守衛，未登入跳 `/auth/login`；直連保護頁時未登入會強制跳轉登錄。 |
| PublicRoute | 公開認證頁守衛，已登入跳 `/case/list`。 |
| 管理端登入守衛 | frontend-admin 中無 admin token 訪問 `/admin/*`（除登錄頁）時重定向至 `/admin/login`（現由 AdminSectionLayout 內檢查 token 並重定向）。 |
| 弱入口 | 可直接 URL 進入但不經強守衛的頁面（如 `/chat/room/:roomId`）。 |
| 候選保留路由 | 主要僅做 redirect 或兼容用途的路由（如 `/auth`）。 |
| 外部 Admin 子頁 | 由 `frontend-admin` 承接的 admin 實際頁面，如 `/admin/health`、`/admin/reports`。 |

## 接口治理術語

| 術語 | 定義 |
|---|---|
| 已使用 | 後端接口存在，且前端有調用證據。 |
| 候選廢棄 | 後端接口存在但前端主流程未接線，尚未正式下線。 |
| 已確認廢棄 | 明確不再使用且已完成替代與下線計劃。 |
| 候選保留 | 在功能文檔中表示「保留既有編碼做歷史索引，但不計入主功能序列」的候選能力狀態。 |
| 平台保留 | 在功能文檔中表示「監控/平台能力保留於附錄，不視為前台主功能」的狀態。 |
| 主註冊表 | `全接口清單-主文檔.md`，接口狀態唯一權威來源。 |

## 流程與狀態術語

| 術語 | 定義 |
|---|---|
| Quick Case | 快速體驗建案，通常由 session 驅動。 |
| Collaborative Case | 同設備雙人輪流輸入的協作聽證案件。 |
| Interview Session | 心理訪談會話，具輪次與 pipeline 狀態。 |
| Chat -> Judgment | 聊天室語料轉換為 case + judgment 的鏈路。 |
| claim-session / 升格 | quick case 在登入/註冊成功後，經 `POST /auth/claim-session` 綁定到正式帳號的後置流程。 |
| canonical session_id | 前端統一 session 管理層持有的當前有效匿名會話 ID；用於 session 身份判定，不以任意 storage 臨時值作準。 |
| 冪等（Idempotent） | 重複請求只產生一次有效結果（例如 chat request-judgment）。 |

## 核心狀態值

| 類別 | 狀態值 | 說明 |
|---|---|---|
| Case | `draft` / `submitted` / `in_progress` / `judgment_failed` / `completed` / `cancelled` | 案件生命週期 |
| Interview | `in_progress` / `processing` / `completed` / `processing_failed` / `abandoned` | 訪談主狀態 |
| ChatRoom | `solo_active` / `invite_pending` / `invite_accepted` / `group_active` / `judgment_requested` / `judgment_completed` / `judgment_failed` / `archived` | 聊天房狀態 |
| Pairing | `pending` / `active` / `cancelled` / `temp` | 配對狀態 |
| Judgment Acceptance | `accepted` / `rejected` | 判決接受結果；一旦成立，不應再次重選。 |
