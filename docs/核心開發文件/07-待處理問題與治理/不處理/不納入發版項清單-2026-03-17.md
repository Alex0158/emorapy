## 1. 判定口徑

1. 「不納入」代表不作當前 Web / Admin / App 完成判定或 release closure 的阻塞條件；不代表功能不存在、永不處理或可以在正式文件中寫成完成。
2. 任何已在現碼落地的能力，不得因保留在本清單而被描述成仍停留在規劃階段。
3. 任何仍是活躍 blocker 的事項，必須由 `待處理/` 專項文件、release audit 或正式測試規範承接，不在本文件重複作活躍治理。
4. 使用本文件時，必須同時查看 `00-活躍問題總覽.md`、`已知風險清單-2026-03-17.md`、App release audit 與對應正式規格；不得只用本文件裁決當前狀態。

## 3. 已修正的舊口徑

| 舊口徑 | 現行裁決 |
| --- | --- |
| App landing 被描述為仍只有規劃文檔 | 過期。App 已有 `mobile/app` route group、`mobile/src/platform` adapter、shared contracts / api-client 消費與 M0-M5 gates；未完成的是 M6 外部 release sign-off |

| 視覺 / 窄視口回歸被描述成單次主鏈交互收口結論 | 應改寫為穩定驗收邊界：targeted route / A11Y / manual evidence 不等於 full-site pixel / device-lab sign-off |
