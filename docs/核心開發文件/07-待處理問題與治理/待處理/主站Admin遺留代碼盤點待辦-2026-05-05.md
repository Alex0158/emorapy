## 1. 裁決

核心文件已明確：主站 `frontend/` 的 `/admin/*` 只負責 `AdminRedirect`，實際 Admin 頁由 `frontend-admin` 承接。只讀審計仍看到主站保留若干 admin 相關 client / guard / layout 檔案。這不一定是功能 bug，但屬於**邊界未完全收斂**，需要盤點是否為死碼、測試輔助或仍被某些入口引用。

## 2. 待盤點範圍

1. `frontend/src/services/api/admin.ts`
2. `frontend/src/components/common/AdminSectionLayout.tsx`
3. `frontend/src/components/common/AdminPermissionRoute.tsx`
4. `frontend/src/utils/adminEntry.ts`
5. `frontend/src/router/index.tsx` 中 `/admin/*` 的真實職責
6. Header / nav 測試中對 ops console、admin login URL 的 mock 是否仍必要

## 3. 目標狀態

1. 主站只保留 AdminRedirect 與必要的外部 Admin URL 工具。
2. 若 admin API client / guard / layout 已無 runtime 引用，刪除或移到 `frontend-admin`。
3. 若仍需保留，文件需明確說明保留理由與引用鏈，避免未來工程師誤以為主站仍承接 Admin 頁。

## 4. 驗收命令

```bash
rg -n "AdminSectionLayout|AdminPermissionRoute|services/api/admin|getAdminLoginUrl|opsConsole|/admin" frontend/src frontend-admin/src
npm run docs:check
npm run test:run --workspace frontend
npm run build --workspace frontend
npm run build --workspace frontend-admin
```
