## 1. 產品定位（現況）

- 核心場景：伴侶衝突中的雙方陳述收斂、AI 判決與可執行後續方案。
- 核心主軸：
  - 快速體驗（低門檻）
  - 正式案件流程（登入後）
  - 訪談畫像（提升判決上下文）
  - 聊天室轉判決（即時對話收斂）

---

## 2. 已實作能力地圖

## 2.1 快速體驗

- 前端：`/quick-experience/create`、`/quick-experience/result/:id`、`/quick-experience/collaborative`
- 後端：
  - `POST /api/v1/sessions/quick|refresh`
  - `POST /api/v1/cases/quick|collaborative`
  - `GET /api/v1/cases/:id/judgment`
  - `POST|DELETE /api/v1/cases/:id/evidence*`
- 特性：可匿名進入、可補傳證據、可後續關聯到註冊帳號。

## 2.2 正式案件流程

- Case：建立、列表、詳情、更新、提交
- Judgment：生成、查看、接受、repair、metrics
- Reconciliation：生成、列表、詳情、選擇
- Execution：confirm/checkin/status/dashboard

## 2.3 心理畫像訪談

- 前端：`/interview/:sessionId`、`/interview/:sessionId/result`、`/profile/my-story`
- 後端：start/respond/skip/end/get/resume/retry + psych-profile/feedback/consent/delete
- 關鍵行為：
  - 需 consent
  - respond/skip 採 SSE
  - async pipeline：分類 -> 摘要 -> 洞察 -> richness -> feedback
  - failed 可 retry

## 2.4 聊天室 Chat v1

- 前端：`/chat/room`、`/chat/room/:roomId`
- 後端：rooms/invites/messages/stream/request-judgment/judgment-status/leave/kick-b
- 規則：
  - A 發起、B 受邀
  - history visibility + message visibility 雙層裁切
  - 轉判決由 A 發起
  - 只納入可見且符合規則的 user_text 訊息

## 2.5 管理台與運維

- 前端 `/admin/*` 僅轉導到獨立 admin 入口 URL
- 後端 `/api/v1/admin/*` 提供 RBAC、報表、作業、告警、配置
- 系統提供 `health/live/ready` 與受保護 `metrics`

## 3. 核心流程（產品視角）

## 3.1 快速體驗主流程

1. 建立/刷新 session  
2. 建 quick case（可選 collaborative）  
3. 可補證據  
4. 拉取判決  
5. 結果頁引導到正式流程  

## 3.2 訪談主流程

1. consent  
2. start  
3. 多輪 respond/skip（SSE）  
4. end  
5. processing -> completed（或 failed -> retry）  
6. result + my-story 展示  

## 3.3 聊天轉判決流程

1. 建房、邀請、加入  
2. 訊息互動 + SSE  
3. A 發起 request-judgment  
4. 後端按可見性與 included_message_ids 裁切  
5. 轉 case 並生成 judgment  
6. 回填房間判決狀態  

---

## 4. 關鍵約束（配置/代碼）

- 訪談：max turns 25、soft target 15、turn interval 3s（預設）
- 訪談：start/respond 有中間件限流，且有業務每小時/每日上限
- 聊天：最小發訊間隔 5 秒，30 秒最多 6 則
- metrics：`METRICS_ENABLED=false` 直接 404；production 需 token 或白名單 IP

---

## 5. 明確邊界（避免誤解）

- 無獨立 `ConsentGuard` 路由守衛（頁面邏輯 + 後端 `requireConsent`）
- 無 `/500` 前端路由
- 無 `/judgment/:id/result` 獨立頁
- 刪除心理畫像時保留 `profile_snapshots`（判決追溯）
- `chat/room*` 路由層未包 `ProtectedRoute`，授權在 API/後端補控
