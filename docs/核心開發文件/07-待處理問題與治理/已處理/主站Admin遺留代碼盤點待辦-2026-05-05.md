## 1. 裁決

核心文件已明確：主站 `frontend/` 的 `/admin/*` 只負責 `AdminRedirect`，實際 Admin 頁由 `frontend-admin` 承接。只讀審計仍看到主站保留若干 admin 相關 client / guard / layout 檔案。這不一定是功能 bug，但屬於**邊界未完全收斂**，需要盤點是否為死碼、測試輔助或仍被某些入口引用。

2026-05-12 二次核驗確認問題仍真實存在：`frontend/src/router/index.tsx` 的 `/admin/*` 只渲染 `AdminRedirect`，但主站 `frontend/src` 仍保留 `services/api/admin.ts`、Admin token/session hooks、`AdminPermissionRoute`、`AdminSectionLayout`、Admin utils/types 與對應 tests。這些 runtime 檔案不再被主站 router 引用，且 `frontend-admin/` 已有完整對應實作，因此判定為主站 Admin Web 實作孤島。

## 2. 待盤點範圍

1. 主站 Admin API client（已刪除）
2. 主站 Admin section layout（已刪除）
3. 主站 Admin permission route（已刪除）
4. `frontend/src/utils/adminEntry.ts`
5. `frontend/src/router/index.tsx` 中 `/admin/*` 的真實職責
6. Header / nav 測試中對 ops console、admin login URL 的 mock 是否仍必要

## 3. 目標狀態

1. 主站只保留 AdminRedirect 與必要的外部 Admin URL 工具。
2. 若 admin API client / guard / layout 已無 runtime 引用，刪除或移到 `frontend-admin`。
3. 若仍需保留，文件需明確說明保留理由與引用鏈，避免未來工程師誤以為主站仍承接 Admin 頁。

## 3.1 五輪方案分析與裁決（2026-05-12）

1. **產品 / 使用者入口輪**：普通用戶主站不應承接任何 Admin session 或治理 UI。最優方案是保留 `/admin/*` 作為外部 Admin Web 入口轉向，讓誤入主站 admin path 的操作者被導到正式管理端，而不是看到主站內殘留治理頁。
2. **工程邊界輪**：`frontend-admin/` 已有獨立 router、Admin API client、permission route、layout、token hooks、OpsJobs / Reports / Settings 等頁面；主站保留同名能力會造成雙實作漂移。最優方案是刪除主站 Admin client/hooks/layout/types/utils，只保留 `frontend/src/utils/adminEntry.ts`。
3. **引用與死碼輪**：`rg` 核驗顯示主站 Admin client/hooks/layout 只在 Admin 孤島內互相引用，沒有被主站 router 接入；`frontend/src/router/index.tsx` 只包含 `path: 'admin/*', element: <AdminRedirect />`。因此刪除孤島比“加註保留理由”更符合現有業務鏈路。
4. **測試與回歸輪**：主站不應再跑 Admin API/hooks tests；Admin Web 的 credential-backed E2E 與 workspace tests 已在 `frontend-admin/` / `e2e/` 承接。主站測試只需保留 `adminEntry`、Header Admin link / version probe 與 router AdminRedirect 行為。
5. **治理防回流輪**：只刪除檔案不足以防止未來回流。應新增 root `web:admin-boundary:check`，明確禁止 `frontend/src` 出現 Admin API client、Admin hooks、permission route、layout、pages 與相關 utils/types；`frontend-admin/` 才是唯一 Admin Web 實作邊界。

## 3.2 本輪實作方案（2026-05-12）

1. 刪除主站 `frontend/src` 內 Admin Web 實作孤島：
   - Admin API client
   - Admin section layout
   - Admin permission route
   - Admin token / session / permission hooks
   - Admin jobs stats query utils
   - Admin ops jobs view-state utils
   - Admin token state utils
   - Admin types
   - 上述孤島的主站單測
2. 保留 `frontend/src/utils/adminEntry.ts` 與 `frontend/src/router/index.tsx` 的 `AdminRedirect`，因它們是主站到外部 Admin Web 的必要入口邊界。
3. 新增 `scripts/check-web-admin-boundary.mjs` 與根層 `npm run web:admin-boundary:check`，防止主站 Admin Web 實作回流。
4. 回寫 `06-接口描述/09-admin.md` 與 `10-Web端/00-Web端凍結基線總覽.md`，將 Admin API client / hooks / layout 的正式承接方固定為 `frontend-admin/`。
