## 使用規則

1. 活躍問題只在本文件與對應狀態目錄維護一份口徑。
2. 問題若處理完成，應先回寫正式正文，再移入 `已處理/`。
3. 問題若不再活躍且無持續決策價值，應降級到 `99-歷史降級索引/`。
4. 問題若涉及 App parity、API/DB/shared package 或平台 adapter 差異，必須同步更新 App / Parity 中層文件或在對應待辦中明確列出尚未更新原因。
