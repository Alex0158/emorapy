# Emorapy 前端測試對齊報告 F01 F02 F04 F07 2026-03-17

## 1. 目的與範圍

本輪對齊依 `docs/測試/00-規範與原則/Emorapy-測試設計鐵律.md` 執行，聚焦本次收斂涉及的前端關鍵鏈路：

- `F01` 快速體驗建案、結果頁、匿名升格與回跳
- `F02` 協作聽證入口與授權邊界
- `F04` 判決詳情頁與 chat handoff
- `F07` Chat room 建立、邀請、發起判決、離房錯誤恢復
- `F09` 認證與 `claim-session` 對 F01 的影響

本輪目標不是追求測試數量，而是確認：

- 現行測試是否仍對應真實業務行為
- 預期失敗分支是否被當成缺陷誤報
- 測試輸出中的噪音是否掩蓋真問題

## 2. 分析結論

結論：本輪未發現新的前端業務缺陷，發現並處理了 2 類測試質量問題。

### 2.1 測試殼路由不完整，導致成功跳轉時產生假性 stderr

文件：

- `frontend/src/pages/Chat/Room/index.test.tsx`

現象：

- `acceptChatInvite` 成功後導向 `/chat/room/:roomId`
- `requestChatJudgment` 成功後導向 `/judgment/:id`
- `leaveChatRoom` 成功後返回 `/chat/room`

原測試只渲染當前頁 route，導致成功導頁後出現 `No routes matched location ...`。

判定：

- 非產品缺陷
- 屬測試容器不完整造成的假性噪音

處理：

- 只補齊成功導頁所需的對應 `Route`
- 不修改任何業務斷言，不弱化錯誤恢復驗證

### 2.2 預期 warning 被直接打到 stderr，降低測試信噪比

文件：

- `frontend/src/store/authStore.test.ts`

現象：

- `login/register` 成功但 `claimSession` 失敗，本就是設計上的「不阻斷登入/註冊成功態」
- store 內部會記錄 `logger.warn`
- 測試未接管 logger，導致預期 warning 混入 stderr

判定：

- 非產品缺陷
- 屬預期觀測點未被測試顯式收斂

處理：

- mock `logger.warn`
- 斷言 warning 確實發生且文案正確
- 保留原本「登入/註冊成功態不被阻斷」的核心斷言
