## 2. 訪談系統關鍵行為

### 2.1 入口與保護

- `POST /api/v1/interview/start` 需要 `authenticate + requireConsent`
- `respond/skip/end/get/retry/resume` 同樣需要 `requireConsent`
- 前端無獨立 `ConsentGuard`，由頁面/流程先檢查，後端作最終防線

### 2.2 限流與節流（實作值）

- 中間件 `interviewStartLimiter`：每小時 10（DDoS 安全網）
- 中間件 `interviewRespondLimiter`：每分鐘 20
- 業務層（`InterviewService.startSession`）：
  - 每小時 substantive session 上限：`INTERVIEW_START_RATE_LIMIT`（預設 3）
  - 每日 substantive session 上限：`INTERVIEW_DAILY_SESSION_LIMIT`（預設 5）
  - substantive 定義：`turns >= 3`
- 單輪節流：`INTERVIEW_TURN_INTERVAL_MS`（預設 3000ms）
- 最大輪數：`INTERVIEW_MAX_TURNS`（預設 25）

### 2.3 SSE 事件契約

訪談 `respond/skip` 會輸出：

- `token`: `{ text }`
- `metadata`: `{ turn_order, intent, target_domains, domains_touched, total_turns, should_end }`
- `safety_alert`: `{ message, severity }`
- `complete`: `{ session_id, status }`
- `error`: `{ code, message }`

前端 `sseRequest.ts` 超時：

- 30s 無 token -> `RESPONSE_TIMEOUT`
- 60s 無事件 -> `CONNECTION_TIMEOUT`

### 2.4 狀態機與 pipeline

訪談狀態：`in_progress | processing | completed | processing_failed | abandoned`

`endSession` 行為：

- turns 或內容不足 -> 直接 `completed`（不進 pipeline）
- 達門檻 -> `processing` 並觸發 async pipeline

pipeline（`AsyncPipelineService`）：

1. narrative extraction（含 domain batch classify）
2. narrative summary
3. insight extraction
4. richness calculation
5. feedback generation

失敗重試：

- `POST /interview/:id/retry`
- 從 `pipeline_step + 1` 續跑
- step retry 為固定延遲 `2s / 4s`（非指數）

## 3. 心理畫像資料行為

### 3.1 profile 資料讀取

- `GET /psych-profile` 回傳：`consent_given`, `consent_at`, `narratives`, `insights`, `richness_score`
- `GET /psych-profile/feedback` 回傳 feedback history（已完成訪談）

### 3.2 遺忘權刪除

`DELETE /psych-profile` 會刪除：

- `profile_insights`
- `profile_narratives`
- `interview_turns`
- `interview_sessions`
- 並重置 user consent

`profile_snapshots` 保留（判決可追溯完整性）

## 4. 聊天室系統（Chat v1）核心規則

### 4.1 角色與狀態

- 角色：`roleA`, `roleB`, `aiMediator`
- 狀態：`solo_active -> invite_pending -> group_active -> judgment_requested -> judgment_completed|judgment_failed`（另有 archived）

### 4.2 訊息可見性

- message scope：`all | summary_only | owner_only`
- room history mode：`share_full_history | share_from_join_time | share_summary_only`
- B 端可見性由兩者共同裁切，且永不可見 `owner_only`

### 4.3 聊天限流

- 同房最小間隔：5 秒
- 同房 30 秒滑窗：最多 6 則

### 4.4 轉判決

- 僅 A 方可觸發 `request-judgment`（現行版本）
- 預設只納入 `message_type=user_text` 且 `visibility_scope=all`
- 可傳 `included_message_ids`，但必須是可納入集合子集
- 房間最終狀態：`judgment_completed` 或 `judgment_failed`

### 4.5 Chat AI Orchestrator

- 僅處理 `visibility_scope=all`
- 約 8 秒節流，避免 AI 洗版
- 命中安全路由會寫 `safety_notice` 並進入 2 分鐘冷卻
