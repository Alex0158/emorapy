## 1. 目標與範圍

- 提供「可落地、可回滾、可驗證」的發佈流程
- 覆蓋前端（Vercel）與後端（Railway 或 PM2）
- 以現有腳本與路由能力為準，不新增不存在的部署步驟

---

## 2. 發佈前檢查（必做）

### 2.1 環境變量

```bash
./scripts/validate-env.sh
```

重點：

- 必填：`DATABASE_URL`、`JWT_SECRET`、`OPENAI_API_KEY`
- 線上環境 metrics 需保護（token 或 IP allowlist）

### 2.2 測試與構建

```bash
./scripts/ci-local.sh
```

包含：

- backend `test:unit` + build
- frontend `test:run` + build

### 2.3 關鍵路徑（建議）

```bash
./scripts/test-critical-paths.sh
```

---

## 3. 發佈流程（建議順序）

1. 合併到 `main`
2. 打版本 tag（如 `vX.Y.Z`）
3. **前端部署（Vercel）**  
   - 客戶端：在 repo 根目錄執行 `vercel --prod`（對應專案 mother-bear-court）  
   - 管理端：在 `frontend-admin/` 執行 `vercel --prod`  
   - 詳見 [Vercel-發布指引.md](./Vercel-發布指引.md)
4. **後端部署（Railway）**  
   - 在 repo 根目錄執行 `railway up`  
   - 詳見 [Railway-發布指引.md](./Railway-發布指引.md)
5. 執行部署驗證
6. 觀察關鍵指標與錯誤日誌

---

## 4. 部署後驗證（必做）

### 4.1 使用腳本

```bash
BACKEND_URL=https://<backend-domain> FRONTEND_URL=https://<frontend-domain> ./scripts/verify-deployment.sh
```

腳本會驗證：

- `/health`
- `/health/ready`
- `/health/live`
- `POST /api/v1/sessions/quick`
- 前端首頁（若傳入非本地 FRONTEND_URL）

### 4.2 手動補充驗證

- 快速體驗：session -> case -> judgment
- 訪談：start -> respond/skip -> end -> result
- 聊天室（若本次有 chat 變更）：建房 -> 發訊息 -> 邀請 -> 接受 -> request-judgment
- 若有 staging gate：確認 `Staging Smoke Gate` 已覆蓋受保護 `/metrics` 的無 token/有 token 兩種情況
- 若有 staging，優先使用 `Staging Deploy and Smoke` workflow，避免只 deploy 未驗證

---

## 5. 回滾策略

### 前端（Vercel）

- 在 Vercel Dashboard 對應專案中，將上一個穩定部署 **Promote to Production**
- 或回退到上一個 tag，依 [Vercel-發布指引.md](./Vercel-發布指引.md) 重新執行 `vercel --prod`

### 後端（Railway）

- 在 Railway Dashboard → 該 Service → Deployments，回到上一個成功部署或從 Git 指定 commit 重新部署
- 若用 PM2，切回上一版進程

### 數據庫

- 優先確保 migration 可前後兼容
- 必要時用 `prisma migrate resolve` 修正狀態（需審慎）

---

## 6. 高風險項檢查

### 訪談系統

- `CONSENT_REQUIRED` 行為正確
- SSE 事件可完整接收
- `processing_failed` 可 retry

### 聊天室轉判決

- A 方觸發權限正確
- `included_message_ids` 邊界驗證正確
- room 狀態最終落在 `judgment_completed|judgment_failed`

### Metrics

- 線上 `/metrics` 不可裸露
- 若有 staging，建議把 `/metrics` 的受保護驗證納入 `Staging Smoke Gate`
