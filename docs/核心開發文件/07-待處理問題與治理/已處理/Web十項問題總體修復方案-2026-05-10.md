## 3. 核心裁決

1. **先地基、再語義、最後共享層**：Admin lint / test entry / skipped test 先恢復，否則後續 Web 修復沒有可信 gate。
2. **Web 對外流程凍結**：十項修復不得改變既有 Web / Admin 對外產品行為；語義修復只改 DOM/control/accessibility/i18n 證據，除非另立產品裁決。
3. **native control 優先**：button / link / input / select / textarea 優先使用原生語義；只有無法用 native element 時才允許 ARIA 補語義，且必須有 role/name/keyboard 測試。
4. **i18n 先於頁面語義收斂**：HTML `lang`、locale runtime、format helper、catalog completeness 必須先於通知日期、表單 label、Chat reply preview 文案等頁面修復建立。
5. **mock-backed 不等於 release 證據**：component/unit/mock E2E 可作局部保護，但 P0 flow 完成必須有真服務或 release 級證據口徑。
6. **shared package 不承接平台副作用**：`packages/api-client` 只能承接 DTO、transport、normalize、platform-neutral domain method；不得下沉 React DOM、router、toast、localStorage、sessionStorage、request cancel registry 或 Admin redirect。
7. **completion audit 是出口條件**：每個問題完成時必須列出代碼 diff、測試命令、文件回寫、剩餘風險與不可替代的證據路徑。

### 5.1 P0-0 baseline freeze / evidence inventory

允許變更：

1. 只新增或回寫文件中的「現況證據」與「待補 gate」。
2. 收集當前失敗命令、skip 名稱、role/name 掃描、hardcoded locale formatting 掃描。

禁止變更：

1. 不得先關掉 failing lint / skipped test / credential gate。
2. 不得把 mock-backed E2E 改名成 release evidence。

最低命令：

```bash
npm run lint --workspace frontend-admin
npm run build --workspace frontend-admin
npm run test:run --workspace frontend
npm run build --workspace frontend
npm run docs:check
rg -n "describe\\.skip|it\\.skip|test\\.skip|role=\"button\"|toLocaleString\\(|toLocaleDateString\\(" frontend frontend-admin e2e
```

完成證據：每個失敗或 skip 必須回寫到對應問題文件，不接受只在對話或 PR 描述中保留。

### 5.2 P0-1 恢復硬 gate

Admin lint：

1. Tailwind 4 / Biome CSS parser 差異應以窄 scope override 或配置修正處理，不得全局關閉 TS/TSX lint。
2. `useImportType` 類型 import warning 要改成 type-only import，不作長期 warning。
3. 保留 reduced motion 可訪問性策略，不得為通過 lint 移除。

Admin test entry：

1. `frontend-admin` 必須有 workspace 級最低測試入口，即使第一階段只覆蓋 utils / route guard / render smoke。
2. credential-gated E2E 可以保留 skip，但 skip 條件、env keys、真服務證據路徑必須文件化。
3. Admin critical flow 若未能跑真服務，不能標成完成，只能標成 local gate restored。

Animation skipped tests：

1. 優先用 deterministic rAF / performance timestamp mock 復原 skipped tests。
2. 若 jsdom 不適合，改成純函數單測或 browser-level smoke，但不能讓 skipped tests 長期留在綠燈總數內。
3. 未解的 animation 測試必須保留待辦，不得只刪除 test case。

最低命令：

```bash
npm run lint --workspace frontend-admin
npm run build --workspace frontend-admin
npm run test:run --workspace frontend-admin
npm run test:run --workspace frontend
rg -n "describe\\.skip|it\\.skip|test\\.skip" frontend frontend-admin e2e
```

### 5.3 P0-2 建立 i18n / A11Y 基礎

HTML lang / i18n：

1. `frontend/index.html` 與 `frontend-admin/index.html` 靜態 `lang` 對齊 default locale。
2. `setLocale()` 或等價 locale 切換成功後同步 `document.documentElement.lang`。
3. zh-TW 作 source catalog；en-US 必須覆蓋所有可見 key；允許 fallback 必須白名單化。
4. 日期、時間、數字格式使用 locale-aware helper，不接受頁面散落裸 `toLocaleString()`。

Icon-only button：

1. 每個 icon-only control 必須有 `type="button"` 與可追溯 accessible name。
2. accessible name 優先使用 i18n key；不同語義不得共用含糊文案。
3. 補 role/name 測試或靜態掃描證據。

Form label / autocomplete：

1. Input / Textarea / Select / file input 必須能以 `getByLabelText` 或 `getByRole(..., { name })` 定位。
2. Placeholder 只能作輔助提示，不能作唯一 label。
3. Credential fields 必須明確 `autocomplete`，Admin credential 與 user credential 不得混用語義。
4. 視覺 label 與 accessible name 要一致；若使用 `aria-label`，必須記錄為例外並說明不能使用 visible label 的原因。

最低命令：

```bash
npm run web:a11y:contracts
npm run test:run --workspace frontend
npm run build --workspace frontend
npm run build --workspace frontend-admin
rg -n "<html lang=|toLocaleString\\(|toLocaleDateString\\(|aria-label=|placeholder=" frontend frontend-admin
```

### 5.4 P1-1 收斂頁面互動語義

Notifications：

1. 不再用 `div role="button"` 承接卡片主要操作。
2. 卡片 open、CTA、snooze、dismiss 必須是互相獨立的 native control，不得 nested interactive。
3. Enter / Space 行為、focus order、CTA 不誤觸 open 必須有測試。
4. 通知時間格式使用 locale helper。

Chat reply preview：

1. reply preview 關閉 / 跳轉 / 展開等互動必須使用 native button 或 link。
2. 補 role/name、keyboard、focus 回歸測試；若視覺樣式要求非 button 外觀，仍保留 native element。
3. 不得為了保視覺而保留 `div role=button`。

最低命令：

```bash
npm run test:run --workspace frontend
npm run build --workspace frontend
rg -n "role=\"button\"|onKeyDown|tabIndex=\\{0\\}" frontend/src/pages/Notifications frontend/src/pages/Chat frontend/src/components
```

### 5.5 P1-2 補 P0 flow release 證據

允許證據：

1. 真服務 backend + dev DB / release-like DB 的 Playwright artifact。
2. Admin credential 明確配置後的 Admin critical flow artifact。
3. 後端 API health、seed / fixture、env redaction、測試帳號 lifecycle 可追溯的證據包。

不接受：

1. 只用 mock route 的 Playwright pass。
2. credential missing 導致 skip，卻把整體 E2E 標綠。
3. 只有截圖沒有命令、環境、commit、artifact 路徑。

最低命令需按實際 runner 回寫到問題文件；若 runner 尚不存在，先建立 runbook / script，再跑一次 failure-preserving baseline。

### 5.6 P2-1 shared API client 漸進遷移

1. 保留 `frontend/src/services/api/*` 對頁面層 export。
2. 每批只遷移一個 domain 到 `@cj/api-client`。
3. 先遷移低風險、純 request / response domain，再遷移含 session mutation、FormData、SSE、toast、router side effect 的高風險 domain。
4. 每批保留 service tests、Web build、Admin build；若 API shape 不足，先補 shared client tests，再切 Web consumer。

## 9. 多輪驗證與 completion audit

每個階段完成後至少跑四輪：

1. **Round 1 - targeted**：只跑被改 domain 的 unit/component/static scan。
2. **Round 2 - workspace**：跑 Web / Admin build、test、lint 中與階段相關的完整 gate。
3. **Round 3 - evidence**：生成或更新 artifact，回寫問題文件與核心文件，保留命令、commit、日期、環境。
4. **Round 4 - regression scan**：掃描 `role="button"`、`it.skip` / `test.skip`、裸 `toLocaleString()`、placeholder-only field、hardcoded accessible name、direct API wrapper 消費等是否新增未登記缺口。

完成判定：

1. 十個問題各自有對應完成證據或明確未完成狀態。
2. 所有相關核心文件、活躍總覽與台賬已同步。
3. `npm run docs:check` 通過，或只剩已登記且與本輪無關的根層文檔漂移。
4. `git diff --check` 通過。
5. 沒有未登記 skipped tests、未登記 mock-only P0 flow 或未登記 shared package 邊界變更。

### 9.3 Case List skipped tests 補充修復（2026-05-11）

2026-05-11 重新核驗 Web 測試地基時，`frontend/src/utils/animations.test.ts` 已無 skipped tests，但全站 skip scan 仍發現 `frontend/src/pages/Case/List/index.test.tsx` 保留 4 條 `it.skip`。這 4 條覆蓋 F03 案件列表錯誤恢復、狀態篩選、類型篩選與排序重新查詢，不能只由 mock-backed P0 E2E 代替，也不能被「animation skipped tests 已修復」誤讀為全站 skipped test 已清零。

五輪方案分析與裁決：

1. **業務鏈路輪**：Case List 是受保護主站入口，篩選 / 排序失效會直接影響使用者回找案件、重試網路錯誤後恢復列表、以及後續 case detail / judgment / reconciliation 流程入口。因此修復目標是恢復單元 / component 層對 `getCaseList` 參數與錯誤恢復的保護，而不是只依賴高層 E2E。
2. **現有實作輪**：`frontend/src/pages/Case/List/index.tsx` 的 state flow 已在 `statusFilter`、`typeFilter`、`sortBy` 改變後重置 page 並觸發 `fetchCases()`；實測失敗首先來自 Radix Select 在 jsdom 中缺少 pointer capture API，並非產品程式碼的篩選狀態流先壞。
3. **測試環境輪**：Radix Select 使用 pointer events / pointer capture 屬於瀏覽器能力，`frontend/src/test/setup.ts` 已集中補齊 `scrollIntoView`、`ResizeObserver` 等 jsdom 缺口；因此最小且可復用的修復是在同一 setup 補 `hasPointerCapture` / `setPointerCapture` / `releasePointerCapture`，避免每個 Select 測試各自 mock。
4. **測試語義輪**：原 skipped tests 使用 `findByText('caseList.statusDraft')` / `findByText('caseList.typeLife')`，會同時匹配卡片 badge 與 dropdown option。修復後必須用 `findByRole('option', { name })` 定位下拉選項，這與本方案要求的 role/name 驗證一致，也避免測試因畫面其他同文案而誤選。
5. **回歸邊界輪**：`beforeEach` 需使用 `vi.resetAllMocks()` 清掉上一個 case 的 one-off mock queue，避免失敗測試污染下一個測試。產品程式碼、service export、API shape 與 UI 行為不變；修復範圍限於測試環境契約與 Case List 測試查找語義。

本輪落地：

1. `frontend/src/test/setup.ts` 補齊 Element pointer capture jsdom fallback。
2. `frontend/src/pages/Case/List/index.test.tsx` 的 4 條 `it.skip` 復原為正式 `it`。
3. Case List Select option 查找改為 Testing Library `findByRole('option', { name })`。
4. Case List 測試 `beforeEach` 改用 `vi.resetAllMocks()`，防止 mock queue 跨 case 污染。
