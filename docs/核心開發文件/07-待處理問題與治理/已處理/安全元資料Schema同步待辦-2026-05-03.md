## 背景

後端已新增 migration：

- `backend/prisma/migrations/20260503224500_add_safety_metadata_columns/migration.sql`

此 migration additive 新增：

- `cases.safety_metadata JSONB`
- `evidences.safety_metadata JSONB`
- 對早期 `evidences.description` transitional JSON 做 best-effort backfill
- `cases_safety_metadata_idx`
- `evidences_safety_metadata_idx`

後端寫入路徑已改為：

- evidence upload assertion -> `Evidence.safety_metadata`
- formal case assertion -> `Case.safety_metadata`
- formal case assertion + `evidence_urls` -> 同批 `Evidence.safety_metadata`
- 通過 gate 的 evidence / formal case assertion 也會 best-effort 寫入 `SafetyAssessment`；敏感但合法內容映射為資料處理敏感風險，未成年人內容映射為安全支持路由。

## Release 注意事項

- 已提交 migration 不代表 release DB 已套用。
- 發布 Railway backend 前，必須確認 release DB migration 狀態。
- 若 release DB 尚未套用，`safety_metadata` 寫入會在 runtime 失敗；`SafetyAssessment` 寫入採 best-effort 容錯，但不得把 warn 降級視為 release DB parity 已完成。
- 不得用 production DB credentials 做本機開發；若需要查 production migration status，僅允許 read-only status command，且不得輸出密鑰。
