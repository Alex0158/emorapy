## 2026-05-12 收口裁決

3. 將已過時的 migration 註釋改為 legacy / shadcn / Tailwind 中性口徑，不再暗示現行業務組件仍依賴 Ant Design 或 Home.less。

### 五輪方案分析（2026-05-12）

1. **產品 / UX 輪**：不重新設計頁面、不碰視覺 token，只清理遷移尾巴。這能避免 UI regression，同時消除「到底是否仍依賴 Ant Design」的信任成本。
2. **工程依賴輪**：runtime 已沒有 antd；最優解是移除 vite optimize/dedupe 與 `less` devDependency，避免 tooling 將不存在的設計系統當成可用依賴。
3. **測試語義輪**：測試中的 `vi.mock('antd')` 已與現行 shadcn / 原生實作不匹配，保留會讓測試描述失真。刪除 mock 後只保留真實元件和業務 mock。
4. **文檔 / 註釋輪**：歷史遷移記錄可以留在本治理文件；源碼註釋應描述現行狀態，不應繼續說「業務組件仍用 Ant Design」或 `Home.less`。
5. **發版治理輪**：收口標準不能只靠 build；必須同時用依賴樹、源碼掃描、focused tests、前台 build、Admin build 與 docs:check 證明 runtime、tooling、測試和文檔一致。

## 決策記錄

| 項目 | 決策 | 原因 |
|------|------|------|
| 組件庫 | shadcn/ui + Radix UI | 設計上限最高，消除樣式衝突 |
| 色彩 | 暖珊瑚 + 蜜金（從鼠尾草綠轉向） | 更溫暖，契合調解平台氣質 |
| 樣式 | 純 Tailwind CSS 4 | 統一，消除 LESS + Tailwind + Ant Design 三重衝突 |
| 圖標 | Lucide React | shadcn 默認 |
| Toast | Sonner（替代 Ant Design message） | 更輕量，支持 shadcn 生態 |
| 表單 | 原生 + react-hook-form + zod | 取代 Ant Form |
