### 3.1 `POST /api/v1/auth/claim-session`

- SSOT 口徑：
  - `docs/核心開發文件/全接口清單-主文檔.md`
  - `docs/核心開發文件/06-接口描述/01-auth-session.md`
- 前端證據：
  - `frontend/e2e/chat/quick-experience-claim-session.e2e.ts`
  - `frontend/src/store/authStore.test.ts`
  - `frontend/src/services/api/auth.test.ts`
- 後端證據：
  - `backend/tests/integration/quick-experience.flow.test.ts`
  - `backend/tests/integration/smoke.test.ts`
  - `backend/tests/unit/services/auth.service.test.ts`
  - `backend/tests/unit/routes/auth.routes.test.ts`
- 抽檢重點：
  - 登入/註冊成功後可關聯 quick session
  - `INVALID_SESSION_ID` / `SESSION_EXPIRED` 有明確錯誤語義
  - 失敗不阻斷 auth 主流程成功
- 抽檢結論：
  - 已由前端 E2E、store 行為與後端 flow/integration 共同覆蓋，口徑一致，判定通過
