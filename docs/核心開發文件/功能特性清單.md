## 完成判定口徑（本版補強）

- `已完成`：可定位到前端可觸發操作，且有對應 API 調用證據。
- `跨功能依賴`：本功能觸發後跳轉到他功能頁完成 API 調用（需明示責任邊界）。
- `待驗證`：後端接口存在，但前端主流程未檢出明確調用點，暫不宣稱「已完成」。

## B. 功能詳解（操作級）

## F01 快速體驗建案與結果

- **目的**：在未登入狀態下完成建案、補證據、查判決與註冊轉化。
- **前置條件**：可匿名訪問；需可用 `session_id`。
- **核心操作**：
  - 建立/刷新 session。
  - 提交案件陳述與可選證據。
  - 讀取案件與判決結果。
  - 引導註冊並 `claim-session`。
- **關鍵規則**：
  - Session 失效時前端會清理並自動嘗試刷新。
  - 證據上傳可走 `X-Session-Id`，不依賴 JWT。

## F02 協作聽證（同設備雙人）

- **目的**：同設備輪流輸入雙方陳述，降低雙人同步操作門檻。
- **前置條件**：無需登入；建議同場景使用。
- **核心操作**：A 方輸入 -> 交接 -> B 方輸入 -> 提交協作 case -> 導向結果頁。
- **關鍵規則**：
  - `phase` 控制（`intro/role_a/handoff/role_b/submitting`）防止流程跳步。
  - 前端最小字數門檻：A 方 `>=30`、B 方 `>=10`，未達不發請求。
  - 本頁不直接拉判決；提交後跳轉結果頁，由 F01 的結果頁流程執行判決讀取。

## F05 和好方案與執行追蹤

- **目的**：把判決建議轉為可落地行動，並跟蹤進展。
- **前置條件**：已有判決 ID。
- **核心操作**：生成方案、篩選、選擇、確認執行、打卡追蹤（含可選照片）。
- **關鍵規則**：生成方案同樣屬 AI 成本接口（`aiLimiter`）。
- **操作鏈路補充**：打卡頁照片提交為三段鏈路：`GET /reconciliation-plans/:id` 取 `case_id` -> `POST /cases/:id/evidence` 上傳 -> `POST /execution/checkin` 寫入打卡；上傳失敗不阻斷純文字打卡。
