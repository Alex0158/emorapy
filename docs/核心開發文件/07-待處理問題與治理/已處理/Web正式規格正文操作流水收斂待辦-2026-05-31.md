## 文件偏差

目前偏差不是「Web 基線錯誤」，而是「Web 正式基線正文承載粒度過細」。這會讓讀者難以快速分辨：

1. Web / Admin Web 的長期平台邊界。
2. 已完成並歸檔的缺陷修復過程。
3. 單次本地測試或 artifact 是否仍代表當前 release / production gate。
4. App 開發應轉譯的 Web 基線，還是只屬 Web 修復歷史的操作細節。

## 目標狀態

1. `10-Web端/00-Web端凍結基線總覽.md` 保留 Web / Admin Web 已實作能力、平台專屬邊界、對 App 的轉譯規則與目前仍有效 gate。
2. Web 十項修復、shared API client 遷移、A11Y / i18n / P0 true-service 的詳細過程留在 `07-待處理問題與治理/已處理/`、`90-證據與盤點/` 或 `文件收斂/`。

4. 若 Web gate 或 artifact 狀態發生變更，先更新對應治理 / evidence 入口，再回寫正式基線的穩定結論。
