# 數據庫設計（企業級版）

**文檔版本**：v2.2  
**最後更新**：2026-02  
**適用環境**：本地開發 / 預發 / 生產（PostgreSQL 15+）

> **實現對齊**：實際表結構與 ENUM 以 `backend/prisma/schema.prisma` 為準；本文為 DDL 草案與設計對照。

---

## 🎯 設計目標與改進重點
- 覆蓋快速體驗 + 完整模式全流程：案件提交流程、AI 判決、和好方案、執行追蹤、背景/關係檔案、內容推薦、郵件驗證、通知。  
- 嚴格防髒數據：核心字段 ENUM 化，責任比例 CHECK=100，快速模式/完整模式行為用約束明確。  
- 企業級運維：軟刪除、審計、過期清理、部分索引、生成列全文索引預留、冷熱分離預案。  
- 可持續擴展：單庫優先，預留分區、對象存儲遷移、讀寫分離升級路徑。  

---

## 🏗️ 選型與環境
- **DB**：PostgreSQL 15+（Supabase/Railway/RDS 均可）。  
- **本地/CI**：優先 Docker Postgres；需要極簡可用 SQLite，但功能差異（ENUM/GIN）需降級策略。  
- **字符集/排序**：UTF8，`en_US.utf8`；全部時間用 `TIMESTAMPTZ` 存 UTC。  

---

## 📚 基礎約定
- snake_case；主鍵 `id` UUID；時間欄位 `*_at`; 重要表加 `deleted_at`（軟刪）。  
- 審計字段：`created_by` / `updated_by` 可空（快速模式或系統作業）。  
- 文件：只存簽名 URL，保留 `mime_type`、`checksum`；不存裸鏈。  
- PII 加密預留：email/電話/頭像可接 KMS 對稱加密；分析環脫敏。

> 未特別標註皆含 `created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now()`；核心表含 `deleted_at`.

### 1) users
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  nickname VARCHAR(50),
  avatar_url VARCHAR(500),
  avatar_mime VARCHAR(50),
  gender gender_enum,
  age INT CHECK (age BETWEEN 18 AND 99),
  relationship_status relationship_status_enum NOT NULL DEFAULT 'single',
  language language_enum NOT NULL DEFAULT 'zh',
  timezone VARCHAR(50),
  notification_enabled BOOLEAN NOT NULL DEFAULT true,
  privacy_level privacy_enum NOT NULL DEFAULT 'private',
  last_login_at TIMESTAMPTZ,
  is_active BOOLEAN NOT NULL DEFAULT true,
  email_verified BOOLEAN NOT NULL DEFAULT false,
  deleted_at TIMESTAMPTZ,
  created_by UUID,
  updated_by UUID,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_users_active          ON users(is_active)           WHERE deleted_at IS NULL;
CREATE INDEX idx_users_email_verified  ON users(email_verified)      WHERE deleted_at IS NULL;
```

### 3) pairings（配對 / 臨時配對）
```sql
CREATE TABLE pairings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user1_id UUID REFERENCES users(id),
  user2_id UUID REFERENCES users(id),
  invite_code VARCHAR(6) UNIQUE,
  status pairing_status_enum NOT NULL DEFAULT 'pending',
  pairing_type pairing_type_enum NOT NULL DEFAULT 'normal',
  session_id VARCHAR(100),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  confirmed_at TIMESTAMPTZ,
  cancelled_at TIMESTAMPTZ,
  expires_at TIMESTAMPTZ,
  deleted_at TIMESTAMPTZ,
  created_by UUID,
  updated_by UUID,
  CONSTRAINT pairing_user_rule CHECK (
    (pairing_type='quick'  AND session_id IS NOT NULL) OR
    (pairing_type='normal' AND user1_id IS NOT NULL AND user2_id IS NOT NULL AND user1_id <> user2_id)
  )
);
-- 一人同時只允許 1 個 pending/active 正常配對
CREATE UNIQUE INDEX ux_pairings_user1_active ON pairings(user1_id) WHERE pairing_type='normal' AND status IN ('pending','active') AND deleted_at IS NULL;
CREATE UNIQUE INDEX ux_pairings_user2_active ON pairings(user2_id) WHERE pairing_type='normal' AND status IN ('pending','active') AND deleted_at IS NULL;
CREATE UNIQUE INDEX ux_pairings_session      ON pairings(session_id) WHERE pairing_type='quick';
CREATE INDEX idx_pairings_status             ON pairings(status)      WHERE deleted_at IS NULL;
CREATE INDEX idx_pairings_expires            ON pairings(expires_at);
```

### 5) cases（案件）
```sql
CREATE TABLE cases (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  pairing_id UUID NOT NULL REFERENCES pairings(id),
  title VARCHAR(200) NOT NULL,
  type VARCHAR(50) NOT NULL,
  sub_type VARCHAR(50),
  plaintiff_id UUID REFERENCES users(id),
  defendant_id UUID REFERENCES users(id),
  plaintiff_statement TEXT NOT NULL,
  defendant_statement TEXT,
  status case_status_enum NOT NULL DEFAULT 'draft',
  mode case_mode_enum NOT NULL DEFAULT 'remote',
  session_id VARCHAR(100),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  submitted_at TIMESTAMPTZ,
  completed_at TIMESTAMPTZ,
  deleted_at TIMESTAMPTZ,
  created_by UUID,
  updated_by UUID,
  CONSTRAINT case_statement_len CHECK (
    char_length(plaintiff_statement) BETWEEN 30 AND 4000 AND
    (defendant_statement IS NULL OR char_length(defendant_statement) BETWEEN 10 AND 4000)
  ),
  CONSTRAINT case_user_rule CHECK (
    (mode='quick' AND plaintiff_id IS NULL AND defendant_id IS NULL) OR
    (mode <> 'quick' AND plaintiff_id IS NOT NULL AND defendant_id IS NOT NULL)
  )
);
-- 1 個 quick session 僅允許 1 個案件（防重複提交）
CREATE UNIQUE INDEX ux_cases_quick_session ON cases(session_id) WHERE mode='quick' AND deleted_at IS NULL;
CREATE INDEX idx_cases_pairing            ON cases(pairing_id)            WHERE deleted_at IS NULL;
CREATE INDEX idx_cases_user_mode          ON cases(mode, plaintiff_id, created_at DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_cases_user_mode2         ON cases(mode, defendant_id, created_at DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_cases_status_created     ON cases(status, created_at DESC) WHERE deleted_at IS NULL;
```

## 🧭 查詢 / 索引策略
- 列表：案件按 `mode + user + created_at DESC`；配對按 `user + status`；判決按 `created_at`。  
- 部分索引：過濾軟刪 / 快速模式 (`WHERE deleted_at IS NULL`、`mode='quick'`)。  
- 全文搜索預留：為 `cases.plaintiff_statement/defendant_statement/judgment_content` 建生成列 `tsvector` + GIN：  
  ```sql
  ALTER TABLE cases ADD COLUMN search_vector tsvector
    GENERATED ALWAYS AS (to_tsvector('english', coalesce(plaintiff_statement,'') || ' ' || coalesce(defendant_statement,''))) STORED;
  CREATE INDEX idx_cases_search ON cases USING GIN (search_vector);
  ```
- 標籤/推薦：`content_items.tags` GIN；`case_content_links` 可按 relation 分流。

## 🧹 數據生命周期與清理
- `quick_sessions`：24h 失效；案件完成可延長至 7 天；批量清理並連帶 `pairings(pairing_type=quick,temp)`。  
- `pairings`：`pending` 超 24h → `cancelled`；`temp` 超 30 天清理。  
- `email_verifications`：過期即刪；`used=true` 保留 24h。  
- 軟刪表：每月歸檔至冷庫或外部儲存，避免主表膨脹；冷數據不參與 OLTP 索引。  

---

## 🔐 安全與合規
- 最小權限：應用連接角色最小化；只讀/寫分離；禁止 superuser。  
- 文件安全：存 checksum/MIME；上傳後掃毒通過再標記可用。  
- PII/隱私：可對 email/電話/頭像 URL 做應用層加密；BI/日志脫敏。  
- GDPR/刪除請求：軟刪 + 定期物理刪除；`audit_logs` 不存正文，僅存摘要。  

---

## 🚀 性能與可擴展
- 連接池：PGBouncer 或 Prisma 池；設置 `statement_timeout`、`idle_in_transaction_timeout`。  
- 分區預案：`cases`、`judgments` 按月範圍分區；歷史判決全文遷冷表/對象存儲。  
- 緩存：Session/限流/熱點配置放 Redis；判決結果短期緩存減少重取。  

---

## 🧪 業務規則 / 觸發器建議
- 證據數量限制（業務規則：≤3 圖或 1 視頻）：使用觸發器在 `evidences` insert 前檢查 `COUNT(*)`。  
- 快速模式防重：`ux_cases_quick_session` 已限制同 Session 只建 1 案件。  
- 配對唯一：`ux_pairings_user*_active` 防止同時多條有效配對。  
- 判決責任：`judgment_ratio_100` 確保比例和為 100。  

---

## 🔄 遷移與環境（簡述）
- 開發：`npx prisma migrate dev`；預發/生產：`prisma migrate deploy`。  
- ENUM 變更：先 `ALTER TYPE ... ADD VALUE` 後調整欄位，避免重建。  
- 加約束前先清洗髒數據，避免 deploy 失敗；使用 `NOT VALID` + `VALIDATE CONSTRAINT` 漸進上線。  

---

## ✅ 覆蓋性檢查（對應產品功能）
- 快速體驗：`quick_sessions + pairings(pairing_type=quick) + cases(mode=quick) + ux_cases_quick_session`。  
- 完整模式配對/解綁：邀請碼唯一、每人同時僅一組有效配對、過期自動失效。  
- 背景/關係檔案：`user_profiles`、`relationship_profiles`。  
- 判決/責任比例：`judgments` + 100% 約束；接受度/評分字段支持質量監控。  
- 和好方案與執行：`reconciliation_plans`、`execution_records`。  
- 內容推薦/等待頁：`content_items` + `case_content_links`。  
- 郵件驗證與通知：`email_verifications`、`notifications`（含 dedup、防重發）。  
- 審計與合規：`audit_logs`、`deleted_at`、生命周期清理策略。
