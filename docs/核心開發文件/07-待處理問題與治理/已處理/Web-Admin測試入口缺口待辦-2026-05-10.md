## 1. 問題

`frontend-admin` 目前沒有 workspace 級測試 script。Admin E2E 存在於根級 `e2e/admin/`，但多數測試依賴 `E2E_ADMIN_EMAIL / E2E_ADMIN_PASSWORD` 或低權限帳號環境變數，未配置時會 skip。這使 Admin Web 的「已落地」更多依賴 build 與有條件 E2E，而缺少穩定可執行的單元 / component / route gate。

## 2. 證據

`frontend-admin/package.json` scripts 只有：

```json
{
  "dev": "vite",
  "build": "tsc -b && vite build",
  "lint": "biome lint src",
  "preview": "vite preview"
}
```

沒有 `test` / `test:run` / `test:coverage`。

Admin E2E 例子：

```ts
test.skip(!adminCreds, '需要設定 E2E_ADMIN_EMAIL / E2E_ADMIN_PASSWORD');
```

出現在 `e2e/admin/admin-critical-flow.e2e.ts`、`admin-config-flow.e2e.ts`、`admin-jobs-flow.e2e.ts`、`admin-reports-flow.e2e.ts` 等。

同輪命令：

```bash
npm run build --workspace frontend-admin
```

結果：通過。

```bash
npm run lint --workspace frontend-admin
```

結果：失敗，見 [Web-AdminLint與Tailwind4Biome配置阻塞待辦-2026-05-10.md](../已處理/Web-AdminLint與Tailwind4Biome配置阻塞待辦-2026-05-10.md)。

## 3. 核心文件依據

1. `10-Web端/00-Web端凍結基線總覽.md` 將 Admin Web 定位為管理與平台治理入口。
2. `08-測試規範與驗收/README.md` 將 `frontend-admin/src`、`e2e/**/*.ts` 列為長期取證入口。
3. `08-測試規範與驗收/05-可訪問性本地化驗收基線.md` 要求 Admin Web 表格、篩選、配置、token、danger action、audit log、報表具備可理解性與鍵盤操作證據。

## 4. 風險

1. Admin 設定、報表、權限、危險操作缺少可在本地穩定重跑的 component / route test。
2. E2E 在未配置帳號時 skip，容易被誤讀為 Admin flow 已完整驗收。
3. Admin lint 目前失敗，導致 build 通過但質量 gate 不閉環。

## 5. 目標狀態

1. `frontend-admin` 補上可本地穩定執行的 `test:run` 或等價測試入口。
2. Admin 主要頁面至少覆蓋 login、permission denied、configs、users、audit logs、reports、settings、jobs 的核心互動分支。
3. E2E skip 條件在 release / smoke 文檔中清楚標示，不能把 skip 當 pass。
4. Admin Web 的 build / lint / test / E2E 證據可以在活躍問題或證據入口中追溯。
