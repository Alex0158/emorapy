# 開發環境配置（代碼對齊版）

**文檔版本**：v3.1  
**最後更新**：2026-03-05  
**對齊基準**：`backend/package.json`、`frontend/package.json`、`.env.example`、`scripts/*.sh`

---

## 1. 工具鏈

- Node.js：`>=20.19.0`
- npm：建議 10+
- DB：PostgreSQL（開發）

---

## 2. 後端環境

路徑：`backend`

### 2.1 安裝與配置

```bash
cd backend
npm install
cp .env.example .env
npm run prisma:generate
npm run prisma:migrate
```

### 2.2 常用腳本

- `npm run dev`：`tsx watch src/index.ts`
- `npm run build`：`tsc`
- `npm run build:prod`：`tsc --project tsconfig.prod.json`
- `npm start`：啟動 `dist/index.js`
- `npm run lint`：Biome
- `npm run test:unit` / `test:integration` / `test:integration:flow`
- `npm run prisma:generate` / `prisma:migrate`

### 2.3 重要注意

- `postinstall` 會觸發 `npm run build`（CI / 新機初始化需注意時間）
- 訪談與聊天室能力依賴 OpenAI key，若需純本地可開 `AI_MOCK=true`

---

## 3. 前端環境

路徑：`frontend`

### 3.1 安裝與配置

```bash
cd frontend
npm install
cp .env.example .env
```

### 3.2 常用腳本

- `npm run dev`：Vite
- `npm run build`：`tsc -b && vite build`
- `npm run lint`：Biome
- `npm run test` / `test:run`：Vitest run
- `npm run test:coverage`
- `npm run test:e2e`：Playwright

### 3.3 重要注意

- 必需 `VITE_API_BASE_URL`
- `/admin/*` 路徑由 `VITE_ADMIN_LOGIN_URL` 轉跳，不是本前台內嵌 admin

---

## 4. 本地啟動順序

1. 後端先起：`npm run dev`
2. 前端再起：`npm run dev`
3. 驗證：
   - `GET /health`
   - 快速體驗能建 session/case
   - 訪談頁能進入（已 consent）

---

## 5. 一鍵與驗證腳本

- `./scripts/start-dev.sh`：同時拉起前後端
- `./scripts/verify-integration.sh`：基礎連通驗證
- `./scripts/ci-local.sh`：本地 CI 回歸

---

## 6. 常見故障排查

### 6.1 後端啟動失敗

- 缺必填 env：`DATABASE_URL/JWT_SECRET/OPENAI_API_KEY`
- Prisma client 未生成或 migration 未跑

### 6.2 前端 API 404/跨域

- `VITE_API_BASE_URL` 配置錯誤
- 後端 `ALLOWED_ORIGINS` 未包含前端地址

### 6.3 訪談/聊天異常

- 檢查 OpenAI 配置與限流
- 檢查後端 log 與前端網路事件流（SSE）
