依「未登入直連優化驗證計畫」§3，以下情境需手動在瀏覽器執行一輪，確認正常流程未被破壞。

| 情境 | 預期 | 驗證方式 |
|------|------|----------|
| 已登入用戶直連 `/case/list` | 顯示案件列表，不跳登錄 | 登入後訪問 `/case/list` |
| 已登入 admin 直連 `/admin/ops/jobs` | 顯示運維頁，不跳登錄 | 設定 token 後訪問 |
| 未登入直連 `/case/list` | 先 loading 或短暫停留，再跳 `/auth/login` | 清 storage 後直連 |
| 未登入直連 `/admin/ops/jobs` | 直接跳 `/admin/login`，不顯示側欄/Alert | 無 token 直連 |
| 登入後回跳 | 從 `/auth/login` 登入後回到直連前的保護頁 | 直連 `/profile/settings` → 登入 → 應到 `/profile/settings` |

**說明**：自動化部分（ProtectedRoute 單元測試）已通過；本清單供發版前或變更後手動回歸使用。
