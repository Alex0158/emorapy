## 4. F02 對應覆蓋點

本輪真 DB 驗證對應的 F02 核心案例如下。

### 4.1 協作聽證主鏈路

- `角色 A 建立 collaborative draft case`
- `角色 B 補完 defendant_statement`
- `getCase(caseId)` 可成功讀回 collaborative case
- `pollJudgment(caseId)` 可在異步生成後讀到 judgment

對應測試：

- `backend/tests/integration/quick-experience.flow.test.ts:431`

驗證重點：

- `phase: a_done -> submitted`
- `case.mode === 'collaborative'`
- `case.status: draft -> submitted`
- `case detail` 可讀
- `judgment summary` 可讀

### 4.2 錯誤 session 應拒絕角色 B 提交

對應測試：

- `backend/tests/integration/quick-experience.flow.test.ts:470`

驗證重點：

- 改用錯誤 `session_id` 後，`POST /api/v1/cases/collaborative` 返回 `403`
- 錯誤碼為 `FORBIDDEN`

### 4.3 重複提交應返回 CASE_NOT_EDITABLE

對應測試：

- `backend/tests/integration/quick-experience.flow.test.ts:491`

驗證重點：

- 角色 B 首次提交成功
- 第二次重複提交返回 `422`
- 錯誤碼為 `CASE_NOT_EDITABLE`
