# 平台控制台環境變數清單（2026-03-05）

> [!WARNING]
> **ARCHIVED（歷史報告）**：本文件為當時點的執行快照，不作為現行部署/配置唯一依據。  
> 請以最新 CI workflow、現行運維文件與實際環境設定為準。

本文件是「可直接貼到平台 Dashboard」版本，對應你們目前建議的正式拓撲：

- Backend：Railway
- Frontend（一般使用者）：Vercel（`frontend`）
- Frontend-Admin（管理台）：Vercel（`frontend-admin`）

---

## 1) Railway（Backend）— 先填這組

## 1.1 生產必填（缺一可能無法啟動）

```env
NODE_ENV=production
PORT=3001
DATABASE_URL=postgresql://<user>:<pass>@<host>:<port>/<db>
JWT_SECRET=<至少32字元強隨機字串>
JWT_EXPIRES_IN=7d
ADMIN_JWT_SECRET=<至少32字元強隨機字串且不可等於JWT_SECRET>
ADMIN_JWT_EXPIRES_IN=12h
OPENAI_API_KEY=sk-<your-key>
FRONTEND_URL=https://app.<your-domain>
ALLOWED_ORIGINS=https://app.<your-domain>,https://admin.<your-domain>
```
若使用 Vercel 預設網域（如客戶端 `https://mother-bear-court.vercel.app`），須將該網址列入 `ALLOWED_ORIGINS`，否則 Quick Experience 等會 CORS 403。

## 1.2 強烈建議（正式環境建議一起開）

```env
FILE_BASE_URL=https://api.<your-domain>
REDIS_URL=redis://<host>:<port>
ENABLE_SCHEDULED_JOBS=true
METRICS_ENABLED=true
METRICS_TOKEN=<強隨機token>
OPENAI_MODEL=gpt-4o-mini
OPENAI_MAX_TOKENS=2000
OPENAI_DAILY_LIMIT=1000
```

## 1.3 運維可選（有需要再開）

```env
METRICS_ALLOWED_IPS=10.0.0.10,10.0.0.11
ALERT_HEALTH_ORIGIN=https://app.<your-domain>
ALERT_SLACK_WEBHOOK_URL=https://hooks.slack.com/services/xxx/yyy/zzz
ALERT_SLACK_DEDUP_WINDOW_SECONDS=600
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=<mail-user>
SMTP_PASS=<mail-pass>
```

## 4) 平台設定（不是 env，但要一起設定）

## 4.1 Railway（Backend）

- Build Command：`cd backend && npm ci && npm run build:prod`
- Start Command：`cd backend && npm start`

> 目前 `railway.json` 與 `backend/railway.toml` build 命令口徑不同，建議統一採 `build:prod`。

## 4.2 Vercel（前台）

- 建議建立獨立專案，Root Directory 設 `frontend`
- Build Command：`npm run build`
- Output Directory：`dist`

## 4.3 Vercel（管理台）

- 建議建立獨立專案，Root Directory 設 `frontend-admin`
- Build Command：`npm run build`
- Output Directory：`dist`

> 現在 repo 根目錄 `vercel.json` 輸出是 `frontend-admin/dist`，若只建單一 root 專案會偏向部署管理台，不是主前台。

## 5) 一次生成安全密鑰（本機執行）

```bash
openssl rand -base64 48
```

產生兩次，分別填：

- `JWT_SECRET`
- `ADMIN_JWT_SECRET`

---

## 6) 填完後的上線核對（最小必做）

1. `GET https://api.<your-domain>/health`
2. `GET https://api.<your-domain>/health/ready`
3. `GET https://api.<your-domain>/health/live`
4. 前台建立 quick session / quick case / 輪詢判決
5. 管理台登入與權限頁可正常打 API
6. 若開 metrics：帶 `X-Metrics-Token` 驗證 `GET /metrics`

---

## 7) 常見填錯點（請避開）

- `ALLOWED_ORIGINS` 漏掉 `admin` 網域，會導致管理台 CORS 被擋。
- `VITE_ADMIN_LOGIN_URL` 用相對路徑（例如 `/admin/login`）會造成跳轉異常，請用完整絕對 URL。
- `ADMIN_JWT_SECRET` 與 `JWT_SECRET` 設成相同，production 會被拒絕。
- `METRICS_ENABLED=true` 卻沒設 `METRICS_TOKEN`/`METRICS_ALLOWED_IPS`，production 會被拒絕。
