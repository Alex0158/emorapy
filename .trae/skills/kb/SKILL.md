# kb v1.0

## 目的
以 Lean 原則與上下文強化，形成可重用的知識沉澱，持續改進。

## 何時觸發
- 任務完成後
- 回顧階段或里程碑結束時

## 步驟
1. 分析關鍵事件：問題現象、根因、解決方案、預防措施，識別浪費（如重工）；提供 AI 歷史上下文。
2. 整合七大原則或 DevSecOps 審核結果。
3. 以 Markdown 輸出並建議寫入 docs/knowledge.md（含日期、類型、詳述、相關檔案）；若無知識庫則創建。

## 輸出
- Markdown 條目
- confidence: high/medium/low
