## 角色與身份

| 術語 | 定義 |
|---|---|
| 訪客（Guest） | 未登入前台用戶，可進快速體驗與部分 chat 弱入口；若無有效 session，不能穩定持有匿名房間能力。 |
| 用戶（User） | 持有 JWT 的前台登入用戶。 |
| 管理員（Admin） | 持有 admin token 的後台運維用戶，走獨立 RBAC。 |
| Session 用戶 | 無 JWT，但持有 `session_id`（`X-Session-Id`）的匿名會話身份；可承接 quick flow 與部分 chat 能力。 |

## 路由/頁面術語

| 術語 | 定義 |
|---|---|
| ProtectedRoute | 需登入路由守衛，未登入跳 `/auth/login`；直連保護頁時未登入會強制跳轉登錄。 |
| PublicRoute | 公開認證頁守衛，已登入跳 `/case/list`。 |
| 管理端登入守衛 | frontend-admin 中無 admin token 訪問 `/admin/*`（除登錄頁）時重定向至 `/admin/login`（現由 AdminSectionLayout 內檢查 token 並重定向）。 |
| 弱入口 | 可直接 URL 進入但不經強守衛的頁面（如 `/chat/room/:roomId`）。 |
| 候選保留路由 | 主要僅做 redirect 或兼容用途的路由（如 `/auth`）。 |
| 外部 Admin 子頁 | 由 `frontend-admin` 承接的 admin 實際頁面，如 `/admin/health`、`/admin/reports`。 |

## 接口治理術語

| 術語 | 定義 |
|---|---|
| 已使用 | 後端接口存在，且已有前端、App adapter、release evidence runner 或穩定運維 / 系統調用證據。 |
| 待承接 | 後端接口存在且仍是目標能力，但當前缺少對應端側頁面 / API client / release runner / 穩定運維調用證據。 |
| 候選廢棄 | 後端接口存在但前端主流程未接線、無穩定運維 / 系統調用證據，且未被當前正式目標明確保留。 |
| 已確認廢棄 | 明確不再使用且已完成替代與下線計劃。 |
| 候選保留 | 在功能文檔中表示「保留既有編碼做歷史索引，但不計入主功能序列」的候選能力狀態。 |
| 平台保留 | 在功能文檔中表示「監控/平台能力保留於附錄，不視為前台主功能」的狀態。 |
| 主註冊表 | `全接口清單-主文檔.md`，接口狀態唯一權威來源。 |
| Error Envelope / 錯誤 envelope | CJ 當前 API 錯誤回應格式：`success=false`、`error.code/message/details?`、`meta.request_id/timestamp`；不等於 RFC 9457 Problem Details。 |
| Problem Details | RFC 9457 定義的 HTTP API 錯誤格式，核心欄位包括 `type/title/status/detail/instance`；CJ 目前只作缺口對標，尚未導入 `application/problem+json`。 |
| CJ-ERR-* | API 錯誤需求 ID，用於追蹤錯誤碼、HTTP status、request id、safe details、retry policy、Problem Details / OpenAPI error schema 缺口。 |
| Retryable Error / 可重試錯誤 | 可由前端、App、後端 worker 或人工流程安全重試的失敗；需在接口或錯誤模型中標明 retry 條件、冷卻、冪等與避免重複 side effect 的 guard。 |

## 流程與狀態術語

| 術語 | 定義 |
|---|---|
| Quick Case | 快速體驗建案，通常由 session 驅動。 |
| Collaborative Case | 同設備雙人輪流輸入的協作聽證案件。 |
| Interview Session | 心理訪談會話，具輪次與 pipeline 狀態。 |
| Chat -> Judgment | 聊天室語料轉換為 case + judgment 的鏈路。 |
| claim-session / 升格 | quick case 在登入/註冊成功後，經 `POST /auth/claim-session` 綁定到正式帳號的後置流程。 |
| canonical session_id | 前端統一 session 管理層持有的當前有效匿名會話 ID；用於 session 身份判定，不以任意 storage 臨時值作準。 |
| 冪等（Idempotent） | 重複請求只產生一次有效結果（例如 chat request-judgment）。 |
| State Machine / 狀態機 | 描述一個 domain object 可處於哪些狀態、由哪些事件轉移、需滿足哪些 guard、產生哪些 side effect、哪些轉移非法以及如何恢復的規格；schema enum 只是狀態來源之一。 |
| Business Invariant / 業務不變式 | 跨狀態和跨流程都不得被破壞的業務約束，例如 quick case 一律 session-bound、formal case 授權必須由 backend 裁決、chat judgment 需要 role / consent guard。 |
| Illegal Transition / 非法轉移 | 狀態機明確不允許的 from/to 或 trigger/guard 組合；應回穩定錯誤碼，不得靠前端按鈕狀態替代後端保護。 |
| Recovery Task / 人工恢復任務 | 當產品狀態卡住、失敗或自動修復不安全時，交由 Admin 或運維人工 review / resolve / dismiss 的治理任務。 |
| 梳理結果（Analysis Result） | AI 與當事人共同梳理的親密衝突分析報告，取代舊「判決」用語。強調協力傾聽而非仲裁。代碼/API 仍用 `judgment` 作為內部標識。 |
| 發起方 / 回應方 | 案件建立者與被邀請方，取代舊「原告/被告」用語，避免對抗性框架。代碼/DB 仍用 `plaintiff`/`defendant` 作為欄位名。 |
| 調整方向 | 取代舊「責任比例」用語。以百分比呈現雙方各自的成長空間，而非分配「對錯」。 |

## 核心狀態值

| 類別 | 狀態值 | 說明 |
|---|---|---|
| Case | `draft` / `submitted` / `in_progress` / `judgment_failed` / `completed` / `cancelled` | 案件生命週期 |
| Interview | `in_progress` / `processing` / `completed` / `processing_failed` / `abandoned` | 訪談主狀態 |
| ChatRoom | `solo_active` / `invite_pending` / `invite_accepted` / `group_active` / `judgment_requested` / `judgment_completed` / `judgment_failed` / `archived` | 聊天房狀態 |
| Pairing | `pending` / `active` / `cancelled` / `temp` | 配對狀態 |
| Judgment Acceptance | `accepted` / `rejected` | 梳理結果接受狀態；一旦成立，不應再次重選。（API 欄位名保持 judgment 向後兼容） |

## 運維術語

| 術語 | 定義 |
|---|---|
| Health 探針 | `/health`、`/health/ready`、`/health/live`。 |
| Metrics 探針 | `/metrics`，Prometheus 格式輸出。 |
| RBAC Permission | admin 權限字串，如 `ops:read`、`config:write`。 |
| SLI | Service Level Indicator；服務健康的量測指標，如 readiness、5xx ratio、AI stream terminal result。 |
| SLO | Service Level Objective；以 SLI 表達的目標值或目標範圍。CJ 目前多數 SLO 仍為待建立基線，不對外承諾 SLA。 |
| Error Budget | SLO 允許的錯誤空間；CJ 尚未建立正式 error budget，不得把 alert threshold 當 error budget。 |
| Burn Rate | error budget 被消耗的相對速度；CJ 尚未建立 burn-rate alerting，不得把 15m 5xx ratio 直接稱為 burn rate。 |
| Telemetry Signal | 可觀測性信號；通常包含 metrics、logs、traces、baggage。CJ 目前有 metrics、logs、request id、App safe telemetry event、App OTLP JSON span summary 與 release evidence；仍不得外推為完整 analytics 或 distributed tracing。 |
| OpenTelemetry / OTel | 可觀測性框架與信號模型；CJ 目前已有 App-side basic provider 與 backend OTLP JSON ingest baseline，但尚未接入 vendor collector、跨服務 distributed tracing 或長期 tracing SLO。 |
| Trace | 一次請求或工作流跨元件的路徑記錄；CJ 目前只接收 App OTLP JSON subset 並最小化保存摘要，不得用 request id 或 App span summary 代替完整 distributed trace 完成。 |
| Span | Trace 中的單段操作，例如 HTTP handler、DB query、AI provider call；CJ 目前可接收 App-side span summary，尚未建立全鏈路 backend/service span baseline。 |
| Alertmanager | Prometheus 生態中負責告警去重、抑制、聚合與通知的元件；CJ 目前不得因有 `/metrics` 或 rule 文件就宣稱 Alertmanager 閉環。 |
| Incident / 事故 | 造成核心流程、資料安全、隱私、可用性、AI 安全或發布閉環異常的事件；分級口徑見 `03-管理端與平台治理/06-SLO可觀測性與事故治理基線.md`。 |
| Incident Drill / 事故演練 | 用真事故、release gate failure 或故障注入按模板走完偵測、分級、緩解、恢復與 follow-up 的驗收活動；口徑見 `08-測試規範與驗收/07-SLO可觀測性與事故演練驗收基線.md`。 |
| SEV | 事故嚴重度分級；CJ 初始使用 SEV0-SEV3。 |
| Postmortem | 事故後復盤記錄，需包含時間線、影響、根因、緩解、恢復證據與防復發任務。 |

| WCAG | Web Content Accessibility Guidelines；CJ 以 WCAG 2.2 作 Web 可訪問性參考，不代表已取得 WCAG conformance。 |
| WAI-ARIA / APG | Web Accessibility Initiative - Accessible Rich Internet Applications 與 Authoring Practices Guide；用於複雜互動元件的 role、name、state、keyboard pattern。 |
| Accessible Name | 輔助技術用來識別控制項的可讀名稱；icon-only button、stream control、upload、toast / error action 不能只靠視覺圖示。 |
| HTML `lang` | Web 根節點或文本片段的語言標記；需與 default / selected locale 一致或有動態更新策略，否則會影響 screen reader、翻譯與語音行為。 |
| Keyboard-only Flow | 不使用滑鼠即可完成主流程的操作口徑；需覆蓋 Tab order、Enter/Space、Esc、focus return 與 no keyboard trap。 |
| Screen Reader Smoke | 以 VoiceOver、TalkBack、NVDA 或等價輔助技術做的最小可理解性檢查；不能由 Testing Library role tests 自動替代。 |
| Locale / 本地化 | 用戶界面語言、日期、時間、數字、百分比、語音朗讀與錯誤提示的區域化策略；CJ 當前主要為 `zh-TW` / `en-US`。 |
| i18n Key Completeness | 檢查各 locale catalog 是否包含同一組可見文案 key，並禁止長期依賴 key fallback 或 humanized fallback 作正式文案。 |
| Content Design / 內容設計 | 對高敏關係衝突、心理推斷、安全風險、AI failure、刪除/保留與錯誤文案進行可理解、非責備、非過度承諾的設計與審查。 |
| CJ-A11Y-* / CJ-L10N-* / CJ-CONTENT-* | 可訪問性、本地化與內容設計需求 ID；入口為 `04-共用機制/07-可訪問性本地化與內容設計治理基線.md` 與 `08-測試規範與驗收/05-可訪問性本地化驗收基線.md`。 |
| Data Dictionary / 資料字典 | 對資料 model / field 的語義、owner、敏感度、來源、相容性狀態、保留/刪除與驗證方式的登記；Prisma 欄位名不等於完整資料字典。 |
| Prisma Schema | `backend/prisma/schema.prisma`；CJ DB model、enum、relation 與 index 的主要開發事實來源，但不等於 release DB 已套用狀態。 |
| Migration History | `backend/prisma/migrations` 與目標 DB `_prisma_migrations` 共同構成 migration history 證據；migration 檔案存在不代表特定 DB 已套用。 |
| Release DB Parity | release / production-like DB 的 `_prisma_migrations` 與 release-blocking migration 清單一致，且沒有 missing、failed 或 rolled back migration。 |
| Expand / Contract Migration | 先新增兼容結構，再 backfill / dual-read 或 dual-write，最後移除舊結構的分階段資料遷移策略；用於 rename、drop、type change、required field、enum contraction 等破壞性變更。 |
| Backfill | 將既有資料補齊到新 schema 或新欄位的過程；需有來源、範圍、冪等、count evidence 與失敗處置。 |
| Schema Drift | 目標 DB schema 與 migration history / Prisma schema 預期不一致；manual hotfix 必須回補 migration history 或 reconciliation 記錄。 |
| Contract Compatibility | DB schema、API response、shared DTO、enum、App storage 對舊 Web/Admin/API client/App 的 source、wire、semantic 相容性。 |
| Deprecated Field / 棄用欄位 | 已有替代來源但仍保留以維持相容的欄位；刪除前需有棄用原因、讀寫策略、保留窗口與移除條件。 |
| CJ-SCHEMA-* | 資料模型、schema migration、release DB parity、backfill、棄用與 compatibility 需求 ID；入口為 `05-工程架構與共享層/03-資料模型SchemaMigration與相容性治理基線.md` 與 `08-測試規範與驗收/06-SchemaMigration與相容性驗收基線.md`。 |
| AI 風險治理基線 | `04-共用機制/03-AI風險與安全治理基線.md`；承接 AI / LLM asset inventory、prompt injection、防過度依賴、ledger、prompt version、output gate 與 App AI 待承接口徑。 |
| AI Asset Inventory | `CJ-AI-ASSET-*`；用於標記每個 AI runtime 的平台層級、主要入口、風險級別、證據與狀態，不代表 AI 效果已量化。 |
| AI Risk ID | `CJ-AI-RISK-*`；用於追蹤 prompt injection、敏感資訊洩露、過度依賴、output downstream gate、安全誤路由、stream recovery、成本濫用、prompt version drift 與 App lifecycle drift。 |
