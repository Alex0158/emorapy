## 3. 代碼事實（高層）

### 前端

- 技術：React + TypeScript + Vite + Zustand + React Router
- 路由集中於：`frontend/src/router/index.tsx`
- 訪談/聊天等頁面採 lazy loading

### 後端

- 技術：Express + TypeScript + Prisma + PostgreSQL
- 路由入口：`backend/src/app.ts`
- 路由實作：`backend/src/routes/*.ts`

### AI 與事件流

- 訪談與分析模型可分離配置（`OPENAI_INTERVIEW_MODEL` / `OPENAI_ANALYSIS_MODEL`）
- 訪談回答採 SSE（`/interview/:id/respond`、`/interview/:id/skip`）

## 4. 文檔治理原則

1. **代碼優先**：文檔若與代碼衝突，以代碼為準  
2. **先核對再更新**：每次更新需先定位相關 route/service/store  
3. **主文件優先同步**：
   - `功能特性清單`
   - `前端路由與接口一覽`
   - `後端 API 設計`
4. **避免重複規格**：重複文檔保留索引與鏈接，不各自演化
