### 6.1 五輪方案分析與裁決（2026-05-11）

1. **業務 / 信任鏈路輪**：P0 true-service smoke 會覆蓋 quick、formal case、evidence upload、judgment、chat analysis 與 DB ownership，這些都是使用者核心信任鏈路。不能用 mock-backed Playwright 代替，也不能在未確認 target 的情況下污染遠端資料。
2. **工程安全輪**：runner 會建立測試使用者、case、evidence、chat room 與 judgment，且目前 `DATABASE_URL` 指向遠端 Supabase Postgres host。最優方案是保留強制 `RUN_WEB_P0_TRUE_SERVICE_SMOKE=true` guard，要求 `WEB_P0_SMOKE_REPORT_PATH`，並對非本機 DB 增加 `WEB_P0_SMOKE_ALLOW_REMOTE_DB=true` 二次確認；先做 preflight，再由明確 target owner 確認可寫入。
3. **成本 / AI 輪**：若只是 no-cost release-like smoke，應使用 `AI_MOCK=true`，但仍必須是真 HTTP、真 backend、target DB。若要付費 AI 驗證，需另行明確成本與模型設定，不混入本輪自動默認。
4. **Admin 治理輪**：Admin E2E 缺 `E2E_ADMIN_EMAIL` / `E2E_ADMIN_PASSWORD` 時只能算 blocker，不能用 frontend-admin Vitest / route axe smoke 代替 credential-backed governance flow。
5. **證據治理輪**：preflight artifact 可證明 blocker 真存在，但不是 pass artifact。完成出口仍必須是 `Web-P0-True-Service-*.json` pass artifact 加 Admin credential pass evidence。

`web:p0:true-service:smoke` 是 mutating runner，會建立測試使用者、case、evidence、chat room 與 judgment，必須顯式設定 `RUN_WEB_P0_TRUE_SERVICE_SMOKE=true` 才能執行，也必須提供 `WEB_P0_SMOKE_REPORT_PATH` 留存 artifact。`AI_MOCK=true` 可作為 no-cost release-like smoke，但仍必須使用真 HTTP、真 backend 與 target `DATABASE_URL`。若 `DATABASE_URL` 非本機 host，必須額外設定 `WEB_P0_SMOKE_ALLOW_REMOTE_DB=true`，表示 target owner 已確認可寫入測試資料；mock-backed Playwright、raw console output、preflight blocker 或缺 artifact 的失敗輸出不能替代。

Admin credential-backed E2E 必須提供 `E2E_ADMIN_EMAIL` / `E2E_ADMIN_PASSWORD`，permission denied flow 還必須提供 `E2E_LIMITED_ADMIN_EMAIL` / `E2E_LIMITED_ADMIN_PASSWORD`。`E2E_STRICT=true` 用於防止 credential 缺失時 silent skip；正式證據必須是 JSON reporter artifact，且 `stats.skipped=0`、`stats.unexpected=0`。
