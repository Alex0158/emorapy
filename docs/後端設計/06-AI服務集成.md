## 🤖 AI服務概述

### 技術選型

- **服務提供商**：OpenAI
- **三套模型配置**（詳見下方 § 心理畫像 AI 模型分配策略表）：
  - **AI_CONFIG**（gpt-3.5-turbo）：判決、和好方案、案件摘要——成本低、效果好
  - **INTERVIEW_AI_CONFIG**（gpt-4o-mini）：v2.0 訪談對話、域分類——快速回應、低成本
  - **ANALYSIS_AI_CONFIG**（gpt-4o）：v2.0 敘事摘要、洞察提取、反饋卡片——高質量分析

### 使用場景

1. **案件類型自動識別**：分析雙方陳述，自動判斷案件類型（AI_CONFIG）
2. **判決書生成**：生成溫暖、公正的判決書（AI_CONFIG + v2.0 心理畫像注入）
3. **和好方案生成**：生成個性化的和好方案（AI_CONFIG）
4. **文本摘要**：生成判決書摘要（AI_CONFIG）
5. **v2.0 訪談對話**：AI 引導式心理訪談 + 實時追問（INTERVIEW_AI_CONFIG）
6. **v2.0 域分類**：將訪談內容分類至 8 大心理域（INTERVIEW_AI_CONFIG）
7. **v2.0 異步分析管線**：敘事摘要 → 洞察提取 → 反饋卡片（ANALYSIS_AI_CONFIG）

### 接口定義

```typescript
interface AIService {
  // 生成文本（通用）
  generateText(prompt: string, options?: GenerateOptions): Promise<string>;
  
  // 判決書生成（AI_CONFIG）
  generateJudgment(prompt: string): Promise<JudgmentResponse>;
  
  // 和好方案生成（AI_CONFIG）
  generateReconciliationPlans(prompt: string): Promise<ReconciliationPlan[]>;
  
  // 案件類型自動識別（AI_CONFIG）
  detectCaseType(plaintiffStatement: string, defendantStatement: string): Promise<string>;
  
  // 判決書摘要（AI_CONFIG）
  generateSummary(content: string): Promise<string>;

  // v2.0: 通用聊天（訪談/域分類/分析管線均通過此方法 + 不同 config 調用）
  chat(messages: ChatMessage[], config: AIConfigType): Promise<string>;
}
```

### 實現

```typescript
import { openai, AI_CONFIG } from '../config/openai';
import { AppError } from '../utils/errors';
import logger from '../config/logger';

export class AIService implements AIService {
  private dailyCallCount = 0;
  private dailyLimit = parseInt(process.env.OPENAI_DAILY_LIMIT || '1000');
  
  // 生成文本（通用方法）
  async generateText(
    prompt: string,
    options: GenerateOptions = {}
  ): Promise<string> {
    try {
      // 檢查每日限額
      this.checkDailyLimit();
      
      const response = await openai.chat.completions.create({
        model: options.model || AI_CONFIG.model,
        messages: [
          {
            role: 'system',
            content: options.systemPrompt || '你是一個有用的助手。'
          },
          {
            role: 'user',
            content: prompt
          }
        ],
        max_tokens: options.maxTokens || AI_CONFIG.maxTokens,
        temperature: options.temperature ?? AI_CONFIG.temperature,
        top_p: options.topP ?? AI_CONFIG.topP,
        frequency_penalty: options.frequencyPenalty ?? AI_CONFIG.frequencyPenalty,
        presence_penalty: options.presencePenalty ?? AI_CONFIG.presencePenalty
      });
      
      // 更新調用計數
      this.dailyCallCount++;
      
      const content = response.choices[0]?.message?.content;
      if (!content) {
        throw new Error('AI返回空內容');
      }
      
      return content;
    } catch (error: any) {
      logger.error('OpenAI API error', { error: error.message, prompt: prompt.substring(0, 100) });
      
      if (error.status === 429) {
        throw new AppError(503, 'AI_SERVICE_RATE_LIMIT', 'AI服務請求過於頻繁，請稍後再試');
      } else if (error.status === 401) {
        throw new AppError(500, 'AI_SERVICE_AUTH_ERROR', 'AI服務認證失敗');
      } else {
        throw new AppError(503, 'AI_SERVICE_ERROR', 'AI服務暫時不可用');
      }
    }
  }
  
  // 檢查每日限額
  private checkDailyLimit(): void {
    if (this.dailyCallCount >= this.dailyLimit) {
      throw new AppError(503, 'AI_SERVICE_LIMIT_EXCEEDED', '今日AI服務調用已達上限');
    }
  }
  
  // 重試機制（指數退避）
  private async retryWithBackoff<T>(
    fn: () => Promise<T>,
    maxRetries: number = 3
  ): Promise<T> {
    let lastError: Error;
    
    for (let i = 0; i < maxRetries; i++) {
      try {
        return await fn();
      } catch (error: any) {
        lastError = error;
        
        // 如果是429錯誤，使用指數退避
        if (error.status === 429 && i < maxRetries - 1) {
          const delay = Math.pow(2, i) * 1000; // 1s, 2s, 4s
          await new Promise(resolve => setTimeout(resolve, delay));
          continue;
        }
        
        throw error;
      }
    }
    
    throw lastError!;
  }
}
```

**角色定位**：溫暖、公正的情感調解法官。使用繁體中文、非指責語氣。

**輸入**：

- `plaintiffStatement`：原告陳述
- `defendantStatement`：被告陳述

- `summaryBrief`（可選）：v2.1 關係摘要提示（注入 `generateSummary`）

**輸出格式**（v2.6 動態結構，AI 根據案例靈活調整，使用「調整空間」而非「責任」措辭）：

可用段落模組（AI 根據臨床判斷決定哪些段落需要多著墨、哪些精簡帶過或合併）：
1. 我聽見你們了（共情開場）
2. 你們之間發生了什麼（事件摘要 + 情緒命名 + 互動循環描述）
3. 這段衝突真正在說什麼（深層需求翻譯）
4. 你們做對了什麼（敘事治療正向肯定）
5. 你們表達愛的方式（如適用——非每個案例都需要）
6. 各自可以調整的方向（**調整比重**：角色 A X% 調整空間，角色 B Y% 調整空間）
7. 可以直接用的對話（根據溝通風格和改變準備度定制，非固定 NVC 句式）
8. 具體可以嘗試的事（按時間軸分組，數量和深度配合嚴重程度）
9. 如果嘗試了但覺得很難（復發預防正常化——v3.2 新增明確段落）
10. 對自己溫柔一點（ACT 自我慈悲元素——v2.6 新增）

#### v2.0 心理畫像注入（`{psychProfileBlock}`）

根據用戶 `richness_score` 動態生成注入段落（由 `JudgmentService` 在構建 prompt 前計算）：

| 級別 | richness_score | 注入內容 | 額外 token 預算 |
|------|---------------|---------|----------------|
| L0 | < 0.05 | 空字串（不注入） | 0 |
| L1 | 0.05–0.2 | `背景參考：{人格特質概述（1 句，confidence ≥ 0.4 的 top-1 insight）}。以上僅供理解溝通風格參考。` | ~50 |
| L2 | 0.2–0.5 | `背景參考：{人格 + 依附風格 + 文化背景（1-2 句泛化描述）}。以上僅供理解雙方互動模式參考，不作為偏袒依據。` | ~120 |
| L3 | ≥ 0.5 | `背景參考：{完整段落：人格、依附、家庭原生、信念價值（泛化表述）+ 互動層分析}。以上僅供深入理解雙方需求與溝通模式參考，不作為偏袒依據。` | ~300（上限 2500 token） |

**隱私保障**：注入內容一律使用泛化表述（如「傾向迴避衝突」），不引用用戶原始敘事或具體事件。詳見 [服務層設計 § JudgmentService 升級](./04-服務層設計.md)。

### 判決生成實現

```typescript
async generateJudgment(caseType: string, plaintiffStatement: string, defendantStatement: string): Promise<JudgmentResponse> {
  try {
    const prompt = this.buildJudgmentPrompt(caseType, plaintiffStatement, defendantStatement);
    const content = await this.generateText(prompt, {
      maxTokens: 2000,
      temperature: 0.7,
      systemPrompt: '你是母熊法庭的情感調解法官。請嚴格遵循格式與要求，溫暖、公正、尊重雙方。'
    });
    
    // 解析判決內容
    const judgmentContent = content;
    
    // 提取責任分比例
    const responsibilityRatio = this.extractResponsibilityRatio(content);
    
    // 生成摘要（50-80字，純文字）
    const summary = await this.generateSummary(content);
    
    return {
      content: judgmentContent,
      responsibilityRatio,
      summary
    };
  } catch (error) {
    logger.error('Failed to generate judgment', { error });
    throw error;
  }
}

// 提取責任分比例
private extractResponsibilityRatio(content: string): { plaintiff: number; defendant: number } {
  // 使用正則表達式提取
  const regex = /原告[：:]\s*(\d+)%\s*責任|被告[：:]\s*(\d+)%\s*責任/g;
  const matches = content.matchAll(regex);
  
  let plaintiffRatio = 50; // 默認
  let defendantRatio = 50;
  
  for (const match of matches) {
    if (match[1]) {
      plaintiffRatio = parseInt(match[1]);
    }
    if (match[2]) {
      defendantRatio = parseInt(match[2]);
    }
  }
  
  // 確保總和為100
  const total = plaintiffRatio + defendantRatio;
  if (total !== 100) {
    plaintiffRatio = Math.round((plaintiffRatio / total) * 100);
    defendantRatio = 100 - plaintiffRatio;
  }
  
  return { plaintiff: plaintiffRatio, defendant: defendantRatio };
}
```

```typescript
async generateReconciliationPlans(prompt: string): Promise<ReconciliationPlan[]> {
  try {
    const content = await this.generateText(prompt, {
      maxTokens: 3000,
      temperature: 0.8, // 更高的創造性
      systemPrompt: '你是母熊法庭的和好方案設計師，輸出需嚴格遵守 JSON 格式。'
    });
    
    // 解析JSON響應
    let plans: ReconciliationPlan[];
    try {
      plans = JSON.parse(content);
    } catch (error) {
      // 如果JSON解析失敗，嘗試提取JSON部分
      const jsonMatch = content.match(/\[[\s\S]*\]/);
      if (jsonMatch) {
        plans = JSON.parse(jsonMatch[0]);
      } else {
        throw new Error('無法解析AI響應');
      }
    }
    
    // 驗證和處理方案
    return plans.map(plan => ({
      ...plan,
      difficulty_level: this.calculateDifficulty(plan),
      estimated_duration: plan.estimated_duration || this.estimateDuration(plan)
    }));
  } catch (error) {
    logger.error('Failed to generate reconciliation plans', { error });
    throw error;
  }
}

// 計算難度等級
private calculateDifficulty(plan: ReconciliationPlan): 'easy' | 'medium' | 'hard' {
  const totalScore = 
    plan.time_cost +
    plan.money_cost +
    plan.emotion_cost +
    plan.skill_requirement;
  
  if (totalScore <= 8) return 'easy';
  if (totalScore <= 12) return 'medium';
  return 'hard';
}

// 估算持續時間
private estimateDuration(plan: ReconciliationPlan): number {
  const totalScore = 
    plan.time_cost +
    plan.money_cost +
    plan.emotion_cost +
    plan.skill_requirement;
  
  if (totalScore <= 8) return 1; // 1-2天
  if (totalScore <= 12) return 5; // 3-7天
  return 14; // 1-4週
}
```

```typescript
export const optimizePrompt = (prompt: string, maxLength: number = 3000): string => {
  if (prompt.length <= maxLength) {
    return prompt;
  }
  
  // 截斷並保留重要部分
  const importantParts = prompt.split('\n\n');
  let optimized = '';
  
  for (const part of importantParts) {
    if (optimized.length + part.length > maxLength) {
      break;
    }
    optimized += part + '\n\n';
  }
  
  return optimized;
};
```

```typescript
export class JudgmentCache {
  private cache: Map<string, CachedJudgment> = new Map();
  private cacheTTL = 24 * 60 * 60 * 1000; // 24小時
  
  // 生成緩存鍵
  private generateCacheKey(case_: Case): string {
    // 基於案件類型和關鍵詞生成
    const keywords = this.extractKeywords(case_.plaintiff_statement + case_.defendant_statement);
    return `${case_.type}:${keywords.join(',')}`;
  }
  
  // 獲取緩存
  get(case_: Case): Judgment | null {
    const key = this.generateCacheKey(case_);
    const cached = this.cache.get(key);
    
    if (!cached) {
      return null;
    }
    
    // 檢查是否過期
    if (Date.now() - cached.timestamp > this.cacheTTL) {
      this.cache.delete(key);
      return null;
    }
    
    return cached.judgment;
  }
  
  // 設置緩存
  set(case_: Case, judgment: Judgment): void {
    const key = this.generateCacheKey(case_);
    this.cache.set(key, {
      judgment,
      timestamp: Date.now()
    });
  }
  
  // 提取關鍵詞
  private extractKeywords(text: string): string[] {
    // 簡單的關鍵詞提取（可優化）
    const words = text.split(/\s+/);
    return words.slice(0, 5); // 取前5個詞
  }
}
```

## 📊 錯誤處理和重試

### 重試機制

```typescript
async generateWithRetry<T>(
  fn: () => Promise<T>,
  maxRetries: number = 3
): Promise<T> {
  let lastError: Error;
  
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error: any) {
      lastError = error;
      
      // 如果是429錯誤（限流），使用指數退避
      if (error.status === 429 && attempt < maxRetries - 1) {
        const delay = Math.pow(2, attempt) * 1000; // 1s, 2s, 4s
        logger.warn(`Rate limited, retrying after ${delay}ms`, { attempt });
        await new Promise(resolve => setTimeout(resolve, delay));
        continue;
      }
      
      // 如果是5xx錯誤，重試
      if (error.status >= 500 && attempt < maxRetries - 1) {
        const delay = Math.pow(2, attempt) * 1000;
        logger.warn(`Server error, retrying after ${delay}ms`, { attempt });
        await new Promise(resolve => setTimeout(resolve, delay));
        continue;
      }
      
      throw error;
    }
  }
  
  throw lastError!;
}
```

### SSE 雙通道格式（設計）與實際回應格式

**設計中的 SSE 格式**：
```
[AI 完整回應文本]
---METADATA---
{"intent":"臨床目的","target_domains":["family_origin"],"should_end":false,"safety_flag":false}
```

**⚠️ 實際實現**：訪談回應**使用** `stream: true` 逐 token 流式輸出。System Prompt 指示 AI 採用「可見文本 + `---METADATA---` + JSON」雙通道格式。代碼同時支援三種回應格式的自動偵測與解析：
1. **分隔符格式**（主要）：AI 先輸出回覆文本，再以 `---METADATA---` 分隔，最後一行輸出 JSON metadata。流式階段即時推送文本給前端（SSE `token` 事件），分隔符後的 metadata 在完成後解析。
2. **純 JSON 格式**（回退）：若 AI 以 `{` 或 `[` 開頭回覆（偵測為 JSON 格式），代碼累積完整內容後以 `fullContent.match(/\{[\s\S]*\}/)` 解析。
3. **純文本格式**（兜底）：若以上兩者均不匹配，全部內容作為回覆文本。

解析後透過 SSE 推送結構化事件（`token` → `metadata` → `safety_alert`（如有）→ `complete`）。

**安全防護**：
- 後端取 AI 輸出的**最後一個** `---METADATA---`（`fullContent.lastIndexOf(DELIMITER)`，防用戶在回答中注入假 metadata）
- 用戶文字存儲前 strip 分隔符字串（`userResponse.replace(/---METADATA---/gi, '')`）
- AI prompt 指令：只在末尾輸出一次分隔符

### 異步處理管線與狀態機

`POST /interview/:id/end` 後，session 狀態由 `in_progress` → `processing`。後端啟動異步管線，依序調用上述 Prompt 2-5：

| 步驟 | AI Call | 失敗策略 |
|------|---------|---------|
| 1. 域分類 + 敘事提取 | Prompt 2（INTERVIEW_AI_CONFIG，gpt-4o-mini）+ 純邏輯 | 域分類失敗回退到原始分類；單域提取失敗不阻塞 |
| 2. 敘事摘要 | Prompt 3（ANALYSIS_AI_CONFIG，GPT-4o） | 單域失敗 → 保留原始敘事 |
| 3. 洞察提取 | Prompt 4（ANALYSIS_AI_CONFIG，GPT-4o） | 單域失敗 → 保留舊 insight |
| 4. 計算豐富度 | 純計算（無 AI） | 不會失敗 |
| 5. 反饋卡片 | Prompt 5（ANALYSIS_AI_CONFIG，GPT-4o） | 使用預設模板 |
| 6. 狀態更新 | 純 DB write | 設 `completed` / `processing_failed` + `partial_success` |

- 每步 AI call 失敗 → 重試 2 次（固定退避 2s/4s）
- 全部完成 → `completed`；全部失敗 → `processing_failed`；部分成功 → `completed` + `partial_success` 標記
- 前端 polling `GET /interview/:id`，60 秒超時非阻塞
- `processing_failed` session 可由用戶在「我的故事」頁面觸發重試

詳見 [服務層設計 § AsyncPipelineService](./04-服務層設計.md) 和 [`UPGRADE_PLAN_PERSONALIZED_JUDGMENT.md` § v10](../../UPGRADE_PLAN_PERSONALIZED_JUDGMENT.md)。
