## 📁 文件存儲服務

### 服務概述

處理用戶上傳的文件（圖片、視頻），包括驗證、處理、存儲。

### 技術選型

- **開發環境**：本地文件系統
- **生產環境**：Cloudinary（免費額度）或AWS S3
- **圖片處理**：Sharp（Node.js）

### 接口定義

```typescript
interface FileService {
  // 上傳文件
  uploadFile(file: Express.Multer.File, userId: string): Promise<FileUploadResult>;
  
  // 處理圖片（壓縮、調整大小）
  processImage(file: Express.Multer.File): Promise<Buffer>;
  
  // 刪除文件
  deleteFile(fileUrl: string): Promise<void>;
  
  // 驗證文件
  validateFile(file: Express.Multer.File): void;
}
```

### 實現

**文件**：`src/services/file.service.ts`

```typescript
import multer from 'multer';
import sharp from 'sharp';
import path from 'path';
import fs from 'fs/promises';
import { AppError } from '../utils/errors';
import logger from '../config/logger';

export class FileService {
  private readonly allowedTypes = {
    image: ['image/jpeg', 'image/png', 'image/gif'],
    video: ['video/mp4']
  };
  
  private readonly maxFileSize = 5 * 1024 * 1024; // 5MB
  
  // 驗證文件
  validateFile(file: Express.Multer.File): void {
    // 檢查文件大小
    if (file.size > this.maxFileSize) {
      throw new AppError(413, 'FILE_TOO_LARGE', '文件大小不能超過5MB');
    }
    
    // 檢查文件類型
    const isValidImage = this.allowedTypes.image.includes(file.mimetype);
    const isValidVideo = this.allowedTypes.video.includes(file.mimetype);
    
    if (!isValidImage && !isValidVideo) {
      throw new AppError(400, 'INVALID_FILE_TYPE', '不支持的文件類型，只支持JPG、PNG、GIF、MP4');
    }
  }
  
  // 處理圖片
  async processImage(file: Express.Multer.File): Promise<Buffer> {
    try {
      // 壓縮圖片，最大尺寸1920x1080
      const processed = await sharp(file.buffer)
        .resize(1920, 1080, {
          fit: 'inside',
          withoutEnlargement: true
        })
        .jpeg({ quality: 85 }) // JPEG質量85%
        .toBuffer();
      
      return processed;
    } catch (error) {
      logger.error('Image processing error', { error });
      throw new AppError(500, 'IMAGE_PROCESSING_ERROR', '圖片處理失敗');
    }
  }
  
  // 上傳文件（本地存儲）
  async uploadFileLocal(
    file: Express.Multer.File,
    userId: string
  ): Promise<FileUploadResult> {
    // 驗證文件
    this.validateFile(file);
    
    // 處理圖片
    let processedBuffer: Buffer;
    if (this.allowedTypes.image.includes(file.mimetype)) {
      processedBuffer = await this.processImage(file);
    } else {
      processedBuffer = file.buffer;
    }
    
    // 生成唯一文件名
    const ext = path.extname(file.originalname);
    const filename = `${Date.now()}-${Math.round(Math.random() * 1E9)}${ext}`;
    const filepath = path.join('uploads', userId, filename);
    
    // 確保目錄存在
    const dir = path.dirname(filepath);
    await fs.mkdir(dir, { recursive: true });
    
    // 保存文件
    await fs.writeFile(filepath, processedBuffer);
    
    // 生成URL（開發環境）
    const fileUrl = `/uploads/${userId}/${filename}`;
    
    return {
      url: fileUrl,
      filename,
      size: processedBuffer.length,
      type: file.mimetype
    };
  }
  
  // 上傳文件（Cloudinary）
  async uploadFileCloudinary(
    file: Express.Multer.File,
    userId: string
  ): Promise<FileUploadResult> {
    const cloudinary = require('cloudinary').v2;
    
    // 驗證文件
    this.validateFile(file);
    
    // 處理圖片
    let uploadBuffer: Buffer;
    if (this.allowedTypes.image.includes(file.mimetype)) {
      uploadBuffer = await this.processImage(file);
    } else {
      uploadBuffer = file.buffer;
    }
    
    // 上傳到Cloudinary
    return new Promise((resolve, reject) => {
      cloudinary.uploader.upload_stream(
        {
          folder: `mother-bear-court/${userId}`,
          resource_type: 'auto',
          transformation: [
            { width: 1920, height: 1080, crop: 'limit' },
            { quality: 'auto' }
          ]
        },
        (error: any, result: any) => {
          if (error) {
            reject(new AppError(500, 'UPLOAD_ERROR', '文件上傳失敗'));
          } else {
            resolve({
              url: result.secure_url,
              filename: result.public_id,
              size: result.bytes,
              type: result.resource_type
            });
          }
        }
      ).end(uploadBuffer);
    });
  }
  
  // 刪除文件
  async deleteFile(fileUrl: string): Promise<void> {
    try {
      // 判斷是本地文件還是Cloudinary文件
      if (fileUrl.startsWith('http')) {
        // Cloudinary文件
        const cloudinary = require('cloudinary').v2;
        const publicId = this.extractPublicId(fileUrl);
        await cloudinary.uploader.destroy(publicId);
      } else {
        // 本地文件
        const filepath = path.join(process.cwd(), fileUrl);
        await fs.unlink(filepath);
      }
    } catch (error) {
      logger.error('File deletion error', { fileUrl, error });
      // 不拋出錯誤，允許刪除失敗
    }
  }
}
```

### 通知表與發送邏輯（MVP 階段說明）

`notifications` 表與 `NotificationService` 目前僅負責**存庫**（`list`、`create`，狀態為 `pending`）。  
**實際郵件發送**由 `EmailService` 在註冊、配對、判決等業務流程中**直接調用**，不經過 `notifications` 表。  
若前端有「通知中心」列表，現階段僅能查詢到待發記錄；後續可增加 Worker/cron 消費 `notifications` 表並與 `EmailService` 整合，統一發送。

## ⏰ 定時任務

### 服務概述

處理定時執行的任務，如郵件回訪、數據清理、緩存更新等。

### 技術選型

- **任務調度**：node-cron

### 任務列表

1. **郵件回訪任務**：判決後7天、30天自動回訪（**MVP 階段未實現**，列為後續迭代）
2. **數據清理任務**：清理過期驗證碼、未激活用戶
3. **緩存更新任務**：更新熱點數據緩存
4. **統計任務**：生成每日統計報告

5. **🧠 訪談 session 過期清理（v2.0 新增）**：`in_progress` 超過 24 小時的訪談 session 自動標記為 `abandoned`。若該 session 已 ≥ 3 輪，先觸發 `AsyncPipelineService.process()` 保存已有數據，再標記 `abandoned`。

**MVP 階段說明**：當前僅實現清理類定時任務（見 `src/jobs/cleanup.job.ts`：過期 Session、孤兒上傳、臨時配對、過期驗證碼、AI 每日計數重置）。7天/30天回訪任務列為**後續迭代**。v2.0 需新增訪談 session 清理任務。

```typescript
import cron from 'node-cron';
import { CaseRepository } from '../repositories/case.repository';
import { EmailService } from '../services/email.service';
import logger from '../config/logger';

export class FollowupJob {
  constructor(
    private caseRepository: CaseRepository,
    private emailService: EmailService
  ) {}
  
  // 啟動任務
  start(): void {
    // 每天凌晨2點執行
    cron.schedule('0 2 * * *', async () => {
      await this.checkFollowups();
    });
    
    logger.info('Followup job started');
  }
  
  // 檢查需要回訪的案件
  private async checkFollowups(): Promise<void> {
    try {
      // 查找7天前完成的案件
      const sevenDaysAgo = new Date();
      sevenDaysAgo.setDate(sevenDaysAgo.getDate() - 7);
      
      const cases7Days = await this.caseRepository.findCompletedAfter(sevenDaysAgo);
      
      for (const case_ of cases7Days) {
        // 檢查是否已發送7天回訪
        if (!case_.followup_7days_sent) {
          await this.send7DayFollowup(case_);
          await this.caseRepository.update(case_.id, {
            followup_7days_sent: true
          });
        }
      }
      
      // 查找30天前完成的案件
      const thirtyDaysAgo = new Date();
      thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);
      
      const cases30Days = await this.caseRepository.findCompletedAfter(thirtyDaysAgo);
      
      for (const case_ of cases30Days) {
        // 檢查是否已發送30天回訪
        if (!case_.followup_30days_sent) {
          await this.send30DayFollowup(case_);
          await this.caseRepository.update(case_.id, {
            followup_30days_sent: true
          });
        }
      }
      
      logger.info('Followup check completed', {
        sevenDays: cases7Days.length,
        thirtyDays: cases30Days.length
      });
    } catch (error) {
      logger.error('Followup job error', { error });
    }
  }
  
  // 發送7天回訪郵件
  private async send7DayFollowup(case_: Case): Promise<void> {
    const user = await userRepository.findById(case_.plaintiff_id);
    if (user && user.notification_enabled) {
      await this.emailService.sendFollowupEmail(
        user.email,
        case_.id,
        '7days'
      );
    }
  }
  
  // 發送30天回訪郵件
  private async send30DayFollowup(case_: Case): Promise<void> {
    const user = await userRepository.findById(case_.plaintiff_id);
    if (user && user.notification_enabled) {
      await this.emailService.sendFollowupEmail(
        user.email,
        case_.id,
        '30days'
      );
    }
  }
}
```

## 🔄 Session管理（快速體驗模式）

### Session存儲

**文件**：`src/services/session.service.ts`

```typescript
import { v4 as uuidv4 } from 'uuid';
import { CaseRepository } from '../repositories/case.repository';
import { AppError } from '../utils/errors';
import logger from '../config/logger';

interface SessionData {
  id: string;
  createdAt: Date;
  expiresAt: Date;
  caseIds: string[];
  pairingId?: string;
  lastAccessedAt: Date;
}

export class SessionService {
  private sessionTTL = 24 * 60 * 60 * 1000; // 24小時
  private completedSessionTTL = 7 * 24 * 60 * 60 * 1000; // 7天（已完成的案件）
  
  // 生成Session ID（格式：guest_timestamp_random）
  generateSessionId(): string {
    const timestamp = Date.now();
    const random = Math.random().toString(36).substring(2, 8);
    return `guest_${timestamp}_${random}`;
  }
  
  // 創建Session
  async createSession(): Promise<string> {
    const sessionId = this.generateSessionId();
    const now = new Date();
    const expiresAt = new Date(now.getTime() + this.sessionTTL);
    
    const sessionData: SessionData = {
      id: sessionId,
      createdAt: now,
      expiresAt,
      caseIds: [],
      lastAccessedAt: now
    };
    
    // 存儲到Redis或數據庫
    await this.saveSession(sessionId, sessionData);
    
    logger.info('Session created', { sessionId });
    
    return sessionId;
  }
  
  // 獲取Session
  async getSession(sessionId: string): Promise<SessionData | null> {
    // 驗證Session ID格式
    if (!sessionId.startsWith('guest_')) {
      return null;
    }
    
    const session = await this.loadSession(sessionId);
    if (!session) {
      return null;
    }
    
    // 檢查是否過期
    if (new Date() > session.expiresAt) {
      await this.deleteSession(sessionId);
      return null;
    }
    
    // 更新最後訪問時間
    session.lastAccessedAt = new Date();
    await this.saveSession(sessionId, sessionData);
    
    return session;
  }
  
  // 添加案件到Session
  async addCaseToSession(sessionId: string, caseId: string): Promise<void> {
    const session = await this.getSession(sessionId);
    if (!session) {
      throw new AppError(401, 'SESSION_EXPIRED', 'Session已過期');
    }
    
    if (!session.caseIds.includes(caseId)) {
      session.caseIds.push(caseId);
      await this.saveSession(sessionId, session);
    }
  }
  
  // 設置配對ID
  async setPairingId(sessionId: string, pairingId: string): Promise<void> {
    const session = await this.getSession(sessionId);
    if (!session) {
      throw new AppError(401, 'SESSION_EXPIRED', 'Session已過期');
    }
    
    session.pairingId = pairingId;
    await this.saveSession(sessionId, session);
  }
  
  // 標記Session為已完成（延長過期時間）
  async markSessionCompleted(sessionId: string): Promise<void> {
    const session = await this.getSession(sessionId);
    if (!session) {
      return;
    }
    
    // 延長過期時間到7天
    session.expiresAt = new Date(Date.now() + this.completedSessionTTL);
    await this.saveSession(sessionId, session);
  }
  
  // 保存Session（使用Redis或數據庫）
  private async saveSession(sessionId: string, data: SessionData): Promise<void> {
    // 優先使用Redis
    if (process.env.REDIS_URL) {
      const redis = require('redis').createClient({ url: process.env.REDIS_URL });
      await redis.setEx(
        `session:${sessionId}`,
        Math.floor((data.expiresAt.getTime() - Date.now()) / 1000),
        JSON.stringify(data)
      );
    } else {
      // 使用數據庫
      await quickSessionRepository.upsert({
        id: sessionId,
        session_data: data,
        expires_at: data.expiresAt,
        last_accessed_at: data.lastAccessedAt
      });
    }
  }
  
  // 加載Session
  private async loadSession(sessionId: string): Promise<SessionData | null> {
    // 優先使用Redis

    } else {

      const session = await quickSessionRepository.findById(sessionId);
      return session ? session.session_data : null;
    }
  }

    } else {

    }
```

## 🧠 訪談併發鎖服務（v2.0 新增）

> 設計依據：`UPGRADE_PLAN_PERSONALIZED_JUDGMENT.md` v10

### 服務概述

`lockService` 為訪談 `/respond` 端點提供 session 級 mutex lock，防止同一 session 同時收到多個用戶回答導致的 turn 重複問題。

### 接口定義

```typescript
interface LockService {
  withLock<T>(key: string, fn: () => Promise<T>, timeoutMs?: number): Promise<T>;
  acquireLock(key: string, timeoutMs?: number): Promise<boolean>;
  releaseLock(key: string): Promise<void>;
}
```

**`withLock` 語義**：
1. 調用 `acquireLock(key)`，若失敗拋 `409 CONCURRENT_REQUEST`
2. 執行 `fn()`（在 respond 場景中，`fn` 覆蓋：AI 調用 → metadata 解析 → turn 持久化 → SSE 流關閉 的**整個過程**）
3. `fn()` 完成後（無論成功或失敗），**立即調用 `releaseLock(key)`**
4. 若 `fn()` 中途崩潰，`releaseLock` 在 finally 區塊中執行，確保鎖不會永久佔用

**與 `InterviewService.respond` 的對應關係**（見 [04-服務層設計](./04-服務層設計.md)）：
```typescript
async *respond(id, userId, message) {
  await lockService.withLock(`interview:respond:${id}`, async () => {
    // AI 調用 → 解析 metadata → 存儲 turn → yield complete
  });
  // 鎖在此處已釋放，下一個 respond 可獲取鎖
}
```

### 實現策略

**Redis 模式**（生產）：
```typescript
const lockKey = `interview:respond:${sessionId}`;
const acquired = await redis.set(lockKey, '1', 'NX', 'PX', 30000);
if (!acquired) throw new AppError(409, 'CONCURRENT_REQUEST', '該會話正在處理中');
// releaseLock: await redis.del(lockKey);
```

**DB Advisory Lock 模式**（開發/無 Redis）：
```typescript
const lockId = hashToInt(sessionId);
const acquired = await prisma.$executeRaw`SELECT pg_try_advisory_lock(${lockId})`;
// releaseLock: await prisma.$executeRaw`SELECT pg_advisory_unlock(${lockId})`;
```

### 使用場景

| 場景 | Lock Key | 超時 |
|------|----------|------|
| 訪談回答 | `interview:respond:{sessionId}` | 30 秒 |
| 判決生成 | `lock:judgment:{caseId}` | 60 秒 |
| 案件建立 | `lock:case:{sessionId}` | 15 秒 |

**說明**：前端已做第一層防護（disable 按鈕），後端 mutex 為第二層。重複請求返回 `409 Conflict`。
