## 2. 頁面設計（按實作）

## 2.1 首頁 `Home`

檔案：`pages/Home/index.tsx`

核心行為：

- CTA 依登入狀態分流：
  - 已登入 -> `/case/create`
  - 未登入 -> `/quick-experience/create`
- 額外入口：
  - 未登入可進 `/quick-experience/collaborative`
- 以 `FlowSimulation` 展示流程可視化

設計重點：

- SEO、可及性 skip link、動畫包裝
- 文案與多語由 `t(...)` 提供

## 2.2 快速體驗建立 `QuickExperience/Create`

檔案：`pages/QuickExperience/Create/index.tsx`

核心行為：

- 進頁先確保 session 存在（必要時 `createSession`）
- 本地草稿自動保存（`localStore`）
- 支持「案件恢復」：檢查當前 session 是否已有進行中/已提交案件
- 提交時：
  - `createQuickCase`
  - 如有附件再調 `uploadEvidence`
  - 成功跳轉 `/quick-experience/result/:id`

關鍵狀態：

- `layoutMode`
- `plaintiffStatement/defendantStatement`
- `evidenceFiles`
- `recoveredCase`

## 2.3 協作模式 `QuickExperience/Collaborative`

檔案：`pages/QuickExperience/Collaborative/index.tsx`

核心行為：

- 四階段 UI：`intro -> role_a -> handoff -> role_b -> submitting`
- `createCollaborativeCase` 兩段式提交：
  - A 先建立 case + session
  - B 再補 `defendant_statement` 完成提交
- 成功導向 `/quick-experience/result/:id`

## 2.4 快速體驗結果 `QuickExperience/Result`

檔案：`pages/QuickExperience/Result/index.tsx`

核心行為：

- 以 `caseId + sessionId` 拉取判決
- `JUDGMENT_PENDING` 走輪詢（含超時和退避）
- `JUDGMENT_FAILED` 提供重試觸發
- 可補傳證據（若案件狀態允許）
- 結果區塊由子組件拆分：
  - `ResultHeader`
  - `SummarySection`
  - `ResponsibilitySection`
  - `JudgmentSection`
  - `EvidenceUploadSection`
  - `RegisterPromptSection`

## 2.5 完整模式建案 `Case/Create`

檔案：`pages/Case/Create/index.tsx`

核心行為：

- 先檢查配對狀態，不滿足則引導至 `/profile/pairing`
- 支援 `remote/collaborative` 建案模式
- 可在建案前顯示 pre-case 訪談引導（richness 判斷）
- 成功建立後跳轉 `/case/:id`

## 2.6 判決詳情 `Judgment/Detail`

檔案：`pages/Judgment/Detail/index.tsx`

核心行為：

- 載入判決詳情 + 責任比例 + Markdown 判決內容
- 可接受/拒絕判決（含評分）
- 可直接發起和好方案生成
- 若畫像 richness 偏低，顯示 post-judgment 訪談引導卡

## 2.7 訪談聊天 `Interview/Chat`

檔案：`pages/Interview/Chat/index.tsx`

核心行為：

- 進頁 `getSession`
- `respond/skip` 走 SSE，逐 token 顯示
- 支持 `cancelStream`
- 根據 `shouldEnd` 或錯誤碼給出結束/跳轉行為
- `safetyAlert` 以獨立組件展示

## 2.8 訪談結果 `Interview/Result`

檔案：`pages/Interview/Result/index.tsx`

核心行為：

- `processing` 狀態輪詢 session
- `processing_failed` 可 `retryFailed`
- 有 `feedback_card` 則渲染 `FeedbackCardComponent`
- 無卡片則給回 profile 的 fallback 結果頁

## 2.9 我的故事 `Profile/MyStory`

檔案：`pages/Profile/MyStory/index.tsx`

核心行為：

- 展示 narratives/insights/richness/feedback history
- 支援繼續訪談與 failed session retry
- 支援刪除全部心理畫像資料

## 2.10 聊天室 `Chat/Room`

檔案：`pages/Chat/Room/index.tsx`

核心行為：

- room 建立/邀請/加入/拒絕/離開/踢人
- 訊息列表虛擬渲染（`react-virtuoso`）
- SSE stream 同步 room status / message event
- 可選訊息後發起 `request-judgment`
- 包含歷史分頁、錨點跳轉、未讀提示等高互動狀態

## 3. 交互一致性規則（實作現況）

1. 所有核心流程皆依 store/service 回傳錯誤碼做 UX 分流  
2. 多數頁面採 mounted guard / stale guard，避免卸載後 setState  
3. 快速體驗與完整模式共存：session 與 auth token 可能同時存在  
4. SSE 流程頁面（訪談、聊天室）都含中斷與重連策略
