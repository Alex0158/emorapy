## 口徑定義

- `GuardType`
  - `ProtectedRoute`：需登入，未登入跳 `/auth/login`；直連保護頁 URL 且未登入時一律跳轉（hydration 逾時亦視為未登入）。
  - `PublicRoute`：登入後不可訪問，會跳 `/case/list`。
  - `Public`：不經路由守衛。
- `InCount`：可到達該頁的上游入口數（按路由跳轉來源去重估算）。
- `OutCount`：從該頁可跳轉的下游頁面數（按路由去重估算）。
- `狀態`
  - `主入口`：主流程高頻路由。
  - `弱入口`：公開或外部導入，易受 URL 直訪影響。
  - `候選保留`：主要只做重定向或兼容用途。
- `功能完成度`
  - `已完成`：頁內可直接觸發核心 API，且有前端證據。
  - `跨功能依賴`：本頁觸發後導頁到他頁完成核心 API。
  - `待驗證`：後端能力存在，但頁面入口或調用證據不足。
  - `待承接`：後端契約或產品目標存在，但當前 `frontend` / `frontend-admin` 尚無對應 service、route 或頁面責任。

## 平台範圍

- 本文只裁決 `frontend/` 與 `frontend-admin/` 的真實 Web/Admin route、guard 與頁面責任；backend-only endpoint 不因同屬 `/api/v1/admin/*` 而自動算入已完成 Admin Web 頁面能力。
- 頁面所承接的上游產品需求以 `00-跨端產品核心/01-產品PRD總章.md` 的 `EMO-PRD-*` 為準；本文只裁決 Web/Admin 頁面投影，不重定義通用級 PRD。
- `mobile/` 已有 Emorapy App route group 與 M0-M5 普通用戶 screen baseline；本文仍不納入 App route，因本表只裁決 Web/Admin L1 route、guard 與頁面責任。
- App 端頁面、導航、原生能力與 Web 差異由 `20-App端/` 與 `50-跨端Mapping與Parity/` 記錄；其中 App screen / native navigation / platform adapter 的實作 gate 見 `20-App端/01-App導航與平台Adapter基線.md`，首輪能力到 Backend / API / DB / shared package 的工程對照見 `50-跨端Mapping與Parity/01-App首輪能力與工程落點Mapping.md`，App smoke / regression / CI / evidence 進場見 `08-測試規範與驗收/03-App測試與證據接入基線.md`。不得因本文 Web 頁面為 `已完成` 推斷 App 端同樣完成。
- App route / screen 的穩定投影應在 `20-App端/` 與 `50-跨端Mapping與Parity/` 維護；不應混入本文 Web/Admin L1 表造成平台語義混淆。

## 可達性補充規則

- `ProtectedRoute` 與 `PublicRoute` 均有 hydration 等待；hydration 未完成前不渲染保護內容，逾時（5 秒）一律導向登錄。
- frontend-admin：無 admin token 直連 `/admin/*`（除 `/admin/login`）會重定向至 `/admin/login`；登入後回跳只接受同源、已註冊且當前角色具權限的 Admin route。
- Header / BottomNav 的常駐入口按登入態分流：未登入只暴露首頁、快速體驗與登入/註冊；登入後主導航暴露 `/case/list`、`/execution/dashboard`、`/chat/room`，`/profile/index`、`/profile/settings` 由用戶下拉或個人入口承接。
- `chat/room*` 可作 URL 弱入口直連，但未登入可直連不等於 Header / BottomNav 常駐顯示該入口。
- 動態路由主入口由業務操作生成：`/case/:id`、`/judgment/:id`、`/reconciliation/:judgmentId/:id`、`/execution/:planId/checkin`、`/interview/:sessionId`。
- `chat/room*` 為弱入口，但不等於匿名即可完成全部操作；chat owner / participant 權限仍受 user/session 身份與房間角色約束。

| `/execution/:planId/replan` | 重新調整這一輪 | `POST /repair-tracks/:id/replan` + `GET /streams/repair_track/:id` | 顯示可理解等待／恢復狀態，成功後回到最新旅程 | stream 失敗時提供 retry 與返回 detail/dashboard |
| `/notifications` | 通知中心操作 | `GET /notifications` + `POST /notifications/:id/(read|dismiss|snooze|act)` | 刷新 inbox、深鏈回旅程或標記狀態變更 | 404/失效通知時回退列表刷新 |
| `/profile/index` | 啟動訪談 | `POST /psych-profile/consent` + `POST /interview/start` | 跳 `/interview/:sessionId` | consent 或限額錯誤提示 |
| `/profile/settings` | 更新個人資料 | `GET/PUT /user/profile` | 保存成功並保留設定上下文 | 失敗可重試並可回到個人資料頁 |
| `/interview/:sessionId/result` | 讀取結果/重試失敗流水線 | `GET /interview/:id` + `POST /interview/:id/retry` | 顯示 completed/processing/failed | 失敗可 retry 或返回 profile |
| `/profile/my-story` | 重試失敗訪談 | `POST /interview/:id/retry` | 進入處理中/可恢復 | 顯示 retry fail |
| `/chat/room/:roomId` | 建立邀請 | `POST /chat/rooms/:roomId/invites` | 產生 invite code | `canonical session_id` 與 `room.session_id` 不匹配或狀態衝突提示 |
| `/chat/room/:roomId` | 發訊息 | `POST /chat/rooms/:roomId/messages` | 訊息即時追加 | 限流提示（5s/30s） |
| `/chat/room/:roomId` | 發起轉判決 | `POST /chat/rooms/:roomId/request-judgment` | judgment ready 後導向 `/judgment/:id`；未登入則先跳 `/auth/login` 再回跳 | 競態/未就緒錯誤提示 |
| `/admin/*` | 配置、報表、AI Stream、media provider 與治理操作 | L1.2 已列 `frontend-admin` service/page 對應的 `/api/v1/admin/*` 與 provider 設定端點 | 區塊數據刷新 | 權限不足 403 保留頁內；backend-only admin notification / product-state recovery task 不在本列完成範圍內 |
