# improve v1.0

## 目的
以 DevSecOps/AI 小步重構與優化，確保測試與持續性。

## 何時觸發
- 大文件或複雜模塊需優化

## 步驟
1. 檢查命名規範；依七大原則/DevSecOps 審查與安全掃描；提供文件上下文。
2. 制定改善計劃與目標；任務分解與清單追蹤，AI 輔助生成方案。
3. 實施：重構 + 100% 單元測試 + 補全字典；運行 linter；評估能源效率。
4. 總結真實時間並更新知識庫。

## 輸出
- 前後比較
- confidence: high/medium/low
- pre-PR 狀態
