# 報告 01：本地開發環境（Local Development）

## 1. 環境定位

- 目標：開發、除錯、快速驗證 API/前端流程。
- 組成：`backend` + `frontend` + `frontend-admin`（admin 與主前台分離啟動）。

## 2. 啟動入口與端口

- 後端：`backend/package.json` -> `npm run dev` (`tsx watch src/index.ts`)
- 前台：`frontend/package.json` -> `npm run dev`（Vite）
- 管理台：`frontend-admin/package.json` -> `npm run dev`（Vite）
- 一鍵腳本：`scripts/start-dev.sh`（目前只啟動 backend + frontend）
- 預設端口：
  - backend: `3001`（`.env.example` 建議值；`env.ts` fallback `3000`）
  - frontend: `5173`
  - frontend-admin: `5174`

## 3. 關鍵環境變量

- 後端必需（`backend/src/config/env.ts`）
  - `DATABASE_URL`
  - `JWT_SECRET`
  - `OPENAI_API_KEY`
- 後端常用
  - `PORT`, `FRONTEND_URL`, `ALLOWED_ORIGINS`, `FILE_BASE_URL`
  - `ENABLE_SCHEDULED_JOBS`, `AI_MOCK`, `REDIS_URL`
  - `METRICS_ENABLED`, `METRICS_TOKEN`, `METRICS_ALLOWED_IPS`
- 前台（`frontend/.env.example`）
  - `VITE_API_BASE_URL`
  - `VITE_ADMIN_LOGIN_URL`
- 管理台（`frontend-admin/.env.example`）
  - `VITE_API_BASE_URL`
  - `VITE_ADMIN_LOGIN_URL`

## 4. 服務依賴（本環境）

- PostgreSQL：必需（除非測試情境另行跳過 DB init）
- OpenAI：配置上必需，可用 `sk-dev-*` 或 `AI_MOCK` 走 mock 路徑
- Redis：可選（無 Redis 會降級為記憶體快取/簡化鎖語義）
- SMTP：可選（未配時郵件服務只記警告不阻斷）
- Prometheus `/metrics`：開發可直接驗證
- 本地檔案上傳：使用 `UPLOAD_DIR`（預設 `./uploads`）

## 5. 驗證清單

- `GET http://localhost:3001/health`
- `GET http://localhost:3001/metrics`
- 前台 `http://localhost:5173`
- 管理台 `http://localhost:5174/admin/login`
- 可用 `scripts/verify-integration.sh` 做基本連通驗證

## 6. 風險與改進

- `PORT` 口徑仍有文件漂移（3000 vs 3001），容易造成反代/前端 API 指向混亂。
- `scripts/start-dev.sh` 未啟動 `frontend-admin`，新同事常以為管理台已啟動。
- 建議補一份單一 SSOT 的本地開發契約表（端口、必要 env、啟動順序、驗證端點）。
