# 熊媽媽法庭 - 開發環境配置詳情

本文件聚焦「實際以倉庫當前代碼為準」的開發環境配置。若與本文不一致，請以對應的 `package.json`、`.env.example` 與 `docs/ENVIRONMENT.md` 為準。

---

## 1) 版本與工具鏈

- Node.js：`>= 20.19.0`（見 `backend/package.json`、`frontend/package.json` 的 `engines.node`）
- 套件管理：npm（專案已有 `package-lock.json`）
- 後端：Express + TypeScript + Prisma
- 前端：React + TypeScript + Vite + Ant Design + Less
- Lint：Biome（`npm run lint`）
- 測試：後端 Jest；前端 Vitest + Testing Library

---

## 2) 前端（frontend）

路徑：`mother-bear-court/frontend`

常用腳本（以 `frontend/package.json` 為準）：

- `npm run dev`：啟動 Vite dev server
- `npm run build`：TypeScript build + Vite build
- `npm run lint`：Biome lint
- `npm run test`：Vitest run（一次性，CI 建議）
- `npx vitest`：watch 模式（本 repo 未提供 `npm run test:watch` 腳本）

環境變量：

- 範例檔：`mother-bear-court/frontend/.env.example`
- 必需：`VITE_API_BASE_URL`（預設 `http://localhost:3001/api/v1`）

聊天室相關注意：

- 訊息列表使用 `react-virtuoso` 虛擬列表（依賴 DOM 量測）。在 jsdom 測試環境下，聊天室頁測試會對 `react-virtuoso` 做 stub（見 `frontend/src/pages/Chat/Room/index.test.tsx`）。

---

## 3) 後端（backend）

路徑：`mother-bear-court/backend`

常用腳本（以 `backend/package.json` 為準）：

- `npm run dev`：`tsx watch` 啟動（熱重載）
- `npm run build` / `npm run build:prod`：編譯 TypeScript
- `npm start`：啟動 `dist/index.js`
- `npm run lint`：Biome lint
- `npm run test:unit` / `npm run test:integration`：Jest 測試
- `npm run prisma:generate`：生成 Prisma Client
- `npm run prisma:migrate`：本地/開發遷移（`prisma migrate dev`）

環境變量：

- 範例檔：`mother-bear-court/backend/.env.example`
- 詳細說明：`mother-bear-court/docs/ENVIRONMENT.md`

資料庫與遷移：

- 開發環境：`npx prisma migrate dev`
- 生產環境：`npx prisma migrate deploy`
- ⚠️ 本專案使用 Prisma migrations 為主，請勿依賴「手寫 SQL 初始 schema」教學文檔（容易與實際 schema 漂移）。

指標與監控：

- `GET /metrics`：Prometheus 指標（目前含 Chat counters）
- Chat 告警示例：`mother-bear-court/backend/ops/prometheus/chat-alerts.rules.yml`、`mother-bear-court/backend/docs/ALERTS_CHAT.md`

---

## 4) 本地啟動順序（建議）

1. 後端：`cd backend && npm install && cp .env.example .env && npm run prisma:generate && npm run prisma:migrate && npm run dev`
2. 前端：`cd frontend && npm install && cp .env.example .env && npm run dev`
3. 驗證：
   - 後端健康：`GET /health`
   - API：`GET /api/v1/...`
   - 指標：`GET /metrics`
