# CJ 核心開發文件（唯一工程 SSOT）

本目錄是 `CJ` 項目唯一有效的工程與產品正式文檔依據。

對整個 `/Users/alex/Desktop/CJ` 而言，Markdown 只允許存在於三個區域：

1. `mother-bear-court/docs/核心開發文件/`
2. `mother-bear-court/docs/核心營銷文件/`
3. 根級 `歸檔/`

後續做任何開發、修改、排查、回歸前，應先查本目錄。

本目錄只保留四類內容：

1. 正式核心規格
2. 必要的測試規範與活躍測試入口
3. 必要的證據與治理入口
4. 歷史回看與文檔治理入口

不直接作為開發依據的營銷材料、dated 驗收記錄、排查稿與歷史快照，不應留在根層正文。

---

## 使用順序

建議按以下順序閱讀：

1. 根層旗艦總覽文檔
2. 對應正式子域 README
3. 對應子域的 `00-*總覽.md`
4. `06-接口描述/`
5. [07-待處理問題與治理/README.md](./07-待處理問題與治理/README.md)（需要決定先修什麼、先治理什麼時）
6. [08-測試規範與驗收/README.md](./08-測試規範與驗收/README.md)（涉及測試策略、覆蓋口徑、驗收標準時）
7. [測試/README.md](./測試/README.md)（需要活躍案例、回歸包與執行入口時）
8. [90-證據與盤點/README.md](./90-證據與盤點/README.md)（需要證據鏈、環境核驗、頁面快照時）
9. [99-歷史降級索引/README.md](./99-歷史降級索引/README.md)（只在需要回溯歷史材料時）
10. [文件收斂/README.md](./文件收斂/README.md)（只在需要理解文檔治理規則與遷移過程時）

營銷、品牌與對外敘事資料不屬於工程 SSOT，需要時請轉到 [../核心營銷文件/README.md](../核心營銷文件/README.md)。

## 根層永久保留文檔

以下文件是 `核心開發文件/` 根層長期保留的旗艦總覽文檔：

| 文檔 | 定位 |
| --- | --- |
| `README.md` | 主入口與目錄使用方式 |
| `功能特性清單.md` | 功能存在、能力邊界與產品能力基線 |
| `頁面清單.md` | Web / Admin / Mobile 頁面入口、守衛與落點基線 |
| `全接口清單-主文檔.md` | API 註冊總表與狀態唯一來源 |
| `接口-功能-頁面-Mapping.md` | API / 功能 / 頁面交叉追溯矩陣，用於影響分析 |
| `業務流程整合.md` | 跨入口、跨角色主流程總覽 |
| `術語表.md` | 關鍵術語與語義口徑 |

根層文檔之間的主從關係固定如下：

1. API 是否在用、候選廢棄或已確認廢棄，只看 `全接口清單-主文檔.md`。
2. 功能是否存在、由哪個頁面承接，只看 `功能特性清單.md` 與 `頁面清單.md`。
3. 交叉追溯、影響分析與回歸範圍，只看 `接口-功能-頁面-Mapping.md`。
4. 跨端主流程只看 `業務流程整合.md`；術語口徑只看 `術語表.md`。
5. 模組級接口契約與風險說明只作為主冊補充，統一由 `06-接口描述/` 承接，不與主冊並列裁決。

## 分層規則

後續維護 `核心開發文件/` 時，統一遵循以下分層規則：

1. 根層只放少量旗艦總覽文檔。
2. 細規格一律進對應正式子域。
3. 測試高位規範一律進 `08-測試規範與驗收/`；活躍案例、回歸包與執行入口進 `測試/`。
4. 活躍問題與治理狀態一律進 `07-待處理問題與治理/`。
5. 證據鏈、環境核驗、頁面快照與手動回歸實體一律進 `90-證據與盤點/`。
6. dated 驗收記錄不得兼任正式規格；若內含穩定真相，應回寫到正式子域後再保留為證據或歷史。
7. 歷史降級與不再作主依據的材料一律進 `99-歷史降級索引/`。
8. 變更追蹤、同步規則與遷移過程一律記在 `文件收斂/`，不在正式正文尾部堆疊「本輪更新」。
9. 營銷、品牌、對外敘事與投放素材一律不進本目錄，統一留在 `docs/核心營銷文件/`。
10. `01-05` 這五個正式子域都應維持 `README.md + 00-*總覽.md` 的雙層入口；README 負責分工與跳轉，`00-*總覽.md` 負責穩定正文。

## 維護口徑

1. 新增文檔前，先判定自己屬於正式規格、測試規範、活躍測試、證據、治理還是歷史材料，再決定目錄。
2. 根層不新增臨時排查報告、發版回歸記錄、營銷分析、環境收口稿或單次驗收結論。
3. 若現行正文與證據、歷史或治理材料衝突，以根層旗艦文檔、對應正式子域正文與現碼為準。
4. 若需要回溯調整過程與原因，查 `文件收斂/` 或 `99-歷史降級索引/`，不要把過程寫回正式正文。
