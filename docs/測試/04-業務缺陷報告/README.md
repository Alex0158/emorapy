# 04-業務缺陷報告

**用途：**

- 只記錄已被測試證明的真實業務代碼/業務行為缺陷
- 不混入純測試問題

**說明：**

- 若某條測試是有效失敗，且確認不是測試設計問題，應在此立案
- 子目錄 `已解決/` 存放已完全修復缺陷的報告
- **已解決的報告應「移動」至 `已解決/`，而非刪除後重新生成**
- 部分修復的缺陷仍保留於本目錄，待完全解決後再移動
- 可另建總覽文件（如 `Emorapy-業務缺陷總覽-YYYYMMDD.md`）彙整所有缺陷

**典型內容：**

- 關聯失敗測試
- 業務邏輯梳理與實際代碼行為
- 問題判定
- 修復方案

本目錄與 `docs/參考文件/測試/04-業務缺陷報告/` 對齊同一套模版與邏輯。
