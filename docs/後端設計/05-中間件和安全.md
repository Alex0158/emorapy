# 中間件與安全（代碼對齊版）

**文檔版本**：v3.0  
**最後更新**：2026-03-05  
**對齊基準**：`backend/src/app.ts`、`backend/src/middleware/*.ts`

---

## 1. 中間件鏈路（實際順序）

由 `app.ts` 可見主要順序：

1. `requestId`（注入 `X-Request-ID`）
2. CORS 前置來源校驗 + `cors(...)`
3. `localeMiddleware`（`X-Locale`/`Accept-Language`）
4. `compression`
5. body parser（JSON/urlencoded）
6. JSON parse error guard（`INVALID_JSON`）
7. `performanceMonitor`
8. `requestLogger`
9. `opsMetrics`
10. `responseFormatter`
11. `/uploads` 保護鏈：`downloadLimiter` -> `optionalAuthenticate` -> `authorizeMedia`
12. 路由
13. 404 handler
14. `errorHandler`

---

## 2. 認證與授權

## 2.1 JWT（`auth.ts`）

- `authenticate`：必須帶 `Bearer token`
- 驗證 token 後會檢查：
  - user 存在且 `is_active`
  - `decoded.token_version === user.token_version`（密碼/安全變更可使舊 token 失效）
- `optionalAuthenticate`：失敗不阻斷請求，僅記 warn log

## 2.2 快速體驗 Session（`validateSession`）

- 支援 `x-session-id` 與 `query.session_id`
- 兩來源衝突時直接拒絕（`INVALID_SESSION_ID`）
- 驗證格式 + 有效性，並更新 `last_accessed_at`

## 2.3 Admin 認證（`adminAuth.ts`）

- 獨立 admin token 驗證（`verifyAdminToken`）
- 檢查 admin 帳號啟用狀態、`deleted_at`
- 以 token `iat` 對比 `admin.updated_at`，敏感變更後舊 token 失效
- 權限守衛：
  - `requireAdminPermissionAny`
  - `requireAdminPermissionAll`
  - `admin:all` 作為超級權限

---

## 3. 媒體資源保護（`authorizeMedia`）

`/uploads` 預設非公開，授權路徑：

1. 先做 IP blacklist（若配置）
2. 生產禁止 `ALLOW_PUBLIC_UPLOADS=true`
3. 若提供簽名 token，校驗檔名/hash/size/mtime/content-hash
4. 否則走 session 或 user 權限校驗：
   - quick case：檔案需屬於該 session 的案件
   - 登入用戶：需屬於本人或可訪問案件

---

## 4. 限流策略（`rateLimiter.ts`）

基線：

- 開發環境會放寬倍數
- `SKIP_RATE_LIMIT=true`（開發）可跳過

主要限流器：

- `generalLimiter`：通用請求
- `authLimiter` / `registerLimiter`
- `verificationCodeLimiter` / `verifyCodeLimiter`
- `resetPasswordLimiter` / `resetConfirmLimiter`
- `pairingJoinLimiter`
- `aiLimiter`
- `uploadLimiter` / `downloadLimiter`
- `interviewStartLimiter` / `interviewRespondLimiter`

> 補充：訪談還有業務層限制（如 substantive 次數），不只中間件限流。

---

## 5. 驗證、錯誤、觀測

## 5.1 驗證（`validator.ts`）

- Joi 驗證 `body/params/query`
- 多錯誤聚合後統一拋 `VALIDATION_ERROR`

## 5.2 響應格式（`responseFormatter.ts`）

- 所有回應統一帶 `success`
- 自動注入 `meta.request_id`、`meta.timestamp`
- 根據 locale 翻譯 `message/error.message`

## 5.3 錯誤處理（`errorHandler.ts`）

- `AppError`：按業務錯誤碼返回
- `multer`：映射為 `FILE_TOO_LARGE` 等
- Prisma：
  - `P2002` -> `EMAIL_EXISTS`/`CONFLICT`
  - `P2025` -> `NOT_FOUND`
- 生產環境隱藏內部錯誤細節

## 5.4 日誌與性能

- `requestLogger`：按環境分層記錄（生產只記錯誤/慢請求）
- `performanceMonitor`：慢請求與錯誤性能指標
- `opsMetrics`：HTTP status 指標（排除 health）

---

## 6. 安全邊界要點

1. CORS 先白名單校驗，再交給 `cors` 中間件輸出頭  
2. `trust proxy` 依環境設置，配合限流與 IP 判斷  
3. JWT 與 Admin JWT 分離治理  
4. 上傳資源非公開，需簽名或權限映射  
5. 快速體驗與登入模式並存，所有 session 來源衝突皆顯式拒絕
