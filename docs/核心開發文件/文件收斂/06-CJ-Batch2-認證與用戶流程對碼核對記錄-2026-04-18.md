## 守衛補強

本輪新增以下 Batch 2 真實性守衛（均已進入 `npm run docs:check:truth`）：

1. `PublicRoute` 回跳規則（`/case/list`、from path 白名單）與 `01-認證與會話` 總覽一致性。
2. `ProtectedRoute` 未水合逾時重定向（`/auth/login`）與 `01-認證與會話` 總覽一致性。
3. `sessions/refresh` 原子旋轉語義（`case_id/pairing_id/session_data`）在 `01-auth-session` 與 `01-認證與會話總覽` 的同步描述。
4. `claim-session` 在登入/註冊後的弱依賴語義（失敗告警但不阻斷主流程）一致性。
5. collaborative 二階段語義（`a_done -> submitted` + `case_id + X-Session-Id`）與 `02-用戶端核心流程總覽` 一致性。
6. `/case/create` pairing gate（非 active 導向 `/profile/pairing`）與流程文檔一致性。
7. pre-case 訪談 trigger（`pre_case` + richness 觸發）與流程文檔一致性。
8. interview 全動作 consent gate（start/respond/skip/end/cancel/retry/resume）與流程文檔、接口文檔一致性。
9. `/notifications` 受保護路由定位與 `02-用戶端核心流程總覽` 一致性。
