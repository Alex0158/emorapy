# Emorapy 測試文檔索引

本目錄只存放測試相關的文檔資產，不存放可執行的測試代碼。

可執行測試代碼位置：

- 後端：`backend/` 下對應測試目錄（依專案實際結構為準）
- 前端：`frontend/` 下對應測試目錄（依專案實際結構為準）

本目錄已按用途拆分如下：

## 00-規範與原則

用途：

- 團隊測試標準
- 長期通用規範

當前重點文件：

- `Emorapy-測試設計鐵律.md`（待建立）

## 01-基線與總覽

用途：

- 測試盤點基線
- 覆蓋現狀
- 任務拆解總表
- 歷史執行結果快照

適合先讀：

- 測試覆蓋現狀基線與缺口矩陣
- 功能點級測試任務拆解表
- 測試明細清單

## 02-專項測試設計

用途：

- 每一個業務專項的測試設計與開發拆解
- 開發前的範圍界定、規則梳理、測試層級安排

## 03-失敗分析與修復

用途：

- 針對既有失敗測試的分組分析
- 測試設計過時、環境問題、測試修正記錄

說明：

- 此類文件多為歷史快照與修復記錄
- 應作為排查歷史問題的依據，不應直接當成當前狀態總表

## 04-業務缺陷報告

用途：

- 只記錄已被測試證明的真實業務代碼/業務行為缺陷
- 不混入純測試問題

說明：

- 若某條測試是有效失敗，且確認不是測試設計問題，應在此立案
- 子目錄 `已解決/` 存放已修復缺陷的報告

## 05-修復方案與風險專題

用途：

- 跨多個缺陷的修復方案總結
- 風險分析
- 不直接屬於單一缺陷單的專題文檔

## 其他

- `.vscode/`：本目錄的編輯器任務與工作區輔助配置（可選），不屬於測試內容本身
