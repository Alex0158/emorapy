# reflect v1.0

## 目的
釐清問題為何反覆出現，形成可執行的改進計劃。

## 何時觸發
- 多輪改進仍有問題
- 需要根因分析與預防策略

## 步驟
1. 回顧多輪改進歷史，使用 PDCA + DevSecOps：計劃缺失？安全偏差？提供完整歷史上下文。
2. 識別根因並提出預防（如採用 Lean、加強規範/自動化）。
3. 如需，更新 ProjectRule。

## 輸出
- Markdown 報告（歷史/根因/計劃）
- confidence: high/medium/low
