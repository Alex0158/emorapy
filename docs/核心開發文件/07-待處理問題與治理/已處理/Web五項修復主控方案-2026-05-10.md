## 4. 詳細設計

### 4.1 Admin lint gate

目標不是關掉 lint，而是讓 lint 規則能正確理解 Tailwind 4 與 Admin CSS。

實作裁決：

1. `@plugin`、`@theme` 若 Biome 當前不能解析，優先在 Biome 配置中用 CSS parser / lint override 明確處理 `frontend-admin/src/index.css`，不得全局關閉 TS/TSX lint。
2. `!important` 的 reduced motion 覆蓋保留，但要用最窄 scope 的 lint override 或重寫為 lint 可接受的全局可訪問性策略。
3. `frontend-admin/src/components/ui/*.tsx` 的 `useImportType` 全部改成 type-only import，不接受長期 warning。
4. 修復後必須同時跑 Admin build，避免 lint 綠但 Tailwind 4 build 失效。

不得做：

1. 不得把 Admin CSS 從 lint scope 中整包隱形排除而不在文檔記錄。
2. 不得為通過 lint 移除 `prefers-reduced-motion`。
3. 不得把 `frontend-admin` 從工作區驗收 gate 移除。

驗收：

```bash
npm run lint --workspace frontend-admin
npm run build --workspace frontend-admin
```

### 4.2 Icon-only button accessibility

實作裁決：

1. 每個 icon-only button 必須補 `type="button"`。
2. accessible name 必須來自 i18n key 或已存在 catalog 文案；不得只寫英文硬編。
3. 關閉提示、稍後再說、清除回覆、關閉引導、關閉 pre-case banner 等不同語義使用不同 key。
4. 高頻元件或修復樣本所在測試檔需新增 role/name 斷言；沒有現成測試時，至少補靜態掃描命令作為 PR 驗收證據。

驗收：

```bash
rg -n "<button(?![^>]*(aria-label|aria-labelledby|>\\s*[^<]))" frontend/src frontend-admin/src --pcre2
npm run test:run --workspace frontend
npm run build --workspace frontend
```

### 4.3 HTML lang / i18n gate

實作裁決：

1. `frontend/index.html` 與 `frontend-admin/index.html` 先把靜態 `lang` 對齊 default locale `zh-TW`。
2. `setLocale()` 成功更新 locale 後同步 `document.documentElement.lang`。
3. 新增或擴充 locale helper，統一日期、時間、數字格式；通知頁不得直接用裸 `toLocaleString()`。
4. 建立 i18n completeness / fallback gate：zh-TW 為 source catalog，en-US 必須覆蓋所有可見 key；任何允許 fallback 的 key 必須有白名單和理由。
5. 缺 key fail policy：非 production 直接 throw `Missing i18n key: <key>`；production 僅允許 `[missing-i18n:<key>]` 標記，不允許可見 key 原文或 humanized fallback。

驗收：

```bash
rg -n "<html lang=|DEFAULT_LOCALE|missing-i18n|toLocaleString\\(|toLocaleDateString\\(" frontend frontend-admin
npm run test:run --workspace frontend
npm run build --workspace frontend
npm run build --workspace frontend-admin
```

### 4.4 Notification semantic interaction

實作裁決：

1. 不再用 `div role="button"` 承接卡片主要操作。
2. 首選方案：卡片正文區改成 `<button type="button">`，只承接 open；CTA、snooze、dismiss 保持獨立 `<Button>` 且不被 open click 包住。
3. 若視覺上需要整卡可點，則拆成標題/摘要的 explicit open button 或 link，不使用 nested interactive。
4. `created_at` 格式化改用 locale-aware helper。
5. 補 Testing Library role/name 測試：Enter / Space 可 open；CTA / snooze / dismiss 不誤觸 open；日期格式跟隨 locale。

驗收：

```bash
npm run test:run --workspace frontend -- Notifications
npm run test:run --workspace frontend
npm run build --workspace frontend
```

### 4.5 Shared API client migration

這是最高風險項，必須用 adapter / strangler pattern。主站 `frontend/src/services/request.ts` 目前包含 token/session、`X-Session-Id`、`X-Locale`、FormData boundary、toast、logout、admin 分流、request cancel 與 retry。這些都是 Web 平台 side effect，不應直接搬進 `packages/api-client`。

實作裁決：

1. 先在 `packages/api-client` 固定純 transport / domain method / response normalize contract，不引入 DOM、React Router、toast、localStorage、sessionStorage。
2. 在 `frontend/src/services` 建 Web adapter，把 shared client 的純 response / error 映射回現有 toast、store、router、storage 行為。
3. 第一批只遷移低副作用 read-only 或已被 App M1-M5 使用的穩定 domain method；每批最多一個 domain。
4. 高風險 domain 順序不得倒置：notification/read-only -> profile/interview read -> quick/session -> case/judgment -> chat stream / upload。
5. chat stream、upload、auth claim、session refresh 不得直接替換；必須先有 contract tests 和現有 Web 回歸測試。
6. 每批遷移後保留相同 public service export，避免頁面層一次性改動。

2026-05-25 回寫核驗：主站已把 M1 `session` / `auth` / quick case、M2 `interview` / `psychProfile`、M3 chat REST、M4 `pairing` / formal case / judgment / reconciliation / execution、M5 notifications / media upload 與 AI stream pure helper 接到 `@cj/api-client`；Web `request.ts` 仍保留 token/session/locale/FormData/toast/cancel/retry/SSE side effect seam。對應 `@cj/api-client` M1-M5 / AI stream tests、frontend service tests、frontend build 與 cached diff check 已通過；Admin、Mobile、release evidence 不併入本 shared-client 批次。

驗收：

```bash
rg -n "from '@cj/api-client'|from \"@cj/api-client\"" frontend/src frontend-admin/src packages/api-client/src
npm run test:m1 --workspace @cj/api-client
npm run test:m2 --workspace @cj/api-client
npm run test:m4 --workspace @cj/api-client
npm run test:m5 --workspace @cj/api-client
npm run test:run --workspace frontend
npm run build --workspace frontend
npm run build --workspace frontend-admin
```

## 5. 測試驅動規則

每個階段必須先補或確認測試，再改實作：

| 階段 | 必需測試 |
| --- | --- |
| Admin lint | lint failure regression、Admin build |
| Icon-only button | role/name query 或靜態掃描清零；關鍵元件單測 |
| HTML lang / i18n | locale switch 後 `document.documentElement.lang`；catalog completeness script；locale formatting helper 單測 |
| Notification semantic | open / action 分離、keyboard、focus order、locale date formatting |
| API client migration | shared client contract tests、Web adapter tests、對應 domain service tests、完整 frontend test run |

任何階段如果引入新缺陷或衍生問題，必須先記錄到對應待辦，修完並重新跑該階段 gate，不能用後續階段掩蓋。
