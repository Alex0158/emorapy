# 依賴注入設計（可選，待後續觸發）

**日期**：2026-02-06  
**關聯**：[per-file-audit-20260206.md](per-file-audit-20260206.md) §5、§7  
**結論**：目前 MVP 維持「直接依賴具體實例」；本設計供「當需提升可測性與 SOLID（DIP）」時參考。

---

## 1. 現狀

- **依賴方式**：Controller / Service 直接 `import { caseService } from '../services/case.service'` 等，使用匯出的單例實例。
- **優點**：簡單、無額外組裝、無循環依賴問題（case 與 judgment 間已用動態 import 規避）。
- **缺點**：單測需透過 jest.mock 替換整個模組，無法僅替換單一依賴；違反 DIP（依賴反轉：應依賴抽象而非具體）。

---

## 2. 目標（若引入 DI）

- **可測性**：Controller 與 Service 單測時可注入 mock 實作，無需 mock 整個模組。
- **SOLID（DIP）**：業務層依賴介面（如 `ICaseService`），具體實作在應用組裝時注入。

---

## 3. 可選方案

**方案 A：介面 + 建構子注入，應用層組裝**

- 為關鍵服務定義介面（例：`ICaseService`、`IJudgmentService`、`IAIService`、`ISessionService`）。
- Controller / 會用到這些服務的 Service 改為「建構子接收介面」，不在內部 import 具體實例。
- 在 `app.ts` 或單一 `container.ts` 中建立具體實例並注入到 Controller（例：`new CaseController(caseService, judgmentService)`）。
- 單測時 `new CaseController(mockCaseService, mockJudgmentService)`。

**方案 B：輕量容器（如 tsyringe / 自寫 map）**

- 註冊介面 → 實作；在啟動時 resolve 並注入到 Controller。
- 單測前替換註冊為 mock。

**方案 C：維持現狀**

- 繼續使用 jest.mock 替換模組；不引入介面與 DI。對當前 MVP 可接受。

---

## 4. 建議優先抽象對象

| 服務 | 依賴方（示例） | 抽象優先級 |
|------|----------------|------------|
| CaseService | CaseController, updateCase 內 judgmentService | 高（核心流程） |
| JudgmentService | CaseController, JudgmentController | 高 |
| AIService | CaseService, JudgmentService, ReconciliationService, cleanup.job | 高 |
| SessionService | CaseService, EvidenceController, cleanup.job | 中 |

先從 1～2 個 Controller（如 CaseController）試行介面 + 建構子注入，再逐步推廣。

---

## 5. 風險與注意

- **循環依賴**：若 CaseService 依賴 JudgmentService、JudgmentService 依賴 CaseService，需保留動態 import 或事件/佇列解耦，DI 本身不解決循環。
- **改動面**：需修改 Controller 建構方式與路由掛載處（由 `new CaseController()` 改為從 container 取得已注入之實例），影響所有使用該服務的檔案。
- **型別**：TypeScript 介面在運行時不存在，需約定「同一 token 僅綁定一實作」或使用字串/symbol token 做容器解析。

---

## 6. 小結

| 項目 | 建議 |
|------|------|
| 當前 | 不實施 DI，維持直接 import 單例；單測繼續用 jest.mock |
| 觸發條件 | 當單測難以撰寫（mock 過重）或希望明確 DIP 時，再引入介面 + 建構子注入 |
| 記錄 | 本設計納入審計後續，需時可與 [per-file-audit-20260206.md](per-file-audit-20260206.md) 一併複查 |

---
