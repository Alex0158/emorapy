# 報告 06：CI 全棧管理台 E2E 環境（Admin E2E）

## 1. 環境定位

- 目標：在 CI 中驗證管理功能的全鏈路可用性（DB + backend + frontend + browser）。
- 入口：`.github/workflows/ci.yml` -> `admin-e2e` job。

## 2. 執行流程（CI）

1. 啟 Postgres service（`postgres:16`）
2. backend 安裝依賴
3. `prisma db push --accept-data-loss` + `npm run prisma:seed`
4. 啟 backend dev server（`PORT=3001`）
5. 啟 frontend dev server（`5173`，注入 `VITE_API_BASE_URL`）
6. e2e 專案安裝 + `playwright install chromium`
7. 執行 `npm test`（Playwright）

## 3. 關鍵變量

- backend seed/start
  - `DATABASE_URL=postgresql://.../mbc_dev`
  - `JWT_SECRET`
  - `OPENAI_API_KEY=sk-dev-test-placeholder`
  - `ENABLE_SCHEDULED_JOBS=false`
  - `ADMIN_SEED_*` / `LIMITED_ADMIN_SEED_*`
- frontend
  - `VITE_API_BASE_URL=http://127.0.0.1:3001/api/v1`
- e2e
  - `E2E_STRICT=true`
  - `E2E_BASE_URL=http://127.0.0.1:5173`
  - `E2E_ADMIN_EMAIL/PASSWORD`
  - `E2E_LIMITED_ADMIN_EMAIL/PASSWORD`

## 4. 服務依賴（本環境）

- PostgreSQL：CI 服務容器
- Playwright + Chromium：瀏覽器自動化
- OpenAI：placeholder（實際走 mock/dev path）
- Redis/SMTP/Slack：非必要

## 5. 驗證清單

- backend/frontend wait-for-ready 成功
- Playwright 全綠
- 失敗時可讀取 `backend-e2e.log`、`frontend-e2e.log` 與 artifacts

## 6. 風險與改進

- CI 中 backend 以 `NODE_ENV=development` 啟動，與 production 行為仍有差異。
- 測試主要覆蓋管理台路徑，對主前台 E2E 覆蓋有限。
- 建議補一條 production-like smoke（`NODE_ENV=production` + build artifact）以降低環境偏差風險。
